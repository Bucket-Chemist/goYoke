# GOgent-035: Implement Failure Detection

## Description
Detect failures from exit codes, error keywords in output, and explicit success=false flags.

## Implementation Intention
Accurately identify tool execution failures using multiple signals to enable consecutive failure tracking.

## Intended End State
- DetectFailure() analyzes tool response for failure signals
- Detects explicit success=false, non-zero exit codes, error keywords
- Returns FailureDetection with IsError and ErrorType
- Test coverage ≥80%

## Dependencies
- GOgent-034: PostToolUse event parsing complete

## Acceptance Criteria
- [ ] `DetectFailure()` detects explicit success=false
- [ ] Detects non-zero exit codes
- [ ] Detects specific Python error types (TypeError, ValueError, etc.)
- [ ] Detects generic error keywords (error, failed, exception)
- [ ] Returns no error for successful operations
- [ ] Tests cover all detection paths
- [ ] `go test ./pkg/memory` passes with ≥80% coverage

## Implementation Details

### File Created
- `pkg/memory/failure_detection.go`

### Structs
```go
// FailureDetection represents detected failure information
type FailureDetection struct {
	IsError    bool
	ErrorType  string
	ErrorMatch string // The specific error keyword that matched
}
```

### Function Signature
```go
func DetectFailure(event *PostToolUseEvent) *FailureDetection
func detectErrorKeywords(text string) (string, string)
func formatExitCode(code int) string
```

### Detection Priority
1. Explicit success=false → "explicit_failure"
2. Non-zero exit code → "exit_code_{N}"
3. Specific error types (TypeError, ValueError, etc.) → error type name
4. Generic keywords (error, failed, exception) → "generic_error", "generic_failure", etc.

### Test Requirements
- Test: Explicit success=false
- Test: Exit code 1, 2, 127
- Test: TypeError in output
- Test: Generic "error" keyword
- Test: Successful operation (no detection)
- Test: All specific error types (ValueError, ImportError, SyntaxError, etc.)

Accurate failure detection is critical. False positives block valid work; false negatives miss real issues.
