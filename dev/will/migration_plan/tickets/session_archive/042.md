# GOgent-042: Behavioral Simulation CI/CD Pipeline

## Description

Extend the existing simulation testing infrastructure with **behavioral testing** that validates the complete learning pipeline across multi-turn sessions, cross-session memory, and concurrent agent scenarios. This completes the testing story for the 037-041 feature series.

## Problem Statement

Current simulation testing validates:
- ✅ Individual hook behaviors (PreToolUse, PostToolUse, SessionEnd)
- ✅ Deterministic fixture scenarios
- ✅ Fuzz testing with invariant checking

What's **missing**:
- ❌ Multi-turn session simulation (tool call sequences)
- ❌ Cross-session context propagation (handoff → next session)
- ❌ Complete learning pipeline validation (capture → classify → aggregate)
- ❌ Concurrent agent chaos testing
- ❌ User intent lifecycle (capture → classify → outcome)

## Implementation Intention

Create a **multi-level CI/CD pipeline** that tests the GOgent system at increasing levels of realism:

| Level | Name | Trigger | Coverage |
|-------|------|---------|----------|
| 1 | Unit Invariants | Every push | Hook behaviors |
| 2 | Session Replay | Every push | Multi-turn sequences |
| 3 | Behavioral Properties | Every PR | System invariants |
| 4 | Chaos Testing | Weekly | Concurrent agents |
| 5 | Week Simulation | Weekly | Weekly aggregation |

## Intended End State

- Session replay harness executes recorded sessions against hooks
- Behavioral invariants validate system properties across arbitrary inputs
- Chaos testing catches race conditions in concurrent scenarios
- Week simulation validates 041b aggregation logic
- CI/CD pipeline runs appropriate tests per trigger
- All tests GREEN required for merge

### Session Event Schema

```go
// SessionEvent represents a single tool use event in a session
type SessionEvent struct {
    Timestamp    int64                  `json:"ts"`
    HookType     string                 `json:"hook"`         // "PreToolUse", "PostToolUse"
    ToolName     string                 `json:"tool_name"`
    ToolInput    map[string]interface{} `json:"tool_input"`
    ToolResponse map[string]interface{} `json:"tool_response,omitempty"`
    Success      bool                   `json:"success"`
}

// Session represents a complete agent session for replay
type Session struct {
    ID           string                 `json:"session_id"`
    Events       []SessionEvent         `json:"events"`
    Expected     SessionExpectations    `json:"expected"`
}

// SessionExpectations defines what should result from replaying the session
type SessionExpectations struct {
    SharpEdgesCreated  int                      `json:"sharp_edges_created"`
    BlockingResponses  int                      `json:"blocking_responses"`
    UserIntents        int                      `json:"user_intents"`
    HandoffCreated     bool                     `json:"handoff_created"`
    FileChecks         map[string][]string      `json:"file_checks,omitempty"`
}
```

### Behavioral Invariants

```go
package harness

// BehavioralInvariants define properties that must hold across any valid session
var BehavioralInvariants = []BehavioralInvariant{
    {
        ID:   "B1",
        Name: "sharp_edges_have_required_fields",
        Check: func(ctx *SessionContext) (bool, string) {
            for _, edge := range ctx.CapturedSharpEdges {
                if edge.ErrorType == "" {
                    return false, "sharp edge missing error_type"
                }
                if edge.ConsecutiveFailures < 3 {
                    return false, fmt.Sprintf("sharp edge has consecutive_failures=%d, want >=3",
                        edge.ConsecutiveFailures)
                }
                if edge.Timestamp == 0 {
                    return false, "sharp edge missing timestamp"
                }
            }
            return true, ""
        },
    },
    {
        ID:   "B2",
        Name: "blocking_at_threshold",
        Check: func(ctx *SessionContext) (bool, string) {
            // Verify blocking happens exactly at threshold, not before
            for _, sequence := range ctx.FailureSequences {
                blockedAt := sequence.BlockedAtCount
                threshold := ctx.Config.MaxFailures

                if blockedAt > 0 && blockedAt < threshold {
                    return false, fmt.Sprintf("blocked at %d, threshold is %d",
                        blockedAt, threshold)
                }
            }
            return true, ""
        },
    },
    {
        ID:   "B3",
        Name: "composite_key_isolation",
        Check: func(ctx *SessionContext) (bool, string) {
            // Failures on different files should NOT compound
            counts := make(map[string]int)
            for _, failure := range ctx.AllFailures {
                key := failure.File + ":" + failure.ErrorType
                counts[key]++
            }

            // Cross-file failures should be separate
            for key, count := range counts {
                // If this key triggered a sharp edge, verify count matches
                if edge, found := ctx.SharpEdgesByKey[key]; found {
                    if edge.ConsecutiveFailures != count {
                        return false, fmt.Sprintf("count mismatch for %s: tracked=%d, edge=%d",
                            key, count, edge.ConsecutiveFailures)
                    }
                }
            }
            return true, ""
        },
    },
    {
        ID:   "B4",
        Name: "intent_classification_coverage",
        Check: func(ctx *SessionContext) (bool, string) {
            for _, intent := range ctx.CapturedIntents {
                if intent.Category == "" {
                    return false, fmt.Sprintf("intent missing category: %q", intent.Question)
                }
            }
            return true, ""
        },
    },
    {
        ID:   "B5",
        Name: "handoff_preserves_schema",
        Check: func(ctx *SessionContext) (bool, string) {
            if ctx.Handoff == nil {
                return true, "" // No handoff expected
            }
            if ctx.Handoff.SchemaVersion != "1.1" {
                return false, fmt.Sprintf("schema version %s, want 1.1",
                    ctx.Handoff.SchemaVersion)
            }
            return true, ""
        },
    },
}
```

### Chaos Runner

```go
package harness

import (
    "sync"
    "time"
)

// ChaosConfig controls chaos testing parameters
type ChaosConfig struct {
    NumAgents         int           // Number of concurrent agents
    FailuresPerAgent  int           // Failures each agent generates
    Duration          time.Duration // Max test duration
    FailureVariety    int           // Number of distinct error types
}

// ChaosRunner simulates concurrent agents with failures
type ChaosRunner struct {
    config      ChaosConfig
    tracker     *FailureTracker
    results     []ChaosResult
    resultsMu   sync.Mutex
}

// Run executes chaos testing
func (c *ChaosRunner) Run() (*ChaosReport, error) {
    var wg sync.WaitGroup

    for i := 0; i < c.config.NumAgents; i++ {
        wg.Add(1)
        go func(agentID int) {
            defer wg.Done()
            c.runAgent(agentID)
        }(i)
    }

    wg.Wait()

    return c.buildReport()
}

func (c *ChaosRunner) runAgent(agentID int) {
    file := fmt.Sprintf("agent_%d_file.go", agentID)
    errorType := fmt.Sprintf("AgentError%d", agentID % c.config.FailureVariety)

    for i := 0; i < c.config.FailuresPerAgent; i++ {
        key := FailureKey{FilePath: file, ErrorType: errorType}
        count, _ := c.tracker.LogFailure(key, "Bash")

        c.resultsMu.Lock()
        c.results = append(c.results, ChaosResult{
            AgentID:   agentID,
            Iteration: i,
            Count:     count,
        })
        c.resultsMu.Unlock()

        // Small sleep to increase interleaving
        time.Sleep(time.Microsecond * time.Duration(rand.Intn(100)))
    }
}

func (c *ChaosRunner) buildReport() (*ChaosReport, error) {
    report := &ChaosReport{
        TotalEvents: len(c.results),
    }

    // Verify each agent's final count
    agentCounts := make(map[int]int)
    for _, r := range c.results {
        agentCounts[r.AgentID] = max(agentCounts[r.AgentID], r.Count)
    }

    for agentID, count := range agentCounts {
        expected := c.config.FailuresPerAgent
        if count != expected {
            report.Errors = append(report.Errors,
                fmt.Sprintf("agent %d: count=%d, expected=%d", agentID, count, expected))
        }
    }

    report.Passed = len(report.Errors) == 0
    return report, nil
}
```

### Week Simulator

```go
package harness

import (
    "time"
)

// WeekSimulator generates and runs a synthetic week of sessions
type WeekSimulator struct {
    config    WeekConfig
    generator *SessionGenerator
}

// WeekConfig controls week simulation
type WeekConfig struct {
    SessionsPerDay    int
    DaysInWeek        int
    IntentsPerSession int
    FailuresPerSession int
    StartTime         time.Time
}

// GenerateWeek creates a synthetic week of sessions
func (w *WeekSimulator) GenerateWeek() []Session {
    var sessions []Session

    for day := 0; day < w.config.DaysInWeek; day++ {
        dayStart := w.config.StartTime.AddDate(0, 0, day)

        for i := 0; i < w.config.SessionsPerDay; i++ {
            sessionStart := dayStart.Add(time.Hour * time.Duration(9+i))
            session := w.generator.GenerateSession(SessionParams{
                StartTime:      sessionStart,
                NumEvents:      20 + rand.Intn(30),
                FailureRate:    0.1,
                IntentRate:     0.05,
                Categories:     []string{"routing", "tooling", "approval", "workflow"},
            })
            sessions = append(sessions, session)
        }
    }

    return sessions
}

// RunWeek executes all sessions and validates aggregation
func (w *WeekSimulator) RunWeek(sessions []Session) (*WeekReport, error) {
    ctx := &SimulationContext{}

    // Run each session
    for _, session := range sessions {
        if err := w.runSession(ctx, session); err != nil {
            return nil, fmt.Errorf("session %s: %w", session.ID, err)
        }
    }

    // Run weekly aggregation
    aggregation, err := session.AggregateWeeklyIntents(
        ctx.AllIntents,
        w.config.StartTime,
        w.config.StartTime.AddDate(0, 0, 7),
    )
    if err != nil {
        return nil, fmt.Errorf("aggregation: %w", err)
    }

    // Validate aggregation
    report := &WeekReport{
        TotalSessions:     len(sessions),
        TotalIntents:      len(ctx.AllIntents),
        CategoryBreakdown: aggregation.CategoryDistribution,
        RecurringPatterns: len(aggregation.RecurringPreferences),
    }

    // Check expected patterns
    if aggregation.TotalIntents != len(ctx.AllIntents) {
        report.Errors = append(report.Errors,
            fmt.Sprintf("intent count mismatch: aggregated=%d, actual=%d",
                aggregation.TotalIntents, len(ctx.AllIntents)))
    }

    report.Passed = len(report.Errors) == 0
    return report, nil
}
```

```yaml
name: Behavioral Simulation

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]
  schedule:
    - cron: '0 4 * * 0'  # Weekly Sunday 4am UTC
  workflow_dispatch:
    inputs:
      chaos_agents:
        description: 'Number of agents for chaos testing'
        default: '10'
      chaos_iterations:
        description: 'Iterations per agent'
        default: '100'

env:
  GO_VERSION: '1.25'

jobs:
  # ===========================================================================
  # Level 1: Unit Invariants (fast, every push)
  # ===========================================================================
  unit-invariants:
    name: Unit Invariants
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: ${{ env.GO_VERSION }}

      - name: Build CLIs
        run: make build-validate build-archive build-sharp-edge

      - name: Run Deterministic Tests
        run: make test-simulation-deterministic

      - name: Run Unit Tests
        run: go test -v -race ./test/simulation/harness/...

  # ===========================================================================
  # Level 2: Session Replay (fast, every push)
  # ===========================================================================
  session-replay:
    name: Session Replay
    runs-on: ubuntu-latest
    needs: unit-invariants
    timeout-minutes: 15

      - uses: actions/checkout@v4

        with:
          go-version: ${{ env.GO_VERSION }}

      - name: Build CLIs
        run: make build-validate build-archive build-sharp-edge

        run: make test-simulation-replay

        uses: actions/upload-artifact@v4
        if: always()

          name: replay-reports-${{ github.run_number }}

  # Level 3: Behavioral Properties (thorough, PRs only)
  # ===========================================================================

    name: Behavioral Properties
    runs-on: ubuntu-latest
    needs: session-replay

    steps:

          go-version: ${{ env.GO_VERSION }}

        run: make test-simulation-behavioral

      - name: Upload Behavioral Reports
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: behavioral-reports-${{ github.run_number }}
          path: test/simulation/reports/behavioral-*.json
          retention-days: 14

      - name: Comment on PR
        if: always()
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const reports = fs.readdirSync('test/simulation/reports')
              .filter(f => f.startsWith('behavioral-'));

            let summary = '## Behavioral Simulation Results\n\n';
            for (const report of reports) {
              const data = JSON.parse(fs.readFileSync(`test/simulation/reports/${report}`));
              summary += `- **${data.name}**: ${data.passed ? '✅' : '❌'}\n`;
              if (!data.passed && data.errors) {
                for (const err of data.errors) {
                  summary += `  - ${err}\n`;
                }
              }
            }

            github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: summary
            });

  # ===========================================================================
  # Level 4: Chaos Testing (weekly or manual)
  # ===========================================================================
  chaos-testing:
    name: Chaos Testing
    runs-on: ubuntu-latest
    if: github.event_name == 'schedule' || github.event_name == 'workflow_dispatch'
    timeout-minutes: 30

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: ${{ env.GO_VERSION }}

      - name: Build CLIs
        run: make build-validate build-archive build-sharp-edge

      - name: Run Chaos Tests
        env:
          CHAOS_AGENTS: ${{ github.event.inputs.chaos_agents || '10' }}
          CHAOS_ITERATIONS: ${{ github.event.inputs.chaos_iterations || '100' }}
        run: make test-simulation-chaos

      - name: Upload Chaos Reports
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: chaos-reports-${{ github.run_number }}
          path: test/simulation/reports/chaos-*.json
          retention-days: 30

  # ===========================================================================
  # Level 5: Week Simulation (weekly or manual)
  # ===========================================================================
  week-simulation:
    name: Week Simulation
    runs-on: ubuntu-latest
    if: github.event_name == 'schedule' || github.event_name == 'workflow_dispatch'
```
