### Implementation Pattern
```go
func ExtractFunctionFromStackTrace(errorOutput string) string {
	// Python pattern: "  File "path", line N, in functionName"
	pythonPattern := regexp.MustCompile(`in\s+(\w+)`)

	// Go pattern: "path.functionName(args)"
	goPattern := regexp.MustCompile(`([a-zA-Z_]\w*)\(`)

	// Try Python first
	if matches := pythonPattern.FindStringSubmatch(errorOutput); len(matches) > 1 {
		return matches[1]
	}

	// Try Go
	if matches := goPattern.FindStringSubmatch(errorOutput); len(matches) > 1 {
		return matches[1]
	}

	// Couldn't extract, return empty (graceful degradation)
	return ""
}
```

### Integration Example
```go
// In gogent-sharp-edge CLI main.go

detection := memory.DetectFailure(event)
if detection.IsError {
	// Extract function if possible
	resp := event.ParseToolResponse()
	function := memory.ExtractFunctionFromStackTrace(resp.Output + " " + resp.Error)

	// Create failure key with function context
	key := memory.FailureKey{
		FilePath:  event.ExtractFilePath(),
		ErrorType: detection.ErrorType,
		Function:  function,
	}

	tracker.LogFailure(key, event.ToolName)
	count, _ := tracker.CountRecentFailures(key)
	// ...
}
```

### Test Requirements
- Test: Python traceback with function name
  ```
  Traceback (most recent call last):
    File "test.py", line 10, in calculateTotal
      result = x / y
  ZeroDivisionError: division by zero
  ```
  → Extracts "calculateTotal"

- Test: Go panic with function
  ```
  panic: runtime error: invalid memory address
  goroutine 1 [running]:
  main.processData(0xc000...)
  ```
  → Extracts "processData"

- Test: Generic error without stack trace
  ```
  error: operation failed
  ```
  → Returns "" (graceful degradation)

- Test: Empty error output → Returns ""

### Expected Behavior
With function extraction:
```
File: src/main.go
Function: calculateTotal
ErrorType: ZeroDivisionError
Failures: 3 → Trigger sharp edge
```

Without function extraction (degraded):
```
File: src/main.go
Function: (empty)
ErrorType: ZeroDivisionError
Failures: 3 → Trigger sharp edge
```

Both work, function just provides finer granularity when available.

## Why This Matters
Function-level tracking prevents false positives in large files. A 500-line file might have 3 different functions with the same error type. Function extraction ensures only genuine loops (same error in same function) trigger detection.

**Optional Enhancement**: If this proves difficult or unreliable during implementation, it can be deferred. The file+error composite key (GOgent-036b) already provides significant improvement over file-only tracking.
