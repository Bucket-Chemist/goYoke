### ExtractAttemptedChange Function
```go
func ExtractAttemptedChange(event *PostToolUseEvent) string {
	switch event.ToolName {
	case "Edit":
		oldStr, _ := event.ToolInput["old_string"].(string)
		newStr, _ := event.ToolInput["new_string"].(string)

		// Truncate for display
		if len(oldStr) > 60 {
			oldStr = oldStr[:60] + "..."
		}
		if len(newStr) > 60 {
			newStr = newStr[:60] + "..."
		}

		return fmt.Sprintf("%s → %s", oldStr, newStr)

	case "Write":
		content, _ := event.ToolInput["content"].(string)
		lines := strings.Split(content, "\n")

		// Show first 3 lines
		preview := strings.Join(lines[:min(3, len(lines))], "\n")
		if len(lines) > 3 {
			preview += "\n..."
		}

		return fmt.Sprintf("Write content:\n%s", preview)

	case "Bash":
		command, _ := event.ToolInput["command"].(string)
		return fmt.Sprintf("Command: %s", command)

	default:
		return ""
	}
}
```

## Why This Matters
Knowing what was attempted is often more valuable than knowing what failed. "Tried to change interface{} to bool" immediately suggests a type assertion problem. Without this context, reviewer must reconstruct intent from code alone.
