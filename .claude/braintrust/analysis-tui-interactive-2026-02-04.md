## Executive Summary

The TUI's inability to handle Claude's interactive features stems from a fundamental architectural gap: **the TUI treats the Claude SDK as a unidirectional event stream when it implements a bidirectional control channel**. The event loop in `useClaudeQuery.ts` processes only 3 of 16+ SDK message types, missing the entire control plane that handles permissions, slash commands, and plan mode.

The solution path is clear and non-invasive: leverage SDK callback options (`canUseTool`, `permissionPromptToolName`) to route control-plane interactions into the existing modal queue system, which already works for MCP tools. This approach requires minimal code changes (approximately 50-80 lines) while enabling full interactive feature support.

**Key recommendation**: Implement `canUseTool` callback as Phase 1, which immediately enables permission prompts using the existing modal infrastructure. Slash commands and plan mode require additional TUI-side interception logic in Phase 2.

```typescript
const eventStream = query({
  prompt: message,
  options: {
    // Load goYoke settings (hooks, CLAUDE.md, etc.)
    settingSources: ['user', 'project', 'local'],
    mcpServers: [mcpServer] as unknown as NonNullable<Parameters<typeof query>[0]["options"]>["mcpServers"],

    // NEW: Permission handling callback
    canUseTool: async (toolName: string, input: unknown, context: { toolUseID: string }) => {
      // Destructive tools get explicit confirmation
      const destructiveTools = ['Bash', 'Write', 'Edit', 'MultiEdit'];
      const isDestructive = destructiveTools.includes(toolName);

      // Enqueue permission modal
      const result = await useStore.getState().enqueue({
        type: 'confirm',
        payload: {
          action: `Allow Claude to use ${toolName}?`,
          destructive: isDestructive,
        },
        timeout: 60000, // 60s timeout
      });

      // Return SDK-expected response format
      if (result.type === 'confirm' && result.confirmed) {
        return { behavior: 'allow' as const, toolUseID: context.toolUseID };
      } else {
        return {
          behavior: 'deny' as const,
          message: 'User denied permission',
          toolUseID: context.toolUseID
        };
      }
    },

    // Default permission mode (can be adjusted)
    permissionMode: 'default',
  },
});
```
