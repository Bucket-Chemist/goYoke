# Explore Skill - Subagent Context

## Quick Reference
- **Purpose:** Reconnaissance + Planning wrapper
- **Output:** User-approved plan via architect
- **Planner:** architect (Sonnet)

## Workflow Summary

```
User Goal → Scout → Architect → User Approval → Execute
```

## Scout Invocation

```bash
find <target> -type f \( -name "*.py" -o -name "*.R" -o -name "*.md" \) | \
```

## Architect Delegation

```javascript
Task({
  description: "Create implementation plan",
  subagent_type: "Explore",
  model: "sonnet",
  prompt: "AGENT: architect\n\n1. TASK: Create plan for <goal>\n..."
})
```

## Key Outputs

1. **specs.md** - Decision record (created by architect)
2. **TaskCreate** - Actionable tasks (called by architect)

## Cost Ceilings

| Phase | Max |
|-------|-----|
| Scout | $0.01 |
| Architect | $0.15 |
| Total pre-execution | $0.20 |

## Remember

- You coordinate, you don't implement
- Scout first for unknown scope
- Architect creates the plan, not you
- User must approve before execution
- Routing is enforced by hooks, not prose
