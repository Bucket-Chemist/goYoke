# Refine Plan Skill

## Purpose

Enrich an implementation plan using the plan-harmonizer agent (3-pass compiler):

1. **Pass 1 — Fix Mapping:** Maps each review finding to the task(s) it affects (file intersection + semantic fallback)
2. **Pass 2 — Dependency Validation:** Verifies the dependency graph against the codebase; detects implicit dependencies
3. **Pass 3 — Readiness Scoring:** Produces a 0–100 readiness score for team-run automation

Requires a prior `/review-plan` run that produced `review-metadata.json` with an `issue_register`. Does **not** auto-invoke `/review-plan`.

## Invocation

- `/refine-plan` — Use `.claude/tmp/implementation-plan.json` + `.claude/tmp/review-metadata.json`
- `/refine-plan <plan.json> <review-metadata.json>` — Explicit paths for both files
- `/refine-plan <plan.json>` — Error: "No review found. Run /review-plan first."
- `/refine-plan --promote-deps` — Promote all implicit deps with confidence >= 0.8 to `blocked_by`
- `/refine-plan --promote-dep <task-id>:<depends-on-id>` — Promote a single specific dependency

## When to Use

**Good Use Cases:**

- After `/review-plan` produces `review-metadata.json`, before handing off to team-run automation
- To get a readiness score that determines if `goyoke-plan-impl` should proceed
- To surface which tasks need rework based on review findings

**When to Skip:**

- No review has been run yet (need `review-metadata.json` with `issue_register`)
- Trivial plan with no review findings to map

**Promotion Use Cases:**

- After reviewing implicit deps in the refine output, promote confirmed ones: `/refine-plan --promote-deps`
- To promote a single specific dependency: `/refine-plan --promote-dep task-006:task-004`

## Cost & Performance

- **Cost:** $0.05-0.15 per run (sonnet+thinking, 10K thinking budget)
- **Duration:** 1-2 minutes (3 sequential passes)
- **Codebase calls:** Up to 15 (verifying file/type existence in Pass 2)

## Workflow

### Phase 1: Validate Inputs

**Step 1:** Resolve file paths from args

```javascript
const args = trimmedArgs.split(/\s+/).filter(Boolean);

let planPath, reviewPath;

if (args.length === 0) {
  // Default: .claude/tmp/ first
  planPath = ".claude/tmp/implementation-plan.json";
  reviewPath = ".claude/tmp/review-metadata.json";
} else if (args.length === 1) {
  // Only plan given — cannot proceed without review
  Output(`[Error] No review found. Run /review-plan first.`);
  Output(`\nThen: /refine-plan ${args[0]} .claude/tmp/review-metadata.json`);
  exit();
} else {
  planPath = args[0];
  reviewPath = args[1];
}
```

**Step 2:** Validate both files exist and are valid JSON

```javascript
let planContent, reviewContent;

try {
  planContent = Read({ file_path: planPath });
} catch {
  Output(`[Error] Plan file not found: ${planPath}`);
  Output(`\nUsage: /refine-plan [plan.json] [review-metadata.json]`);
  Output(`Default: /refine-plan uses .claude/tmp/implementation-plan.json`);
  exit();
}

try {
  reviewContent = Read({ file_path: reviewPath });
} catch {
  Output(`[Error] Review file not found: ${reviewPath}`);
  Output(`\nNo review found. Run /review-plan first.`);
  exit();
}

let plan, review;
try {
  plan = JSON.parse(planContent);
} catch (error) {
  Output(`[Error] Plan file is not valid JSON: ${planPath}`);
  Output(`Error: ${error.message}`);
  exit();
}
try {
  review = JSON.parse(reviewContent);
} catch (error) {
  Output(`[Error] Review file is not valid JSON: ${reviewPath}`);
  Output(`Error: ${error.message}`);
  exit();
}

// Validate review has required fields
if (!review.issue_register || !Array.isArray(review.issue_register)) {
  Output(`[Error] Review file missing issue_register array: ${reviewPath}`);
  Output(`\nExpected: { issue_register: [...], verdict: "...", ... }`);
  Output(`Generated by /review-plan (staff-architect-critical-review agent).`);
  exit();
}
```

**Step 3:** Derive output path

```javascript
// Enriched plan sits alongside the original
const planDir = planPath.replace(/\/[^/]+$/, "") || ".";
const planBasename = planPath.replace(/.*\//, "").replace(/\.json$/, "");
```

### Phase 3: Collect and Present Results

**Step 6:** Collect agent result

```javascript
const result = mcp__goyoke_interactive__get_agent_result({
  agent_id: spawnResult.agent_id,
});
```

**Step 7:** Parse and validate result

```javascript
let harmonized;
try {
  harmonized = JSON.parse(result.output);
} catch (error) {
  Output(`[Error] Plan-harmonizer returned non-JSON output: ${error.message}`);
  Output(`\nOriginal plan unchanged: ${planPath}`);
  Output(`\nRetry: /refine-plan ${planPath} ${reviewPath}`);
  exit();
}

if (harmonized.status !== "complete") {
  Output(`[Error] Plan-harmonizer did not complete. status=${harmonized.status}`);
  Output(`\nOriginal plan unchanged: ${planPath}`);
  exit();
}
```

**Step 8:** Write enriched plan

```javascript
// Write to enrichedPlanPath alongside original
// If enrichedPlanPath is NOT under .claude/, use Write() directly:
Write({ file_path: enrichedPlanPath, content: JSON.stringify(harmonized.enriched_plan, null, 2) });
// If enrichedPlanPath IS under .claude/tmp/, use Bash:
// Bash({ command: `echo '${JSON.stringify(harmonized.enriched_plan)}' | scripts/claude-edit.sh --write ${enrichedPlanPath}` });
```

**Step 9:** Present results

```javascript
const score = harmonized.readiness_score;
const mapping = harmonized.mapping_report;
const warnings = harmonized.warnings || [];
const meta = harmonized.metadata || {};

// Determine status label
let statusLabel;
if (score === null) {
  statusLabel = "INCOMPLETE (truncated pass — see warnings)";
} else if (score.total >= 70) {
  statusLabel = "READY";
} else if (score.total >= 50) {
  statusLabel = "READY WITH CAVEATS";
} else {
  statusLabel = "NOT READY";
}

Output("\n" + "=".repeat(80));
Output("REFINE PLAN COMPLETE");
Output("=".repeat(80));

// Readiness score
Output("\nREADINESS SCORE");
Output("-".repeat(40));
if (score === null) {
  Output(`Status: ${statusLabel}`);
} else {
  Output(`Total:  ${score.total}/100 — ${statusLabel}`);
  Output(`  fix_coverage:      ${score.dimensions.fix_coverage}/5  (weight ×7)`);
  Output(`  dep_validity:      ${score.dimensions.dep_validity}/5  (weight ×7)`);
  Output(`  schema_completeness: ${score.dimensions.schema_completeness}/5  (weight ×6)`);
}

// Mapping summary
Output("\nFINDING → TASK MAPPING");
Output("-".repeat(40));
Output(`Findings: ${mapping.total_findings} total, ${mapping.mapped} mapped, ${mapping.unmapped} unmapped`);

if (mapping.mappings && mapping.mappings.length > 0) {
  Output("\n  ID       Tasks           Method              Confidence");
  Output("  " + "-".repeat(64));
  for (const m of mapping.mappings) {
    const tasks = (m.mapped_tasks || []).join(", ").padEnd(16);
    const method = (m.method || "").padEnd(20);
    const conf = m.confidence != null ? m.confidence.toFixed(2) : "n/a";
    Output(`  ${m.finding_id.padEnd(8)} ${tasks} ${method} ${conf}`);
  }
}

if (mapping.unmapped_findings && mapping.unmapped_findings.length > 0) {
  Output(`\nUnmapped findings (${mapping.unmapped_findings.length}):`);
  for (const u of mapping.unmapped_findings) {
    Output(`  ${u.finding_id}: ${u.reason} → ${u.recommended_action}`);
  }
}

// Implicit dependencies
const implicitDeps = [];
for (const task of (harmonized.enriched_plan?.tasks || [])) {
  if (task.implicit_dependencies && task.implicit_dependencies.length > 0) {
    for (const dep of task.implicit_dependencies) {
      implicitDeps.push({ task: task.task_id, ...dep });
    }
  }
}
if (implicitDeps.length > 0) {
  Output("\nIMPLICIT DEPENDENCIES DETECTED (require review)");
  Output("-".repeat(40));
  Output("");
  Output("  Task       Depends On   Reason                                                         Confidence");
  Output("  " + "-".repeat(96));
  for (const dep of implicitDeps) {
    const conf = dep.confidence != null ? dep.confidence.toFixed(2) : "n/a";
    const task = (dep.task || "").padEnd(10);
    const dependsOn = (dep.depends_on || "").padEnd(12);
    const reason = (dep.reason || "").substring(0, 60).padEnd(60);
    Output(`  ${task} ${dependsOn} ${reason} ${conf}`);
  }
  const promotable = implicitDeps.filter(d => d.confidence >= 0.8).length;
  Output("");
  Output(`  ${implicitDeps.length} implicit dependencies found (${promotable} with confidence >= 0.80)`);
  Output("");
  Output("  To promote all high-confidence deps: /refine-plan --promote-deps");
  Output("  To promote a single dep:             /refine-plan --promote-dep <task-id>:<depends-on-id>");
  Output("  Or manually add to blocked_by in the plan JSON.");
}

// Warnings
if (warnings.length > 0) {
  Output("\nWARNINGS");
  Output("-".repeat(40));
  for (const w of warnings) {
    Output(`  ! ${w}`);
  }
}

// Files
Output("\n" + "-".repeat(40));
Output("FILES SAVED");
Output("-".repeat(40));
Output(`Enriched plan: ${enrichedPlanPath}`);
Output(`Original plan: ${planPath} (unchanged)`);

// Metadata
if (meta.codebase_calls_used != null) {
  Output(`Codebase calls used: ${meta.codebase_calls_used}/15`);
}

// Next steps
Output("\n" + "=".repeat(80));
Output("NEXT STEPS");
Output("=".repeat(80));

if (score === null) {
  Output(`WARNING: Readiness score could not be computed (truncated pass).`);
  Output(`See warnings above. Fix inputs and re-run: /refine-plan ${planPath} ${reviewPath}`);
} else if (score.total >= 70) {
  Output(`Plan is READY for team-run automation.`);
  Output(`\nNext: /implement (uses enriched plan) or goyoke-plan-impl directly`);
  if (implicitDeps.length > 0) {
    Output(`\nNote: Review ${implicitDeps.length} implicit dependencies before automation.`);
  }
} else if (score.total >= 50) {
  Output(`Plan is READY WITH CAVEATS (score: ${score.total}/100).`);
  Output(`\nOptions:`);
  Output(`1. Review warnings and implicit dependencies, then proceed`);
  Output(`2. Revise plan to address unmapped findings, then re-run /refine-plan`);
} else {
  Output(`Plan is NOT READY (score: ${score.total}/100).`);
  Output(`\nRecommended actions:`);
  if (score.dimensions.fix_coverage < 2) {
    Output(`- Fix coverage too low (${score.dimensions.fix_coverage}/5): tasks don't address most review findings`);
    Output(`  → Revise plan to add tasks for unmapped findings, then re-run`);
  }
  if (score.dimensions.dep_validity < 3) {
    Output(`- Dependency validity low (${score.dimensions.dep_validity}/5): referenced files/types not found`);
    Output(`  → Verify file paths in plan tasks are correct`);
  }
  if (score.dimensions.schema_completeness < 3) {
    Output(`- Schema completeness low (${score.dimensions.schema_completeness}/5): tasks missing required fields`);
    Output(`  → Ensure all tasks have: task_id, subject, description, agent, target_packages, acceptance_criteria`);
  }
}

Output("\n");
```

### Phase 4: Dependency Promotion (Optional)

When invoked with `--promote-deps` or `--promote-dep`, the skill promotes implicit dependencies to `blocked_by` in the enriched plan.

**Step 10:** Parse promotion mode from args

```javascript
const promoteAll = args.includes("--promote-deps");
const promoteSingle = args.find(a => a.startsWith("--promote-dep"));

if (!promoteAll && !promoteSingle) {
  // No promotion requested — skip Phase 4
  return;
}

// Load the enriched plan (must exist from a prior /refine-plan run)
const enrichedPlan = JSON.parse(Read({ file_path: enrichedPlanPath }));
```

**Step 11:** Collect promotable dependencies

```javascript
let toPromote = [];

if (promoteAll) {
  // Promote all implicit deps with confidence >= 0.8
  for (const task of enrichedPlan.tasks) {
    for (const dep of (task.implicit_dependencies || [])) {
      if (dep.confidence >= 0.8 && !dep.promoted) {
        toPromote.push({ task_id: task.task_id, depends_on: dep.depends_on, dep });
      }
    }
  }
  Output(`[Promote] Found ${toPromote.length} implicit deps with confidence >= 0.80`);

} else if (promoteSingle) {
  // Parse --promote-dep task-006:task-004
  const spec = promoteSingle.split("=").pop() || args[args.indexOf(promoteSingle) + 1];
  const [taskId, dependsOn] = spec.split(":");

  const task = enrichedPlan.tasks.find(t => t.task_id === taskId);
  if (!task) { Output(`[Error] Task not found: ${taskId}`); return; }

  const dep = (task.implicit_dependencies || []).find(d => d.depends_on === dependsOn);
  if (!dep) { Output(`[Error] No implicit dependency ${taskId} → ${dependsOn}`); return; }
  if (dep.promoted) { Output(`[Info] Already promoted: ${taskId} → ${dependsOn}`); return; }

  toPromote.push({ task_id: taskId, depends_on: dependsOn, dep });
  Output(`[Promote] Promoting: ${taskId} → ${dependsOn} (confidence: ${dep.confidence})`);
}

if (toPromote.length === 0) {
  Output("[Promote] No dependencies to promote.");
  return;
}
```

**Step 12:** Cycle detection via Kahn's algorithm

Before committing any promotions, validate that adding them doesn't create cycles:

```javascript
// Build adjacency graph: existing blocked_by + proposed promotions
const graph = new Map();
for (const task of enrichedPlan.tasks) {
  graph.set(task.task_id, new Set(task.blocked_by || []));
}

// Add proposed promotions to graph
for (const p of toPromote) {
  graph.get(p.task_id)?.add(p.depends_on);
}

// Kahn's algorithm — detect cycles
const inDegree = new Map();
for (const [node, deps] of graph) {
  if (!inDegree.has(node)) inDegree.set(node, 0);
  for (const dep of deps) {
    inDegree.set(dep, (inDegree.get(dep) || 0) + 1);
  }
}

const queue = [];
for (const [node, deg] of inDegree) {
  if (deg === 0) queue.push(node);
}

let processed = 0;
while (queue.length > 0) {
  const node = queue.shift();
  processed++;
  for (const dep of (graph.get(node) || [])) {
    const newDeg = inDegree.get(dep) - 1;
    inDegree.set(dep, newDeg);
    if (newDeg === 0) queue.push(dep);
  }
}

if (processed < graph.size) {
  Output("[Error] Cycle detected! Promotion rejected.");
  Output("[Error] The following promotions would create a circular dependency:");
  for (const p of toPromote) {
    Output(`  ${p.task_id} → ${p.depends_on}`);
  }
  Output("\nResolve the cycle manually or skip these promotions.");
  return;
}
```

**Step 13:** Apply promotions and write updated plan

```javascript
for (const p of toPromote) {
  const task = enrichedPlan.tasks.find(t => t.task_id === p.task_id);

  // Add to blocked_by
  if (!task.blocked_by) task.blocked_by = [];
  if (!task.blocked_by.includes(p.depends_on)) {
    task.blocked_by.push(p.depends_on);
  }

  // Mark as promoted in implicit_dependencies
  const dep = task.implicit_dependencies.find(d => d.depends_on === p.depends_on);
  dep.promoted = true;

  // Log to harmonization_log
  if (!enrichedPlan.harmonization_log) enrichedPlan.harmonization_log = [];
  enrichedPlan.harmonization_log.push({
    source: "implicit_dep_promotion",
    task_id: p.task_id,
    depends_on: p.depends_on,
    confidence: dep.confidence,
    timestamp: new Date().toISOString(),
  });
}

// Write updated plan
Write({ file_path: enrichedPlanPath, content: JSON.stringify(enrichedPlan, null, 2) });

Output(`\n[Promote] ${toPromote.length} dependencies promoted to blocked_by.`);
Output(`[Promote] Enriched plan updated: ${enrichedPlanPath}`);
Output(`[Promote] Kahn's algorithm: no cycles detected.`);

for (const p of toPromote) {
  Output(`  ✓ ${p.task_id} now blocked_by ${p.depends_on}`);
}
```


### Only Plan Path Given

```javascript
// Single arg: no review
Output(`[Error] No review found. Run /review-plan first.`);
Output(`\nThen: /refine-plan ${args[0]} .claude/tmp/review-metadata.json`);
```

### Harmonizer Agent Failure

```javascript
try {
  const spawnResult = mcp__goyoke_interactive__spawn_agent({...});
  const result = mcp__goyoke_interactive__get_agent_result({ agent_id: spawnResult.agent_id });
} catch (error) {
  Output(`[Error] Plan-harmonizer failed: ${error.message}`);
  Output(`\nOriginal plan unchanged: ${planPath}`);
  Output(`\nPossible causes:`);
  Output(`- Plan JSON is too large (>50 tasks)`);
  Output(`- Agent timeout (increase timeout or simplify plan)`);
  Output(`\nRetry: /refine-plan ${planPath} ${reviewPath}`);
}
```

### Truncated Pass (Null Readiness Score)

When the harmonizer returns `readiness_score: null`, a pass was truncated (ran out of codebase call budget or context). The enriched plan is still written with partial enrichment, but the readiness score cannot be used for automation gating.

```javascript
if (score === null) {
  Output(`WARNING: Readiness score is null — one or more passes were truncated.`);
  Output(`See harmonized.warnings for which pass and why.`);
  Output(`Enriched plan written but NOT suitable for automated gating.`);
}
```

## Output Files

| File | Description |
| ---- | ----------- |
| `{plan-basename}-enriched.json` | Enriched plan with annotations, harmonization log, readiness score |
| Original plan | **Unchanged** — harmonizer never modifies source |

**Enriched plan structure additions:**
- `enrichment_version`: e.g., `"1.0.0"` (increments on re-harmonization)
- `review_annotations[]`: one entry per finding (finding_id, severity, classification, mapped_tasks, recommendation)
- `harmonization_log[]`: verification findings from Pass 2 (file existence, type checks, conflicts)
- `prior_harmonization_logs[]`: preserved from prior runs
- `tasks[].implicit_dependencies[]`: advisory deps to review before promotion to `blocked_by`
- `readiness_score`: full score object (total, dimensions, formula, thresholds)

---

## Limitations

- **Requires prior review:** Will not auto-invoke `/review-plan`. Run it first.
- **Does not modify `blocked_by`:** Implicit dependencies go to `tasks[].implicit_dependencies` only; human review required before promoting to `blocked_by` (Kahn's algorithm integrity).
- **Does not fix the plan:** Harmonizer annotates and scores; it does not rewrite tasks. Revise the plan manually for corrections, then re-run `/refine-plan`.
- **Codebase cap:** Pass 2 is capped at 15 codebase tool calls. Large plans may get partial dependency validation.
- **Semantic mapping confidence:** Findings with < 0.5 semantic similarity to any task are left unmapped. Check `unmapped_findings` in output.
