# Performance Benchmark Tests

## Overview

This directory contains performance benchmark tests for the TypeScript TUI cutover validation. These tests measure key metrics required for production sign-off.

## Metrics Measured

| Metric | Target | Acceptable | Measurement Method |
|--------|--------|------------|-------------------|
| Cold start | <500ms | <800ms | Median of 5 process spawns |
| Memory idle | <80MB | <120MB | RSS after 2s startup (ps) |
| Memory active | <100MB | <150MB | RSS after simulated activity |
| Input latency | <32ms | <50ms | Event loop response time (median of 5) |

## Running the Tests

### Full Performance Suite

```bash
npm run test:performance
```

This runs all performance benchmarks and generates:
- Test pass/fail results
- JSON output at `.claude/tmp/ts-benchmark-vitest.json`

### Skip Performance Tests

Performance tests are expensive (spawn processes, take ~60s). To skip them:

```bash
SKIP_PERF=1 npm test
```

This skips all benchmarks in the suite while running other tests normally.

### Prerequisites

Before running performance tests:

```bash
npm run build  # Must have dist/index.js
```

## Test Structure

### `benchmarks.test.ts`

Main performance test file with the following test groups:

1. **Cold Start Time** - Process startup latency
2. **Memory Idle** - Baseline memory consumption
3. **Memory Active** - Memory under load
4. **Input Latency** - UI responsiveness
5. **Cutover Sign-Off Validation** - Combined pass/fail for production approval

## Deterministic Testing

Tests include variance checks (±10% tolerance) to ensure measurements are stable across runs. This prevents flaky test failures from environment noise.

## Troubleshooting

### Test Timeout

If tests timeout, increase Vitest timeout:

```typescript
beforeAll(async () => {
  // ... benchmarks
}, 180000); // Increase to 3 minutes
```

### Process Won't Start

Ensure `dist/index.js` exists and is executable:

```bash
npm run build
node dist/index.js --version  # Should not error
```

### Memory Measurement Fails

Requires `ps` command (standard on Linux/macOS). On Windows, use WSL or skip these tests.

### High Variance

If measurements vary >10%, check for:
- Background processes consuming resources
- CPU throttling (power saving mode)
- Virtualized environment overhead

## Comparison to `scripts/benchmark.ts`

| Feature | `scripts/benchmark.ts` | `tests/performance/benchmarks.test.ts` |
|---------|------------------------|----------------------------------------|
| Framework | Standalone TypeScript | Vitest test suite |
| Output | Markdown report + JSON | JSON + test pass/fail |
| CI Integration | Manual | Native Vitest integration |
| Validation | None | Assertions against targets |
| Usage | `npm run benchmark` | `npm run test:performance` |

Both tools measure the same metrics. Use:
- `benchmark.ts` for manual analysis and reports
- `benchmarks.test.ts` for automated CI/CD validation
