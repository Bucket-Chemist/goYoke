## Executive Summary

The TUI rendering corruption (banner duplication, text overwriting, resize instability) is caused by **missing alternate screen buffer mode** combined with **no screen clear on resize events**. Ink 5.x writes to the primary terminal buffer by default, causing each render frame to append rather than replace previous output. The fix requires two changes: (1) enter alternate screen buffer on startup, and (2) clear screen on resize events.

## Root Cause Analysis

### Primary Cause: Missing Alternate Screen Buffer

**Evidence:**
- `src/index.tsx:57` calls `render(<App />)` with NO options
- Ink 5.x defaults to primary buffer mode
- Banner duplicated 12x = each resize triggered a new render that appended below previous

**Mechanism (Frame Smearing):**
```
Time T0: Render Frame 0 (80x24)
         [Banner at row 1]

Time T1: Terminal resize to 120x30
         Previous frame scrolls up (not erased)

Time T2: Render Frame 1 (120x30)
         [Banner at row 1] <- NEW
         [Banner visible from scroll] <- OLD (still visible)
```

Without alternate buffer:
- Ink attempts differential ANSI updates
- Terminal preserves scroll history
- New renders append, don't replace
- Visual result: "smeared" duplicate content

### Secondary Cause: No Screen Clear on Resize

**Evidence:**
- `Layout.tsx:31-35` reads `stdout.rows` synchronously in render
- No debounce on resize events
- No explicit clear screen mechanism anywhere in codebase

**Mechanism:**
1. SIGWINCH fired on terminal resize
2. `useStdout()` returns new dimensions immediately
3. React re-renders with new heights
4. Old frame content remains visible (no clear)
5. New frame overlaps old frame

### Contributing Factor: Height Calculation Inconsistency

**Evidence:**
- `ClaudePanel.tsx:341`: `<Box height={maxHeight - 6}>`
- `ClaudePanel.tsx:345`: `<Viewport height={maxHeight - 8}>`
- Mismatch causes 2-row discrepancy

**Impact:** Minor contributor to layout confusion, not primary cause of corruption.

```typescript
process.stdout.write('\x1b[?1049h'); // Enter alternate buffer
process.stdout.write('\x1b[2J');     // Clear screen

    patchConsole: true,
  }

const cleanup = () => {
  process.stdout.write('\x1b[?25h');  // Show cursor
  process.stdout.write('\x1b[?1049l'); // Exit alternate buffer
};

// Ensure cleanup on all exit paths
process.on('exit', cleanup);
process.on('SIGINT', () => { cleanup(); process.exit(0); });
process.on('SIGTERM', () => { cleanup(); process.exit(0); });

await waitUntilExit();
```

**ANSI Escape Sequences Explained:**
| Sequence | Meaning |
|----------|---------|
| `\x1b[?1049h` | Enter alternate screen buffer (like vim, htop) |
| `\x1b[?1049l` | Exit alternate screen buffer (restore previous) |
| `\x1b[2J` | Clear entire screen |
| `\x1b[H` | Move cursor to home position (0,0) |
| `\x1b[?25l` | Hide cursor |
| `\x1b[?25h` | Show cursor |

### Fix 2: Clear Screen and Debounce on Resize (HIGH)

**Confidence:** HIGH (both analysts agree)

**New File:** `src/hooks/useTerminalDimensions.ts`

```typescript
import { useStdout } from "ink";
import { useState, useEffect, useRef } from "react";

interface Dimensions {
  rows: number;
  columns: number;
}

/**
 * Custom hook for terminal dimensions with:
 * - Immediate screen clear on resize (prevents smearing)
 * - Debounced dimension updates (prevents render storms)
 */
export function useTerminalDimensions(debounceMs = 100): Dimensions {
  const { stdout } = useStdout();
  const [dimensions, setDimensions] = useState<Dimensions>({
    rows: stdout?.rows ?? 24,
    columns: stdout?.columns ?? 80,
  });
  const timeoutRef = useRef<NodeJS.Timeout | null>(null);

  useEffect(() => {
    const handleResize = () => {
      // Cancel pending update
      if (timeoutRef.current) {
        clearTimeout(timeoutRef.current);
      }

      // IMMEDIATE: Clear screen to prevent frame smearing
      process.stdout.write('\x1b[2J\x1b[H');

      // DEBOUNCED: Update dimensions state
      timeoutRef.current = setTimeout(() => {
        if (stdout) {
          setDimensions({
            rows: stdout.rows ?? 24,
            columns: stdout.columns ?? 80,
          });
        }
      }, debounceMs);
    };

    // Listen for resize events
    process.stdout.on('resize', handleResize);

    return () => {
      process.stdout.off('resize', handleResize);
      if (timeoutRef.current) {
        clearTimeout(timeoutRef.current);
      }
    };
  }, [stdout, debounceMs]);

  return dimensions;
}
```

## Appendix: Ink 5.x vs 4.x Differences

Ink 5.x changed default behaviors:
- No built-in `fullscreen` option (must use ANSI manually)
- `useStdout()` more reactive to dimension changes
- Yoga layout engine updated

The old Ink 4.x pattern `render(app, { fullscreen: true })` doesn't work in 5.x.
Manual ANSI escape sequences are the recommended approach.
