## 1. Problem Statement

### Original Input
> My TUI needs a massive facelift. It is super basic and...quite shit. There's no QoL things, I can't scroll the conversation window, the agents subpanel doesn't actually display any of the agents, and none of the agent details and what they are doing are in agent detail (ideally I'd like to select an agent in the agents top panel and have its actions display in the detail panel beneath, rather than pollute the main window).

### Clarified Statement
The GOfortress TUI (Ink/React-based) has fundamental UX deficiencies:

1. **Broken Agent Display**: The agents panel shows "No agents yet" because nothing populates the agents store - the `useClaudeQuery` hook processes SDK events but never calls `addAgent()`. This is an **architectural gap**, not a rendering bug.

2. **No Conversation Scrolling**: Users cannot scroll through conversation history. A Viewport component exists but may not be properly integrated or have UX issues.

3. **Incomplete Master-Detail Pattern**: Agent selection exists but:
   - No agents to select (see #1)
   - Detail panel only shows static metadata, not live activity
   - Agent tool calls/streaming output not captured or displayed

4. **General QoL**: The TUI lacks polish expected in a production tool.

### Success Criteria
A successful analysis will provide:
- Clear architectural recommendations for agent data flow
- Concrete implementation approach for live agent activity display
- Viewport/scrolling fixes
- Prioritized improvement roadmap
