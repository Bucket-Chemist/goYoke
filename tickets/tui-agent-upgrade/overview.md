## Strategic Approach

**4 phases, dependency-ordered:**

1. **Data Model Extension** — Parse health fields from config.json that gogent-team-run already writes
2. **Full-Width Drawer Promotion** — Extract drawers from right panel to global layout level (highest risk)
3. **Right Panel 50/50 Split** — Side-by-side agents + health at Wide/Ultra, tab-toggle at Standard
4. **Teams Health Dashboard** — New display-only component with budget bars, wave grouping, health indicators

Phase ordering is critical: data model first (unblocks everything), layout refactor second (behavior-preserving, establishes stable skeleton), split third (builds on refactored layout), component last (additive, no existing code modified).
