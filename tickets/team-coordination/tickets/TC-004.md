# TC-004: Implement Full Daemon Pattern in Go Binary

**Phase**: Phase 2: Go Binary Implementation
**Priority**: HIGH
**Blocked By**: none
**Effort**: 1 day

## Description

Implement proper Unix daemon pattern for `gogent-team-run` to enable true background execution independent of the TUI terminal session. This supports the user's plan to run concurrent teams (braintrust + review simultaneously) with proper isolation and double-start prevention.

The daemon pattern includes:
1. **Process group isolation** via `setsid()` syscall
2. **File descriptor management** (close stdin, redirect stdout/stderr)
3. **PID file lifecycle** (write on start, remove on exit, double-start prevention)
4. **Signal handling** for graceful shutdown

### Step-by-Step Implementation

#### 1. Process Group Isolation

**In `main()`** (before any other logic):
```
func main() {
    // Become session leader if not already (enables Ctrl+C immunity)
    if err := syscall.Setsid(); err != nil {
        // Already a session leader - not an error for re-exec scenarios
        // or when launched by systemd
    }

    // ... rest of main()
}
```

**When spawning child `claude` processes**:
```
cmd := exec.CommandContext(ctx, "claude", args...)
cmd.Dir = teamConfig.ProjectRoot
cmd.SysProcAttr = &syscall.SysProcAttr{
    Setsid: true,  // Each child gets own process group
}
```

This ensures:
- SIGINT (Ctrl+C) in TUI terminal does NOT propagate to gogent-team-run or children
- SIGHUP (terminal close) does NOT kill the binary
- Process group tree: TUI (pgid=1234) | gogent-team-run (pgid=5678) | claude-1 (pgid=5679), claude-2 (pgid=5680)

#### 2. File Descriptor Management

**Stdin**: Close after reading config.json (no further input needed)
```
func daemonizeStdin() error {
    // We already read config from disk, no stdin needed
    if err := os.Stdin.Close(); err != nil {
        return fmt.Errorf("close stdin: %w", err)
    }
    // Reopen stdin to /dev/null to avoid "bad file descriptor" errors
    devNull, err := os.Open("/dev/null")
    if err != nil {
        return fmt.Errorf("open /dev/null: %w", err)
    }
    if err := syscall.Dup2(int(devNull.Fd()), int(os.Stdin.Fd())); err != nil {
        return fmt.Errorf("dup2 /dev/null to stdin: %w", err)
    }
    return nil
}
```

**Stdout/Stderr**: Redirect to team_dir/runner.log
```
func redirectOutput(teamDir string) (*os.File, error) {
    logPath := filepath.Join(teamDir, "runner.log")
    logFile, err := os.OpenFile(logPath, os.O_CREATE|os.O_WRONLY|os.O_APPEND, 0644)
    if err != nil {
        return nil, fmt.Errorf("open runner.log: %w", err)
    }

    // Redirect both stdout and stderr to log file
    if err := syscall.Dup2(int(logFile.Fd()), int(os.Stdout.Fd())); err != nil {
        return nil, fmt.Errorf("dup2 stdout: %w", err)
    }
    if err := syscall.Dup2(int(logFile.Fd()), int(os.Stderr.Fd())); err != nil {
        return nil, fmt.Errorf("dup2 stderr: %w", err)
    }

    return logFile, nil  // Caller must close on exit
}
```

#### 3. PID File Lifecycle

**Type definitions**:
```
const PIDFileName = "gogent-team-run.pid"

type PIDFile struct {
    path string
    pid  int
}
```

**Write on startup** (with double-start prevention):
```
func acquirePIDFile(teamDir string) (*PIDFile, error) {
    pidPath := filepath.Join(teamDir, PIDFileName)

    // Check for existing PID file
    if data, err := os.ReadFile(pidPath); err == nil {
        existingPID, _ := strconv.Atoi(strings.TrimSpace(string(data)))
        if existingPID > 0 && processExists(existingPID) {
            return nil, fmt.Errorf("team already running (PID %d)", existingPID)
        }
        // Stale PID file - clean up
        os.Remove(pidPath)
    }

    // Write current PID
    pid := os.Getpid()
    if err := os.WriteFile(pidPath, []byte(fmt.Sprintf("%d\n", pid)), 0644); err != nil {
        return nil, fmt.Errorf("write PID file: %w", err)
    }

    return &PIDFile{path: pidPath, pid: pid}, nil
}

// processExists checks if a process with given PID is running
func processExists(pid int) bool {
    process, err := os.FindProcess(pid)
    if err != nil {
        return false
    }
    // Send signal 0 to check existence without killing
    err = process.Signal(syscall.Signal(0))
    return err == nil
}
```

**Remove on exit** (via defer and signal handler):
```
func (pf *PIDFile) Release() error {
    if err := os.Remove(pf.path); err != nil && !os.IsNotExist(err) {
        return fmt.Errorf("remove PID file: %w", err)
    }
    return nil
}
```

#### 4. Signal Handling

**Setup signal handler in main()**:
```
func main() {
    // ... daemon setup ...

    pidFile, err := acquirePIDFile(teamDir)
    if err != nil {
        log.Fatalf("PID file error: %v", err)
    }
    defer pidFile.Release()  // Best-effort cleanup on normal exit

    // Signal handler for graceful shutdown
    sigCh := make(chan os.Signal, 1)
    signal.Notify(sigCh, syscall.SIGTERM, syscall.SIGINT)

    // Spawn wave execution in goroutine
    doneCh := make(chan error, 1)
    go func() {
        doneCh <- runWaves(ctx, teamRunner)
    }()

    // Wait for completion or signal
    select {
    case err := <-doneCh:
        // Waves completed normally
        if err != nil {
            log.Printf("Wave execution failed: %v", err)
        }
    case sig := <-sigCh:
        log.Printf("Received signal %s, shutting down", sig)
        cancel()  // Cancel context to stop waves
        // Wait up to 10s for graceful shutdown
        select {
        case <-doneCh:
            log.Printf("Graceful shutdown completed")
        case <-time.After(10 * time.Second):
            log.Printf("Shutdown timeout, forcing exit")
        }
    }

    // PID file cleanup via defer
}
```

**Signal cascade to children**:
```
func (tr *TeamRunner) killAllChildren() {
    tr.childrenMu.Lock()
    defer tr.childrenMu.Unlock()

    for pid := range tr.childPIDs {
        process, err := os.FindProcess(pid)
        if err != nil {
            continue
        }
        // SIGTERM first (graceful)
        process.Signal(syscall.SIGTERM)
    }

    // Grace period
    time.Sleep(5 * time.Second)

    // SIGKILL stragglers
    for pid := range tr.childPIDs {
        process, err := os.FindProcess(pid)
        if err != nil {
            continue
        }
        process.Signal(syscall.SIGKILL)
    }
}
```

### Key Type Definitions

```
type TeamRunner struct {
    config    *TeamConfig
    configMu  sync.Mutex

    childPIDs   map[int]struct{}  // Track spawned child PIDs
    childrenMu  sync.Mutex
}

func (tr *TeamRunner) registerChild(pid int) {
    tr.childrenMu.Lock()
    defer tr.childrenMu.Unlock()
    tr.childPIDs[pid] = struct{}{}
}

func (tr *TeamRunner) unregisterChild(pid int) {
    tr.childrenMu.Lock()
    defer tr.childrenMu.Unlock()
    delete(tr.childPIDs, pid)
}
```

### Execution Flow

```
main()
  ├─ syscall.Setsid()              // Become session leader
  ├─ acquirePIDFile(teamDir)       // Write PID, check double-start
  ├─ redirectOutput(teamDir)       // Redirect stdout/stderr to runner.log
  ├─ daemonizeStdin()              // Close stdin, reopen to /dev/null
  ├─ signal.Notify(SIGTERM, SIGINT)
  ├─ go runWaves(ctx, teamRunner)  // Execute waves in goroutine
  └─ select {
      case <-doneCh:                // Waves completed
      case <-sigCh:                 // Signal received
        ├─ cancel()                 // Cancel context
        ├─ killAllChildren()        // Cascade SIGTERM -> SIGKILL
        └─ pidFile.Release()        // Remove PID file
    }
```

## Code Guidelines

From go.md conventions:

1. **Error wrapping**: `fmt.Errorf("acquire PID file: %w", err)` for full context
2. **Atomic file operations**: `os.WriteFile()` for PID file (atomic on Linux)
3. **Defer cleanup**: `defer pidFile.Release()` ensures cleanup even on panic
4. **Signal 0 trick**: `process.Signal(syscall.Signal(0))` checks existence without killing
5. **Grace periods**: 5s for SIGTERM before SIGKILL (configurable via env var if needed)

- [ ] SIGINT (Ctrl+C) behaves identically to SIGTERM
