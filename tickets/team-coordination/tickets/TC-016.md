# TC-016: Duplicate Launch Prevention via PID File

**Phase**: Independent Tickets (Post-MVP)
**Priority**: LOW
**Blocked By**: TC-008 (Go binary implementation)
**Effort**: 0.5 days

## Description

Prevent two `gogent-team-run` instances from launching for the same team directory. If the launch command is accidentally run twice, the second instance would overwrite the first's config.json.

This ticket adds **stale PID file detection** on top of TC-004's daemon pattern. TC-004 handles PID file creation/cleanup on normal operations; TC-016 handles the edge case where a previous instance crashed (`kill -9`) and left a stale PID file.

## Implementation Instructions

### 1. PID File Location
- Path: `{team_dir}/gogent-team-run.pid`
- Format: Plain text file containing decimal PID (e.g., `"12345\n"`)

### 2. Startup Sequence (Before daemon setup)

In `cmd/gogent-team-run/main.go`, add this check **before** writing the PID file:

```go
// Check for existing PID file
pidPath := filepath.Join(teamDir, "gogent-team-run.pid")
if pidData, err := os.ReadFile(pidPath); err == nil {
    // PID file exists - parse it
    pidStr := strings.TrimSpace(string(pidData))
    pid, err := strconv.Atoi(pidStr)
    if err != nil {
        // Malformed PID file - log warning and replace
        log.Printf("Warning: Stale PID file with invalid content %q, removing", pidStr)
        os.Remove(pidPath)
    } else {
        // Valid PID - check if process is alive
        if isProcessAlive(pid) {
            fmt.Fprintf(os.Stderr, "Error: Team already running (PID %d)\n", pid)
            fmt.Fprintf(os.Stderr, "Use /team-cancel to stop it first, or check {team_dir}/runner.log\n")
            os.Exit(1)
        } else {
            // Process is dead - stale PID file
            log.Printf("Warning: Cleaning up stale PID file from dead process %d", pid)
            os.Remove(pidPath)
        }
    }
}

// Now write our PID (existing TC-004 logic)
if err := os.WriteFile(pidPath, []byte(fmt.Sprintf("%d\n", os.Getpid())), 0644); err != nil {
    return fmt.Errorf("failed to write PID file: %w", err)
}
```

### 3. Process Liveness Check

Add helper function (Linux-specific, safe cross-platform fallback):

```go
// isProcessAlive checks if a process with the given PID is running.
// Uses syscall.Kill(pid, 0) which returns nil if the process exists.
func isProcessAlive(pid int) bool {
    process, err := os.FindProcess(pid)
    if err != nil {
        return false // Process doesn't exist
    }

    // Send signal 0 (no-op signal) to test liveness
    err = process.Signal(syscall.Signal(0))
    return err == nil // nil = process exists and is alive
}
```

### 4. Error Messages

- **Alive process detected**: `"Error: Team already running (PID NNNNN)\nUse /team-cancel to stop it first, or check {team_dir}/runner.log\n"`
- **Stale PID cleaned**: Log at `INFO` level: `"Warning: Cleaning up stale PID file from dead process NNNNN"`
- **Malformed PID**: Log at `WARN` level: `"Warning: Stale PID file with invalid content \"abc\", removing"`

### 5. Integration with TC-004

TC-004 already implements:
- PID file write on startup
- PID file removal on clean exit (signal handler)

TC-016 adds:
- Pre-startup check for existing PID file
- Stale PID detection via `syscall.Kill(pid, 0)`

- **File I/O**: `os.ReadFile`, `os.WriteFile`, `os.Remove` (standard library)

- **Process check**: `os.FindProcess` + `process.Signal(syscall.Signal(0))`

## Success Criteria

1. **Second launch to same team dir fails with clear error**:
   - Start team A in directory `/tmp/team1`
   - Attempt second launch to `/tmp/team1` → exits with `"Team already running (PID NNNNN)"`

2. **Stale PID file from crashed process is cleaned up**:
   - Create fake PID file: `echo "99999" > /tmp/team1/gogent-team-run.pid`
   - Launch team → logs `"Cleaning up stale PID file"` and proceeds normally

3. **Clean exit removes PID file** (TC-004 coverage):
   - Launch team, send SIGTERM → PID file removed

4. **Malformed PID file handled gracefully**:
   - Create malformed file: `echo "not-a-number" > /tmp/team1/gogent-team-run.pid`
   - Launch team → logs warning, replaces file, proceeds

### Known Limitations (Review Item 31)

1. **Same-machine only**: PID liveness check (`kill -0 $PID`) only works on the same machine. If the PID file is on a shared filesystem (NFS), a different machine's PID may collide. This is acceptable for GOgent-Fortress which is single-machine.

2. **PID reuse**: On long-running systems, the OS may reuse a PID. Mitigate by adding a timestamp to the PID file.

### PID File Format Change

Old format (PID only):
```
12345
```

New format (PID + timestamp):
```
12345
1738856422
```

Line 1: PID. Line 2: Unix timestamp of write.

**Liveness check with timestamp:**
```go
func checkPIDFile(pidPath string) (alive bool, stale bool, err error) {
    data, err := os.ReadFile(pidPath)
    if err != nil { return false, false, err }

    lines := strings.SplitN(strings.TrimSpace(string(data)), "\n", 2)
    pid, _ := strconv.Atoi(lines[0])

    var timestamp int64
    if len(lines) > 1 {
        timestamp, _ = strconv.ParseInt(lines[1], 10, 64)
    }

    if pid <= 0 {
        return false, true, nil // Invalid PID
    }

    processAlive := processExists(pid)

    // If timestamp > 24h old AND process exists, likely PID reuse
    if timestamp > 0 && time.Since(time.Unix(timestamp, 0)) > 24*time.Hour && processAlive {
        return false, true, nil // Treat as stale (PID reuse)
    }

    return processAlive, !processAlive, nil
}
```
