# TC-017: gogent-validate Level 2 Enforcement

**Phase**: Phase 1: Schema Design (Critical Blocker)
**Priority**: CRITICAL
**Blocked By**: None
**Blocks**: TC-008 (testing requires Level 2 agents to delegate to haiku scouts)
**Effort**: 1 day
**Status**: Open

---

## Problem Statement

TC-007 documents that team-spawned agents (Level 2) should be allowed to use `Task(haiku)` and `Task(sonnet)` for delegation, but not `Task(opus)`. However, the current `gogent-validate` implementation blocks ALL `Task()` calls when `nestingLevel > 0`, preventing legitimate delegation patterns.

**Impact**: Team-spawned Einstein/Staff-Architect agents cannot delegate to haiku scouts for mechanical work (file discovery, pattern extraction), forcing expensive Opus-level work for simple operations.

---

## Current Behavior

File: `cmd/gogent-validate/main.go` (lines 17-19, 97-109)

```go
const (
    MAX_TASK_NESTING_LEVEL = 0 // Strict: Only Router (Level 0) can use Task()
)

// Line 101:
if nestingLevel > MAX_TASK_NESTING_LEVEL {
    logNestingBlock(event, nestingLevel, isExplicit)
    response := routing.BlockResponseForNesting(nestingLevel)
    outputJSON(response)
    return
}
```

**Result**: Any Level 1+ agent attempting `Task(haiku)` or `Task(sonnet)` is blocked, even though this is a valid pattern.

## Required Changes

### 1. Update `cmd/gogent-validate/main.go`

Replace the block at lines 97-109 with model-aware enforcement:

```go
if nestingLevel > MAX_TASK_NESTING_LEVEL {
    // Level 1+: Allow haiku/sonnet, block opus
    if taskInput, err := routing.ParseTaskInput(event.ToolInput); err == nil {
        if taskInput.Model == "opus" {
            logNestingBlock(event, nestingLevel, isExplicit)
            response := routing.BlockResponse(
                fmt.Sprintf(
                    "Task(opus) blocked at nesting level %d. Use Task(haiku) or Task(sonnet) instead.",
                    nestingLevel,
                ),
            )
            outputJSON(response)
            return
        }
        // Task(haiku) and Task(sonnet) allowed at Level 1+
    } else {
        // Cannot parse model — block defensively
        logNestingBlock(event, nestingLevel, isExplicit)
        response := routing.BlockResponseForNesting(nestingLevel)
        outputJSON(response)
        return
    }
}
```

**Rationale**:
- Level 1+ agents can spawn cheaper models for delegation
- Blocks expensive opus spawning from sub-agents
- Defensive blocking if model cannot be parsed

```go
                require.Contains(t, response.Message, "Task(opus) blocked", "Should mention opus specifically")

                require.Equal(t, routing.Allowed, response.Status, tc.description)
            }
        })
    }
}
```

## Implementation Notes

### Parsing Task Model

The `routing.ParseTaskInput` function (or equivalent) must extract the `model` field from the tool input JSON. Example structure:

```go
type TaskInput struct {
    Model       string `json:"model"`
    Description string `json:"description"`
    Prompt      string `json:"prompt"`
}

func ParseTaskInput(input string) (*TaskInput, error) {
    var task TaskInput
    if err := json.Unmarshal([]byte(input), &task); err != nil {
        return nil, fmt.Errorf("parse task input: %w", err)
    }
    return &task, nil
}
```

### Defensive Blocking Rationale

When model cannot be parsed (e.g., malformed JSON, missing field), we block defensively because:
1. We cannot verify the spawn is safe (not opus)
2. Parsing failure indicates either tooling bug or adversarial input
3. False negative (blocking valid spawn) is safer than false positive (allowing expensive opus spawn)

### Logging Enhancement

Update `logNestingBlock` to include model information when available:

```go
func logNestingBlock(event *routing.Event, nestingLevel int, model string, isExplicit bool) {
    log.Printf(
        "[NESTING-BLOCK] Level %d attempted Task(%s) - Explicit=%v",
        nestingLevel,
        model,
        isExplicit,
    )
}
```
