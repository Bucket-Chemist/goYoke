# TC-005: Verify and Document CLI JSON Output Format

**Phase**: Phase 2: Go Binary Implementation
**Priority**: HIGH
**Blocked By**: none (but MUST be completed before TC-008 cost extraction implementation)
**Effort**: 0.5 days

## Description

The `extractCostFromCLIOutput()` function in the implementation plan assumes a `cost_usd` field exists in the Claude CLI's JSON output. The existing `spawnAgent.ts:parseCliOutput()` code tries `cost_usd || total_cost_usd`, suggesting field name uncertainty. No official documentation exists for the `claude -p --output-format json` output format.

This is a **HIGH-risk assumption** flagged by both Einstein and Staff-Architect. The implementation plan depends on parsing this undocumented format to track costs and enforce budget ceilings.

### Go Implementation Pattern

Based on verified format, implement defensive parser:

```
type CLIOutput struct {
    // Document actual fields here after verification
    Content     string   `json:"content,omitempty"`
    CostUSD     *float64 `json:"cost_usd,omitempty"`
    TotalCost   *float64 `json:"total_cost_usd,omitempty"`
    Error       string   `json:"error,omitempty"`
    // Add other fields discovered during investigation
}

func extractCostFromCLIOutput(rawOutput []byte) (float64, error) {
    var output CLIOutput
    if err := json.Unmarshal(rawOutput, &output); err != nil {
        // Log raw output for debugging
        log.Printf("[WARN] Failed to parse CLI output: %v\nRaw output:\n%s", err, string(rawOutput))
        return 0, fmt.Errorf("parse CLI JSON: %w", err)
    }

    // Try primary field (update based on verification)
    if output.CostUSD != nil {
        return *output.CostUSD, nil
    }

    // Fallback to alternate field name
    if output.TotalCost != nil {
        return *output.TotalCost, nil
    }

    // No cost field found - log raw for debugging
    log.Printf("[WARN] No cost field in CLI output. Available fields: %+v", output)
    return 0, fmt.Errorf("no cost field in CLI output")
}
```

### Fallback Strategy

If cost extraction fails, the team runner should:
1. **Log raw output** to `runner.log` with `[COST-PARSE-ERROR]` prefix
2. **Continue execution** (don't block team on parse failure)
3. **Estimate cost** using model pricing + token count (if available in output)
4. **Mark cost as estimated** in config.json: `cost_usd_estimated: true`

### Budget Enforcement Layers

**Primary**: Cost extraction from CLI JSON
**Secondary**: `--max-budget-usd` flag passed to CLI (user decision)
**Tertiary**: Team-level budget gate (check `budget_remaining_usd` before each spawn)

## Code Guidelines

From go.md conventions:

1. **Defensive parsing**: Use pointer fields (`*float64`) to distinguish missing vs zero
2. **Error logging**: Log raw output on parse failure with `[WARN]` prefix
3. **Struct tags**: Use `omitempty` to handle optional fields
4. **Multiple fallbacks**: Try primary field → fallback field → estimation
5. **Non-blocking errors**: Cost parse failure logs warning but doesn't halt execution
