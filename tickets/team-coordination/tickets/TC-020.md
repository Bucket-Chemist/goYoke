## Description

TC-013 covers 3 major orchestrator rewrites (Mozart, ReviewOrchestrator, ImplManager) but only shows skeleton/pseudocode. Architecture review (Finding 12) flagged that actual prompt templates should be in separate design documents for each rewrite.

These design documents will serve as the detailed specifications for implementing the orchestrator rewrites. Each document must contain complete prompt templates, config generation logic, stdin file specifications, launch commands, and test cases.

---

## Context

The current orchestrators (Mozart, Review-Orchestrator, Implementation-Manager) use foreground Task() invocations with agent-by-agent sequencing. The rewrite transitions them to background `gogent-team-run` execution with parallel agent coordination via the Go binary.

**Key architectural shift:**
- **Before**: Orchestrator LLM calls Task() for each agent sequentially
- **After**: Orchestrator LLM generates config.json + stdin files, spawns `gogent-team-run` in background, exits

This shift requires detailed prompt engineering to transform interview-driven orchestrators into config-generation orchestrators.

## Implementation Notes

### Design First, Code Second

These design docs are PREREQUISITES for TC-013 implementation. Do not start rewriting orchestrators until these documents are reviewed and approved.

### Prompt Engineering Focus

The core challenge is transforming interview-driven orchestration into config-driven orchestration. The prompts must:
1. Gather sufficient context (interview or input parsing)
2. Generate valid config.json + stdin files
3. Launch gogent-team-run correctly
4. Exit cleanly (no blocking on results)

### Absolute Path Resolution

All file paths in stdin files MUST be absolute paths. Use `filepath.Abs()` in Go or equivalent in prompt instructions. Relative paths will break when agents run in different working directories.

### Error Handling

Design prompts to detect errors BEFORE launching gogent-team-run:
- Budget too low → abort, suggest alternatives
- Empty input → abort, explain what's needed
- Circular dependencies → abort, show cycle

Failing early is better than spawning a team that will fail.
