# TC-014: Add `cli_flags` to agents-index.json

**Phase**: Phase 1: Schema Design

**Priority**: MEDIUM
**Blocked By**: none
**Effort**: 1 day

## Description

Three spawn paths (`Task()`, `spawnAgent.ts`, `gogent-team-run`) each construct CLI args independently. Adding a `cli_flags` field (or `allowed_tools` list) to agents-index.json lets all three paths read from the same source for per-agent tool permissions and other flags.

This mitigates the "three spawn paths drift" risk identified by both analysts.

---

## Implementation Instructions

### Current Problem

Currently, tool permissions and CLI flags for agents are defined in three separate locations:

1. **`Task()` calls in Router** - Hardcoded or inferred from agent YAML files
2. **`spawnAgent.ts` (TUI MCP tool)** - Hardcoded in `buildCliArgs()` function (~line 150)
3. **`gogent-team-run` Go binary** (planned) - Will need to construct CLI args for each agent

When a new tool is added to an agent or a CLI flag changes, all three locations must be updated manually. This creates:
- **Drift risk**: Forgetting to update one path leads to inconsistent behavior
- **Maintenance burden**: Changes require coordinating across TypeScript, Go, and agent YAML files
- **Testing complexity**: Must verify behavior across all three spawn mechanisms

### Solution: Centralized CLI Configuration

Add a `cli_flags` field to each agent in `agents-index.json` that specifies:
1. **Allowed tools** for this agent (Read, Write, Glob, Grep, Bash, Edit, etc.)
2. **Other CLI flags** (e.g., `--permission-mode delegate`, `--max-budget-usd`, etc.)

All three spawn paths read from this single source of truth.

---

### Field Structure

Add the following fields to each agent in `agents-index.json`:

```json
{
  "id": "einstein",
  "name": "Einstein",
  "model": "opus",

  "cli_flags": {
    "allowed_tools": ["Read", "Write", "Glob", "Grep", "TaskGet"],
    "additional_flags": [
      "--permission-mode delegate"
    ]
  }
}
```

**Field-by-field specification:**

| Field | Type | Required | Description | Example |
|-------|------|----------|-------------|---------|
| `cli_flags` | object | No (defaults to empty) | Container for all CLI-related configuration | See above |
| `cli_flags.allowed_tools` | array of strings | Yes | Tools this agent is permitted to use. Translates to `--allowedTools "Read,Write,..."` CLI argument | `["Read", "Write", "Glob"]` |
| `cli_flags.additional_flags` | array of strings | No | Other CLI flags to pass to `claude` command. Each array element is a separate flag. | `["--permission-mode delegate", "--max-budget-usd 5.0"]` |

**Current implementation** (approximate, around line 150):

```typescript
function buildCliArgs(agent: AgentConfig, model: string): string[] {
  const args = ['-p'];

  // Hardcoded tool list
  args.push('--allowedTools', 'Read,Write,Glob,Grep,Bash,Edit');

  // Model override

    args.push('--model', model);
  }

  return args;

```

**New implementation** (reads from agents-index.json):

```typescript
function buildCliArgs(agent: AgentConfig, model: string): string[] {
  const args = ['-p'];

  // Read from agent config (cli_flags field from agents-index.json)
  if (agent.cli_flags?.allowed_tools) {
    args.push('--allowedTools', agent.cli_flags.allowed_tools.join(','));
  } else {
    // Fallback to default if not specified
    args.push('--allowedTools', 'Read,Write,Glob,Grep,Bash,Edit');
  }

  // Additional flags from agent config
  if (agent.cli_flags?.additional_flags) {
    args.push(...agent.cli_flags.additional_flags);
  }

  // Model override
  if (model) {
    args.push('--model', model);
  }

  return args;
}
```

### Integration Point 2: Go Binary (gogent-team-run)

**File**: `cmd/gogent-team-run/main.go` (or wherever agent spawning happens)

**Design pattern**:

```go
package main

import (
	"encoding/json"
	"os"
	"os/exec"
	"strings"
)

// AgentCliFlags matches the agents-index.json schema
type AgentCliFlags struct {
	AllowedTools    []string `json:"allowed_tools"`
	AdditionalFlags []string `json:"additional_flags,omitempty"`
}

// AgentConfig is a subset of the full agent definition
type AgentConfig struct {
	ID        string         `json:"id"`
	Model     string         `json:"model"`
	CliFlags  *AgentCliFlags `json:"cli_flags,omitempty"`
}

// LoadAgentsIndex reads agents-index.json
func LoadAgentsIndex(path string) (map[string]AgentConfig, error) {
	data, err := os.ReadFile(path)
	if err != nil {
		return nil, err
	}

	var index struct {
		Agents []AgentConfig `json:"agents"`
	}

	if err := json.Unmarshal(data, &index); err != nil {
		return nil, err
	}

	// Convert to map for fast lookup
	agents := make(map[string]AgentConfig)
	for _, agent := range index.Agents {
		agents[agent.ID] = agent
	}

	return agents, nil
}

// BuildCliArgs constructs CLI args for spawning an agent
func BuildCliArgs(agentID string, agentsIndex map[string]AgentConfig, modelOverride string) []string {
	agent, exists := agentsIndex[agentID]
	if !exists {
		// Fallback to safe defaults
		return []string{"-p", "--allowedTools", "Read,Glob,Grep"}
	}

	args := []string{"-p"}

	// Allowed tools
	if agent.CliFlags != nil && len(agent.CliFlags.AllowedTools) > 0 {
		args = append(args, "--allowedTools", strings.Join(agent.CliFlags.AllowedTools, ","))
	} else {
		// Default fallback
		args = append(args, "--allowedTools", "Read,Glob,Grep")
	}

	// Additional flags
	if agent.CliFlags != nil && len(agent.CliFlags.AdditionalFlags) > 0 {
		args = append(args, agent.CliFlags.AdditionalFlags...)
	}

	// Model override (from team config)
	model := modelOverride
	if model == "" {
		model = agent.Model
	}
	args = append(args, "--model", model)

	return args
}

// Example usage in spawnAndWait
func spawnAndWait(teamDir string, member *Member, agentsIndex map[string]AgentConfig) error {
	// Build CLI args from agents-index.json
	args := BuildCliArgs(member.AgentID, agentsIndex, member.Model)

	// Build prompt envelope
	prompt, err := buildPromptEnvelope(teamDir, member)
	if err != nil {
		return err
	}

	// Spawn process
	cmd := exec.Command("claude", args...)
	cmd.Stdin = strings.NewReader(prompt)
	cmd.Dir = projectRoot  // From config.json

	// ... rest of spawn logic
}
```

## Code Guidelines

### Naming Conventions

- **Field name**: `cli_flags` (snake_case to match JSON conventions)
- **Sub-fields**: `allowed_tools`, `additional_flags` (snake_case)
- **Tool names**: Match Claude Code tool names exactly: `Read`, `Write`, `Glob`, `Grep`, `Bash`, `Edit`, `TaskCreate`, `mcp__gofortress__spawn_agent`

### Required vs Optional Fields

- `cli_flags`: **Optional** (agents without this field fall back to safe defaults)
- `cli_flags.allowed_tools`: **Required if `cli_flags` is present**
- `cli_flags.additional_flags`: **Optional**

### Default Fallback Behavior

If `cli_flags` is missing or `allowed_tools` is empty:

| Spawn Path | Default Behavior |
|------------|------------------|
| **spawnAgent.ts** | Use `["Read", "Write", "Glob", "Grep", "Bash", "Edit"]` |
| **gogent-team-run** | Use `["Read", "Glob", "Grep"]` (safe read-only) |
| **Task()** | Agent reads own YAML file via Claude Code's internal mechanism |

**Rationale**: Different defaults reflect different security contexts. TUI (interactive) is more permissive than background Go binary (autonomous).

### Fallback Defaults Standardization (Review Item 29)

**Problem**: Missing `cli_flags` / `allowed_tools` in agents-index.json gets different fallback defaults depending on spawn path:
- TUI path (spawnAgent.ts): Full tools — `Read, Write, Glob, Grep, Bash, Edit`
- Go binary path (gogent-team-run): Read-only — `Read, Glob, Grep`

This inconsistency masks bugs: an agent works in TUI but fails in team execution (or vice versa).

**Resolution**: If `allowed_tools` is missing in agents-index.json, **fail with an explicit error** rather than applying silent defaults:

```go
agentConfig, err := loadAgentConfig(member.Agent)
if err != nil {
    return fmt.Errorf("agent %q not found in agents-index.json", member.Agent)
}
if len(agentConfig.AllowedTools) == 0 {
    return fmt.Errorf("agent %q has no allowed_tools defined — add to agents-index.json", member.Agent)
}
```

This aligns with Item 24 (agent validation on startup): agents must be fully specified in agents-index.json before use.

**Backward-compatible alternative** (if strict mode is too disruptive): Use the SAME read-only default across ALL paths: `["Read", "Glob", "Grep"]`. Write/Edit/Bash require explicit opt-in in agents-index.json.

### Consistency with Existing Tools Field

The existing `tools` field in agents-index.json documents what tools an agent uses. The new `cli_flags.allowed_tools` field specifies what tools are **permitted** via CLI.

**Recommendation**: Keep both in sync. Add a validation script:

```bash
#!/bin/bash
# validate-agents-index.sh
# Checks that cli_flags.allowed_tools matches the tools field

jq -r '.agents[] | select(.cli_flags != null) |
  if (.tools | sort) == (.cli_flags.allowed_tools | sort) then empty
  else "Mismatch: \(.id)" end' agents-index.json
```

### Validation Script

**File**: `.claude/scripts/validate-cli-flags.sh`

```bash
#!/bin/bash
# Validates that cli_flags.allowed_tools matches the tools field

set -e

echo "Validating cli_flags consistency in agents-index.json..."

AGENTS_INDEX=".claude/agents/agents-index.json"

# Check that all agents with tools field also have cli_flags
mismatches=$(jq -r '.agents[] | select(.tools != null and .cli_flags == null) | .id' "$AGENTS_INDEX")

if [ -n "$mismatches" ]; then
  echo "ERROR: Agents missing cli_flags but have tools field:"
  echo "$mismatches"
  exit 1
fi

# Check that allowed_tools matches tools field (ignoring order)
jq -r '.agents[] | select(.cli_flags != null and .tools != null) |
  if (.tools | sort) == (.cli_flags.allowed_tools | sort) then empty
  else "Mismatch: \(.id)\n  tools: \(.tools)\n  allowed_tools: \(.cli_flags.allowed_tools)" end' \
  "$AGENTS_INDEX"

if [ $? -eq 0 ]; then
  echo "✓ All cli_flags.allowed_tools match tools field"
else
  echo "ERROR: Mismatches found (see above)"
  exit 1
fi
```
