## Description

Migrate the `/braintrust` workflow from foreground MCP-spawn orchestration to a hybrid foreground-interview + background-execution pattern using `gogent-team-run`.

**Current behavior:** Mozart conducts interview (foreground), then spawns Einstein + Staff-Architect + Beethoven via `mcp__gofortress__spawn_agent` (foreground, ~6-9 minutes TUI freeze).

**Target behavior:** Mozart conducts interview (foreground, ~30s), generates team config + stdin files, launches `gogent-team-run` (background), returns to user immediately. Background execution: Wave 1 (Einstein + Staff-Architect parallel) -> inter-wave synthesis (`gogent-team-prepare-synthesis`) -> Wave 2 (Beethoven synthesis).

**Key challenge:** Mozart's `mozart.md` Phase 2.5 stdin templates (from TC-018) do not comply with TC-009's `einstein.json` schema. These must be reconciled.

## Learnings from TC-013a Live Testing (2026-02-08)

These findings were discovered during end-to-end validation of the review workflow with `gogent-team-run`. They apply directly to TC-013b implementation.

### What works (validated)

| Component | Status | Evidence |
|-----------|--------|----------|
| LLM config.json generation | **100% pass** | 11/11 files valid across 3 scenarios (2-reviewer, 4-reviewer, backend-only) |
| LLM stdin generation | **100% pass** | All files pass `schemas/stdin/reviewer.json` validation AND envelope builder checks |
| `gogent-team-run` binary launch | **Working** | PID written to config.json, runner.log produced, daemon mode functional |
| Agent execution | **Working** | Haiku reviewer ran 20s, exit code 0, produced quality review output |
| Cost extraction | **Working** | `parseCLIOutput()` extracts `total_cost_usd` from CLI JSON array ($0.071 from haiku) |
| Budget reconciliation | **Working** | $2.00 → $1.93 after single agent completion, matches extracted cost |
| Stdout file writing | **Working** | `writeStdoutFile()` creates file from CLI output (but see format issue below) |

### What was broken and fixed

1. **`agents-index.json` flag format** — All 32 agents had `"--permission-mode delegate"` as ONE string. Go's `exec.Command` passes each argv element as a separate OS argument, so the CLI received `--permission-mode delegate` as one arg and failed to parse it. Agents crashed with exit code 1 in <0.5s. **Fixed**: split to `"--permission-mode", "delegate"`.

2. **Missing `--model` flag** — `buildCLIArgs()` in `spawn.go` constructed args as `["-p", "--output-format", "json", "--allowedTools", "Read,Glob,Grep", "--permission-mode", "delegate"]` but never included `--model`. Agents defaulted to the session model (Opus at $0.24/run) instead of their configured model (haiku at $0.07/run). **Fixed**: `--model` now passed from agent config.

3. **CLI output format mismatch** — `claude --output-format json` outputs a JSON **array** `[{type: "system"}, {type: "assistant"}, {type: "result"}]`, not a single object. `extractCostFromCLIOutput()` tried `json.Unmarshal(output, &map[string]interface{})` which failed: "cannot unmarshal array into Go value". **Fixed**: new `parseCLIOutput()` iterates the array, finds `type: "result"`, extracts `total_cost_usd` and `result` text.

4. **No stdout file writer** — After agent completion, the binary validated the stdout file but never wrote it. The agent's response stayed in the captured CLI output buffer. **Fixed**: new `writeStdoutFile()` in `finalizeSpawn()` with three-tier extraction: (1) parse result as JSON object, (2) extract ```json code block, (3) fallback wrap in `{"agent": ..., "raw_output": true, "result": "..."}`.

### Stdout format compliance — RESOLVED (2026-02-08)

**Previously broken:** Agents responded in natural markdown because `buildPromptEnvelope()` had no output format instructions.

**Fix applied in TC-013a:** `buildPromptEnvelope()` in `envelope.go` now:
1. Accepts `workflowType` parameter (passed from `TeamConfig.WorkflowType` in `prepareSpawn()`)
2. Calls `resolveStdoutSchema(workflowType, agentID)` which uses candidate-based file lookup:
   - Exact: `{workflow}-{agentID}.json` (e.g., `braintrust-einstein.json`)
   - Suffix-stripped: strips `-reviewer`, `-critical-review`, `-pro`, etc. (e.g., `braintrust-staff-architect.json` from `staff-architect-critical-review`)
   - Worker fallback: `{workflow}-worker.json`
3. Embeds the `stdout` portion of the contract schema in an "Expected Output Format" section
4. Instructs agent: "Your response MUST be a single JSON code block" + schema + `$schema` field instruction
5. `writeStdoutFile()` tier-2 extracts the ```json code block; `validateStdout()` checks `$schema` and `status` fields

**What this means for TC-013b:**
- Einstein, Staff-Architect, and Beethoven stdout compliance is handled automatically — `braintrust-einstein.json`, `braintrust-staff-architect.json`, `braintrust-beethoven.json` all have `stdout` sections that will be resolved and embedded
- `gogent-team-prepare-synthesis` should receive structured JSON stdout files, enabling proper field extraction for `pre-synthesis.md`
- **Risk:** Opus agents are more instruction-compliant than Haiku, so JSON compliance should be higher for braintrust than review. Still verify with a live test.
- **Fallback:** If an agent ignores the instruction, `writeStdoutFile()` tier-3 wraps in `{"raw_output": true, ...}`. `gogent-team-prepare-synthesis` should handle this gracefully (degraded but functional)

### 2. Add Phase 6 Team-Run Dispatch to mozart.md

Mozart Phases 6-7 currently describe MCP spawn pattern. Add a parallel section for team-run dispatch:

**After interview confirmation (Phase 5):**
1. Create team directory: `$GOGENT_SESSION_DIR/teams/{timestamp}.braintrust/` (uses `GOGENT_SESSION_DIR` env var set by TUI, matching TC-013a pattern)

### 3. Inter-Wave Synthesis Integration

The braintrust template sets `waves[0].on_complete_script: "gogent-team-prepare-synthesis"` for Wave 1. After Einstein + Staff-Architect complete, the Go binary calls this script which:
1. Reads `stdout_einstein.json` and `stdout_staff-arch.json` (note: `staff-arch`, not `staff-architect` — see D-6 fix)
2. Extracts key sections with graceful degradation
3. Writes `pre-synthesis.md` to team directory
4. Beethoven (Wave 2) receives this as input

**Stdout compliance:** The TC-013a envelope fix (`resolveStdoutSchema` in `envelope.go`) automatically embeds the stdout schema from `braintrust-einstein.json` and `braintrust-staff-architect.json` into agent prompts. Agents should produce compliant JSON stdout. The `raw_output` fallback path in `writeStdoutFile()` remains as safety net.

**Verify before starting TC-013b:**
1. `gogent-team-prepare-synthesis` binary is on PATH (`go install ./cmd/gogent-team-prepare-synthesis/` — symlink to `~/.local/bin/` since `~/go/bin/` is not on PATH)
2. The binary handles both structured JSON stdout AND the fallback `raw_output` format (test with both)
3. Run a quick review workflow (`/review` with team pattern) to confirm structured JSON stdout end-to-end before attempting braintrust

### 4. Beethoven Stdin — pre_synthesis_path Mechanism

Beethoven's stdin is written **before Wave 1 runs**, so it cannot contain Einstein/Staff-Architect output objects. Instead, Beethoven receives a file path and reads the synthesis at runtime.

**Mechanism:** The `beethoven.json` schema now requires `pre_synthesis_path` (absolute path string) instead of `einstein_analysis`/`staff_architect_review` objects. The lifecycle:
1. Config generator writes `stdin_beethoven.json` with `pre_synthesis_path` pointing to `{team_dir}/pre-synthesis.md`
2. Wave 1 runs (Einstein + Staff-Architect produce stdout files)
3. Inter-wave script (`gogent-team-prepare-synthesis`) reads stdout files → writes `pre-synthesis.md`
4. Wave 2 starts: Beethoven reads `pre_synthesis_path` via Read tool at runtime

**stdin_beethoven.json must include:**
- `pre_synthesis_path`: `"{team_dir}/pre-synthesis.md"` (absolute path)
- `problem_brief`: copy of the problem brief from Mozart
- `description`: `"Synthesize analyses for: {problem_brief.title}"`

### Binary Installation Note (from TC-013a testing)

`go install` puts binaries in `~/go/bin/` but `~/go/bin/` is NOT on PATH. The system PATH includes `~/.local/bin/`. All other gogent binaries are symlinked: `ln -sf ~/go/bin/gogent-* ~/.local/bin/`. After any `go install`, you must create/update symlinks:

```bash
ln -sf ~/go/bin/gogent-team-run ~/.local/bin/gogent-team-run
ln -sf ~/go/bin/gogent-team-prepare-synthesis ~/.local/bin/gogent-team-prepare-synthesis
```
