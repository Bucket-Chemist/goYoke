# TC-006: Add `project_root` to Team Config Schema

**Phase**: Phase 1: Schema Design

**Priority**: HIGH

**Blocked By**: none

**Effort**: 0.5 days

## Description

The Go binary uses `cmd.Dir = projectRoot` when spawning agents, but `projectRoot` is never defined in the config schema. The binary needs to know the project root for agents' Read/Write tool calls to resolve paths correctly.

Add `project_root` as a top-level field in the team config.json schema. The planner (Mozart or Router) resolves this at planning time and writes it into config.json. The Go binary reads it and sets `cmd.Dir` accordingly.

### Step 2: Add `project_root` Field to Schema

For each team template, add at top level (before `waves` field):

```json
{
  "$schema": "https://json-schema.org/draft-07/schema",
  "type": "object",
  "required": ["project_root", "waves", "budget_remaining_usd"],
  "properties": {
    "project_root": {
      "type": "string",
      "description": "Absolute path to project root. Agents spawn with cmd.Dir set to this path.",
      "pattern": "^/",
      "examples": ["/home/user/Documents/GOgent-Fortress"]
    },
    "waves": {
      "type": "array",
      "items": { "$ref": "#/definitions/Wave" }
    },
    "budget_remaining_usd": {
      "type": "number",
      "minimum": 0
    }
  }
}
```

**Key properties**:
- `required: ["project_root", ...]`: Enforces presence (planner must fill)
- `pattern: "^/"`: Must be absolute path (no relative paths like `./`)
- Position: Before `waves` (top-level, easy to find)

### Step 4: Document Resolution Logic

Create `.claude/schemas/teams/PROJECT-ROOT-RESOLUTION.md`:

```markdown
# Project Root Resolution

## Overview

`project_root` is resolved by the planner (Mozart or Router) at team creation time.

## Resolution Priority

1. **Explicit environment**: `GOGENT_PROJECT_DIR`
2. **Claude CLI environment**: `CLAUDE_PROJECT_DIR`
3. **Current working directory**: `os.Getwd()`

## Implementation

### In Planning Agent (Mozart/Router)

```go
func resolveProjectRoot() string {
    if root := os.Getenv("GOGENT_PROJECT_DIR"); root != "" {
        return root
    }
    if root := os.Getenv("CLAUDE_PROJECT_DIR"); root != "" {
        return root
    }
    root, _ := os.Getwd()
    return root
}
```

Write this to `config.json` at team creation.

### In Go Binary (gogent-team-run)

Read from config.json and use for spawning:

```go
cmd := exec.Command("claude", args...)
cmd.Dir = config.ProjectRoot // Set working directory
```

## Validation

Team config schema enforces:
- Required field (cannot be omitted)
- Absolute path (starts with `/`)
- String type

Runtime validation in Go binary:

```go
if !filepath.IsAbs(config.ProjectRoot) {
    return fmt.Errorf("project_root must be absolute: %s", config.ProjectRoot)
}
if _, err := os.Stat(config.ProjectRoot); err != nil {
    return fmt.Errorf("project_root does not exist: %w", err)
}
```

## Agent Behavior

When agent receives stdin with `paths.project_root`, relative paths in prompts are resolved:

```
Read the file at src/main.go
→ Resolves to {project_root}/src/main.go
```

Tool calls (Read, Write, Glob) operate within project_root scope.
```

## Code Guidelines

### Schema Conventions
- Use `snake_case` for JSON field names (consistent with `budget_remaining_usd`)
- Place top-level metadata fields BEFORE structural fields (project_root before waves)
- Use JSON Schema `pattern` for validation (absolute path: `^/`)

### Go Code Conventions
- Use `filepath.IsAbs()` for path validation (stdlib, cross-platform)
- Use `os.Stat()` to verify directory exists (fail fast on misconfiguration)
- Error messages include field value for debugging: `"invalid project_root: %s"`

### Reference Pattern

See `cmd/gogent-validate/main.go:66-74` for environment variable resolution:

```go
projectDir := os.Getenv("GOGENT_PROJECT_DIR")
if projectDir == "" {
    projectDir = os.Getenv("CLAUDE_PROJECT_DIR")
}
if projectDir == "" {
    projectDir, _ = os.Getwd()
}
```

Use same pattern in planning agents.

func TestProjectRootValidation(t *testing.T) {
    tests := []struct {
        name      string
        input     string
        expectErr bool
        errMsg    string
    }{
        {
            name:      "valid absolute path",
            input:     `{"project_root": "/home/user", "waves": [], "budget_remaining_usd": 10}`,
            expectErr: false,
        },
        {
            name:      "missing project_root",
            input:     `{"waves": [], "budget_remaining_usd": 10}`,
            expectErr: true,
            errMsg:    "required property 'project_root'",
        },
        {
            name:      "relative path",
            input:     `{"project_root": "./relative", "waves": [], "budget_remaining_usd": 10}`,
            expectErr: true,
            errMsg:    "does not match pattern",
        },
    }

    for _, tc := range tests {
        t.Run(tc.name, func(t *testing.T) {
            var config TeamConfig
            err := json.Unmarshal([]byte(tc.input), &config)

            // Note: Go's json.Unmarshal doesn't validate against JSON Schema
            // This test would need a JSON Schema validator library
            // OR validate after unmarshal with custom logic

            if tc.expectErr {
                assert.Error(t, err)
                assert.Contains(t, err.Error(), tc.errMsg)
            } else {
                assert.NoError(t, err)
            }
        })
    }
}
```
