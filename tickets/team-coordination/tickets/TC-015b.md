# TC-015b: Team Visibility in TUI

**Phase**: Independent
**Priority**: P1 (HIGH)
**Blocked By**: none
**Effort**: 2-3 days
**Status**: pending

---

## Description

Implement TUI components for background team visibility. Users launch teams via `/implement`, `/ticket`, or `/braintrust` but have no way to see progress beyond the `StatusLine` team count badge. This ticket implements the `useTeams` hook, `TeamList` component, and `TeamDetail` component from the TUI-team-visibility-spec.

**Why this matters:** Team-run (TC-013a/b/c) moves orchestration to background Go processes. Users need visibility into what those processes are doing — which teams exist, wave progress, member status, cost breakdown. Without this, background execution feels like a black box.

**Reference:** `tickets/team-coordination/TUI-team-visibility-spec.md` (full exploration spec)

## Implementation Instructions

### 1. Replace `useTeamCount` with `useTeams` hook

**File:** New `packages/tui/src/hooks/useTeams.ts` (replaces or wraps `useTeamCount`)

```typescript
interface TeamSummary {
  dir: string;
  name: string;
  workflowType: string;
  status: "pending" | "running" | "completed" | "failed";
  backgroundPid: number | null;
  alive: boolean;
  budgetMax: number;
  budgetRemaining: number;
  startedAt: string | null;
  completedAt: string | null;
  totalCost: number;
  waveCount: number;
  currentWave: number;
  memberCount: number;
  completedMembers: number;
  failedMembers: number;
}
```

- Enumerate `$GOGENT_SESSION_DIR/teams/*/config.json`
- Parse each config.json, compute summary fields
- PID liveness check via existing `isPidAlive` utility
- Adaptive polling: 5s for running teams, 30s if all completed
- Return sorted by `created_at` descending

### 2. Update TeamsSlice in Zustand store

**Files:**
- `packages/tui/src/store/types.ts` — Update `TeamsSlice` interface
- `packages/tui/src/store/slices/teams.ts` — Implement new actions

Replace the current minimal `backgroundTeamCount: number` with:

```typescript
interface TeamsSlice {
  teams: TeamSummary[];
  selectedTeamDir: string | null;
  selectedTeamDetail: TeamConfig | null;
  setTeams: (teams: TeamSummary[]) => void;
  selectTeam: (dir: string | null) => void;
  setTeamDetail: (config: TeamConfig | null) => void;
}
```

### 3. TeamList component

**File:** New `packages/tui/src/components/TeamList.tsx`

- Right panel, accessible via `rightPanelMode: "teams"`
- Shows team name, workflow type, status, wave progress, cost, duration

## Open Questions (from visibility spec)

1. **Poll vs fs.watch:** Start with polling (5s). `fs.watch` is unreliable on some Linux filesystems.
2. **Historical teams:** Keep completed teams in list for session duration. Cleared by `gogent-archive`.
3. **Completion toast:** Detect status transitions in `useTeams` and fire `addToast` when team completes.
4. **Cost rollup:** Defer rolling team costs into `SessionSlice.totalCost` — keep them separate for now.
