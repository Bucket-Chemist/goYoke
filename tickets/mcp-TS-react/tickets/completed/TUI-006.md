# TUI-006: Modal Queue with Promise Resolution

## Description

Implement the modal queue slice with Promise-based resolution. This enables MCP tools to call `enqueue()` and await user response.

## Acceptance Criteria

- [x] Modal requests can be enqueued
- [x] Enqueue returns a Promise that resolves on user response
- [x] Multiple modals queue correctly (FIFO)
- [x] Cancel rejects the Promise
- [x] Timeout support (optional per-request)
- [x] Type safety for different modal types
- [x] Unit tests pass

## Tasks

1. Define modal types (`src/store/slices/modal.ts`):
   ```typescript
   interface ModalRequest<T = unknown> {
     id: string;
     type: "ask" | "confirm" | "input" | "select";
     payload: T;
     resolve: (response: ModalResponse) => void;
     reject: (error: Error) => void;
     timeout?: number;
   }

   interface AskPayload {
     message: string;
     options?: string[];
     defaultValue?: string;
   }

   interface ConfirmPayload {
     action: string;
     destructive?: boolean;
   }

   interface InputPayload {
     prompt: string;
     placeholder?: string;
   }

   interface SelectPayload {
     message: string;
     options: Array<{ label: string; value: string }>;
   }

   type ModalResponse =
     | { type: "ask"; value: string }
     | { type: "confirm"; confirmed: boolean; cancelled: boolean }
     | { type: "input"; value: string }
     | { type: "select"; selected: string; index: number };
   ```

2. Implement modal slice:
   ```typescript
   interface ModalSlice {
     modalQueue: ModalRequest[];
     enqueue: <T>(request: Omit<ModalRequest<T>, "id" | "resolve" | "reject">) => Promise<ModalResponse>;
     dequeue: (id: string, response: ModalResponse) => void;
     cancel: (id: string) => void;
   }
   ```

3. Implement enqueue with Promise:
   ```typescript
   enqueue: (request) => {
     return new Promise((resolve, reject) => {
       const id = nanoid();
       const fullRequest = { ...request, id, resolve, reject };
       set((state) => ({
         modalQueue: [...state.modalQueue, fullRequest]
       }));

       // Optional timeout
       if (request.timeout) {
         setTimeout(() => {
           reject(new Error("Modal timeout"));
           // Remove from queue
         }, request.timeout);
       }
     });
   }
   ```

4. Write unit tests:
   - Enqueue/dequeue flow
   - Multiple modal queueing
   - Timeout behavior
   - Cancel behavior
