# TUI-022: Integration Tests & Rollback Rehearsal

## Description

Run full integration test suite and perform rollback rehearsal to verify Go TUI can resume sessions created by TypeScript TUI.

## Acceptance Criteria

- [ ] All integration tests pass
- [ ] Rollback rehearsal completed successfully
- [ ] Session created in TS readable by Go
- [ ] Session created in Go readable by TS
- [ ] No data loss during rollback
- [ ] Performance comparison documented
- [ ] Sign-off checklist complete

## Tasks

### Integration Test Suite

1. Run full test suite:
   ```bash
   cd packages/tui
   npm run test
   npm run test:integration
   npm run typecheck
   npm run lint
   ```

2. Verify all quality gates:
   - [ ] TypeScript strict mode, zero `any`
   - [ ] ESLint clean (no warnings)
   - [ ] Test coverage >80%
   - [ ] No known memory leaks

### Rollback Rehearsal

3. Execute rollback test procedure:
   ```bash
   # Step 1: Create session in TS TUI
   ./bin/gofortress-tui
   # Have conversation, accumulate 10+ messages
   # Note session ID and cost
   # Exit

   # Step 2: Resume in Go TUI
   ./bin/gofortress-tui --legacy --session <session-id>
   # Verify: session loads
   # Verify: cost preserved
   # Verify: can continue conversation

   # Step 3: Return to TS TUI
   ./bin/gofortress-tui --session <session-id>
   # Verify: session loads
   # Verify: all data intact
   ```
