# TUI-005: Zustand Store Foundation

## Description

Implement the Zustand store with all slices for messages, agents, session, and UI state. This is the central state management for the entire TUI.

## Acceptance Criteria

- [ ] Store created with TypeScript types
- [ ] Messages slice: add, update partial
- [ ] Agents slice: tree operations (add, update, select)
- [ ] Session slice: cost, tokens (Go format compatible)
- [ ] UI slice: focus, streaming state
- [ ] **Input slice: input history for up/down navigation (ephemeral, not persisted)**
- [ ] Messages persist across component re-renders
- [ ] Partial messages update in place (no flicker)
- [ ] Unit tests pass

## Input History Behavior (GAP-2 Resolution)

The input history slice is **ephemeral** (not persisted to session files):

```typescript
// src/store/slices/input.ts
interface InputSlice {
  inputHistory: string[];        // Max 100 entries
  inputHistoryIndex: number;     // -1 = not navigating, 0+ = position in history

  addToHistory: (input: string) => void;      // Adds to front, dedupes
  navigateHistory: (dir: "up" | "down") => string | null;
  resetHistoryIndex: () => void;              // Called on new input
}

// Behavior:
// - Up arrow: index++, return history[index]
// - Down arrow: index--, return history[index] or "" if index < 0
// - On submit: addToHistory(input), resetHistoryIndex()
// - On any typing: resetHistoryIndex()
```

**Why ephemeral:** Input history is session-local UX convenience. Claude CLI handles conversation persistence; we only track input history for up-arrow recall within the current TUI session.
