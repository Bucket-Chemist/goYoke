## Executive Summary

GOgent-Fortress conflates static configuration and runtime state into `.claude/`, which CC's sandbox now blocks for writes. This migration introduces a clean separation: `.claude/` = read-only config (CLAUDE.md, agents, conventions, schemas), `.gogent/` = read-write runtime (sessions, memory, teams, telemetry). A central `RuntimeDir()` function extends the existing `WithProjectDir` pattern in `pkg/config/paths.go`, and two session tracks handle project-local hooks vs user-global TUI sessions separately.

## Strategic Approach

- **Conceptual Framework**: XDG Base Directory model at project scope
- **Primary Abstraction**: `RuntimeDir(projectDir)` with `sync.Once` lazy init
- **Session Split**: Hook sessions → `{projectDir}/.gogent/sessions/`, TUI → `~/.gogent/sessions/`
- **Env Override**: `GOGENT_RUNTIME_DIR` for testing/CI flexibility
- **Rejected**: InitWorkspace (premature abstraction), schema mirroring (stale copies), feature flags (unnecessary complexity), partial migration (split-brain)
