# MIG-022: Integration canary test for sandbox behavior

## Description

Create a lightweight integration test in pkg/config/sandbox_canary_test.go that validates the migration premise:

1. Test that RuntimeDir() creates .gogent/ successfully
2. Test that os.WriteFile to RuntimeDir()/.../test.txt succeeds
3. Test that ProjectMemoryDir() creates .gogent/memory/ successfully
4. Document (via test name/comment) that .claude/ writes are blocked by CC sandbox — this test is the canary

This is a build-time validation, not a CC sandbox test (we can't run CC sandbox in unit tests). Its value is ensuring the new paths work and serving as documentation.
