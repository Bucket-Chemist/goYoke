## Description

Complete the binary with JSON output writing, manifest management, Makefile targets, and integration tests. After this ticket, the binary is fully functional: it discovers files, extracts Go symbols, writes schema-compliant JSON, and updates the manifest.

## Deliverables

### 1. Output Writer (`output.go`)

- `WriteModuleExtraction(outputDir, extraction)` → writes JSON to `extract/{module}.json`
- Atomic writes: write to `.tmp` then `os.Rename`
- Pretty-printed JSON (indented)
- `moduleToFilename()` → converts `cmd/gogent-validate` to `cmd--gogent-validate`

### 2. Manifest Management (`manifest.go`)

- `LoadManifest(path)` → reads existing manifest.json
- `UpdateManifest(root, modules, outputDir)` → writes/updates manifest with:
  - Per-file timestamps (mtime, extracted_at)
  - Per-module status
  - Stats (total files, symbols, functions, types)
  - `git_commit` (current HEAD)
  - `extractor_version`

### 3. Makefile Targets

Add to project Makefile:
```makefile
gogent-codebase-extract:
	go build -o bin/gogent-codebase-extract ./cmd/gogent-codebase-extract/

install-gogent-codebase-extract: gogent-codebase-extract
	cp bin/gogent-codebase-extract $(GOPATH)/bin/

test-codebase-extract-coverage:
	go test -coverprofile=cover.out ./cmd/gogent-codebase-extract/...
	go tool cover -func=cover.out
```

### 4. Integration Tests

- Full pipeline on testdata fixtures
- Full pipeline on GOgent-Fortress repo
- Output validates against `extraction.schema.json`
- Manifest written correctly with per-file timestamps
