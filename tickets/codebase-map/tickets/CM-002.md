# CM-002: Design Extraction JSON Schema

**Phase**: Phase 0: Design & Evaluation
**Priority**: HIGH
**Blocked By**: none
**Effort**: 0.5 days

## Description

Define the raw output format of `gogent-codebase-extract`. This schema represents the **deterministic** output from tree-sitter parsing - no LLM involvement. It must be language-agnostic (same schema whether parsing Go, Python, TypeScript, or R).

This schema is the contract between Stage 1 (Go binary) and Stage 2 (LLM enrichment).

## Implementation Instructions

### Step 3: Document Module Boundary Rules

Define how the Go binary determines module boundaries:

| Language | Module Boundary | Rule |
|----------|----------------|------|
| Go | Directory with `.go` files | Each directory = one module. Module name = relative path from project root. |
| Python | Directory with `__init__.py` OR directory with `.py` files | Package directories = modules. Single `.py` files without package = file-level module. |
| TypeScript | Directory | Each directory = one module. |
| R | Directory with `.R` files | Each directory = one module. |

**Special cases:**
- `vendor/`, `node_modules/`, `__pycache__/` → always excluded
- Files matching `.gitignore` → excluded
- Files with `// Code generated` or `# Generated` headers → excluded (configurable)
