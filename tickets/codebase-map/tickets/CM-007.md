# CM-007: Implement Cross-Reference and Call Graph Extraction

**Phase**: Phase 1: Go Binary Implementation
**Priority**: HIGH
**Blocked By**: CM-005 (core binary with parsed AST access), CM-006a (for non-Go call queries — Python/TS call queries require their respective language extractors)
**Effort**: 1-2 days

> **Note**: Go call graph can be implemented with CM-005 alone. Python/TypeScript call queries depend on CM-006a (TypeScript) being complete. If implementing before CM-006a, implement Go call graph only and add Python/TS call queries after CM-006a ships.

## Description

Add call site detection to `gogent-codebase-extract`. For each function/method, determine which other functions it calls (`calls` field) and which functions call it (`called_by` field). This enables dependency graph generation in Stage 3.

Call graph extraction operates as a post-processing pass after symbol extraction. It uses tree-sitter queries to find function call sites, then resolves them against the symbol table built during extraction.

```go
// CallSite represents a function call found in source code
type CallSite struct {
    CallerFunc string  // Function containing the call
    CalleeName string  // Name being called
    Line       int     // Line number of the call
    IsMethod   bool    // Is this a method call (x.Foo())?
    Receiver   string  // Receiver expression if method call
}

// ExtractCallSites finds all function/method call sites in a parsed tree
func ExtractCallSites(tree *sitter.Tree, source []byte, lang LanguageExtractor) ([]CallSite, error) {
    query, err := sitter.NewQuery(lang.CallQuerySource(), lang.TreeSitterLang())
    if err != nil {
        return nil, fmt.Errorf("compile call query: %w", err)
    }

    cursor := sitter.NewQueryCursor()
    cursor.Exec(query, tree.RootNode())

    var sites []CallSite
    for {
        match, ok := cursor.NextMatch()
        if !ok { break }

        site := parseCallMatch(match, source, tree)
        sites = append(sites, site)
    }

    return sites, nil
}

// ResolveCallGraph populates calls/called_by fields on extracted symbols
func ResolveCallGraph(modules map[string]*ModuleExtraction) {
    // Build global symbol table: fully qualified name → *Symbol
    symbolTable := buildSymbolTable(modules)

    for _, module := range modules {
        for fi := range module.Files {
            for si := range module.Files[fi].Symbols {
                sym := &module.Files[fi].Symbols[si]

                // For each call site within this symbol's line range
                for _, site := range sym.callSites {
                    // Resolve callee name against symbol table
                    // Try: local (same file), module-scoped, then cross-module
                    resolved := resolveCallee(site, module, symbolTable)
                    if resolved != "" {
                        sym.Calls = appendUnique(sym.Calls, resolved)

                        // Add reverse reference
                        if target, ok := symbolTable[resolved]; ok {
                            target.CalledBy = appendUnique(target.CalledBy, sym.Name)
                        }
                    }
                }
            }
        }
    }
}

// SymbolTable provides two-level lookup: qualified name first, then module-scoped
// short name. This avoids the collision bug where global short-name indexing causes
// last-write-wins for common names like "New", "String", "Do".
//
// BEHAVIOR CONTRACT (implementing agent writes code against CM-001 selected library):
type SymbolTable struct {
    qualified map[string]*Symbol             // "module.Name" → sym (globally unique)
    perModule map[string]map[string]*Symbol   // modulePath → shortName → sym (module-scoped)
}

// buildSymbolTable creates a two-level lookup from qualified + module-scoped names to Symbol
func buildSymbolTable(modules map[string]*ModuleExtraction) *SymbolTable {
    table := &SymbolTable{
        qualified: make(map[string]*Symbol),
        perModule: make(map[string]map[string]*Symbol),
    }
    for modPath, mod := range modules {
        modTable := make(map[string]*Symbol)
        for fi := range mod.Files {
            for si := range mod.Files[fi].Symbols {
                sym := &mod.Files[fi].Symbols[si]
                // Qualified name: "module.SymbolName" or "module.Receiver.MethodName"
                qualName := modPath + "." + sym.Name
                if sym.Receiver != nil {
                    qualName = modPath + "." + *sym.Receiver + "." + sym.Name
                }
                table.qualified[qualName] = sym
                // Module-scoped short name (NOT global — prevents collision)
                modTable[sym.Name] = sym
            }
        }
        table.perModule[modPath] = modTable
    }
    return table
}

// Resolve looks up a callee name with priority: qualified → same-module short name → nil
func (st *SymbolTable) Resolve(calleeName, callerModule string) *Symbol {
    // 1. Try qualified name first (cross-module calls like "internal/routing.BlockResponse")
    if sym, ok := st.qualified[calleeName]; ok {
        return sym
    }
    // 2. Try module-scoped short name (intra-module calls like "validateTask")
    if modTable, ok := st.perModule[callerModule]; ok {
        if sym, ok := modTable[calleeName]; ok {
            return sym
        }
    }
    // 3. Unresolved — do NOT guess, return nil
    return nil
}
```

### Call Site Queries per Language

**Go** (`queries/go-calls.scm`):

```scheme
;; Function calls
(call_expression
  function: (identifier) @call.name
) @call.site

;; Method calls
(call_expression
  function: (selector_expression
    operand: (_) @call.receiver
    field: (field_identifier) @call.name
  )
) @call.site

;; Qualified package calls (pkg.Func())
(call_expression
  function: (selector_expression
    operand: (identifier) @call.package
    field: (field_identifier) @call.name
  )
) @call.site
```

**Python** (`queries/python-calls.scm`):

```scheme
;; Function calls
(call
  function: (identifier) @call.name
) @call.site

;; Method calls
(call
  function: (attribute
    object: (_) @call.receiver
    attribute: (identifier) @call.name
  )
```

```scheme
(call_expression
```

### Resolution Strategy

Call resolution follows this priority order:

1. **Local scope**: Is the callee defined in the same file? → Use short name
2. **Module scope**: Is the callee defined in the same module (different file)? → Use short name
3. **Cross-module**: Does the callee match `package.Name` pattern? → Resolve via import graph
4. **Unresolved**: Log as unresolved (stdlib calls, external library calls). Do NOT include in `calls` field.

**What to include in `calls`/`called_by`**:
- Calls to functions/methods defined within this project
- Use short names for intra-module calls (e.g., `validateTask`)
- Use qualified names for cross-module calls (e.g., `internal/routing.BlockResponse`)

**What to exclude**:
- Stdlib calls (`fmt.Println`, `os.ReadFile`)
- External library calls
- Built-in function calls (`make`, `len`, `append` in Go; `print`, `range` in Python)

### Integration with Main Pipeline

The call graph extraction runs as an additional pass after symbol extraction:

```go
// In main.go, after all modules extracted:
func main() {
    // ... existing extraction ...

    // Post-process: call graph (CM-007)
    if !*skipCallGraph {
        log.Println("Building call graph...")
        ResolveCallGraph(allModules)

        // Re-write modules with populated calls/called_by
        for modulePath, extraction := range allModules {
            WriteModuleExtraction(*outputDir, extraction)
        }
    }
}
```

Add `--skip-call-graph` flag for faster extraction when call graph isn't needed.

## Code Guidelines

- Call graph is best-effort. Missing calls are acceptable; false positives are not.
- Do not attempt to resolve dynamic calls (reflection, callbacks, interface dispatch)
- Keep resolution simple: static name matching, not type analysis
- Log unresolved calls at `--verbose` level for debugging
- **Resolution priority**: qualified name → same-module short name → unresolved (skip). Never fall back to global short-name matching.
- **Behavior contracts**: Code examples describe WHAT behavior is expected, not specific API calls. The implementing agent writes code against the library selected in CM-001.
