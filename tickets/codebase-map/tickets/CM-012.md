# CM-012: Agent Context Injection for Module Maps

**Phase**: Phase 3: Incremental & Context Integration
**Priority**: LOW
**Blocked By**: CM-009 (enriched JSON files must exist in `.claude/codebase-map/enriched/`)
**Effort**: 1 day

## Description

When the router dispatches an implementation agent (e.g., `go-pro` for work on `internal/routing`), automatically inject the relevant module map from `.claude/codebase-map/` into the agent's context. This gives agents structural awareness without requiring them to discover the codebase themselves.

## Implementation Instructions

### Approach: Convention-Based Context Loading

Rather than a hook (which would require a new Go binary), use a convention where the router includes relevant module map data in the Task() prompt.

**Router behavior when dispatching to implementation agents:**

```
1. User request mentions files/modules → identify target module(s)
2. Check if .claude/codebase-map/{module}.json exists
3. If yes, extract summary (module_description, key_types, key_functions, import graph)
4. Include summary in Task() prompt under CONTEXT section
```

### Context Injection Format

**Template for injection (added to Task() prompt):**

```markdown
## Module Context (from codebase-map)

**Module**: {module_name}
**Category**: {module_category}
**Description**: {module_description}

**Key Types**: {key_types as comma-separated list}
**Key Functions**: {key_functions as comma-separated list}

**Dependencies** (imports from):
- {internal_import_1}
- {internal_import_2}

**Depended on by**:
- {modules that import this one}

**Symbol Summary** ({total_count} symbols):
- Functions: {function_count} ({exported_count} exported)
- Types: {type_count}
- Interfaces: {interface_count}
```

**Token budget**: Cap injection at ~500 tokens per module. If the enriched JSON is larger, extract only the summary fields above. Never inject the full symbol list.

### Router Implementation

Add a helper function the router calls before dispatching to implementation agents:

```javascript
// Pseudocode for router context injection
function getModuleContext(targetFiles) {
    const modules = identifyModules(targetFiles);
    const contexts = [];

    for (const mod of modules) {
        const mapPath = `.claude/codebase-map/${mod.replace(/\//g, '--')}.json`;
        if (fileExists(mapPath)) {
            const map = readJSON(mapPath);
            contexts.push(formatModuleSummary(map));
        }
    }

    return contexts.join('\n\n');
}

// Usage in Task() dispatch
Task({
    prompt: `AGENT: go-pro

TASK: Implement retry logic in internal/routing
CONTEXT:
${getModuleContext(['internal/routing'])}

... rest of prompt ...`
});
```

### Module Identification Heuristics

How the router determines which module(s) are relevant:

| User Request Pattern | Module Detection |
|---------------------|------------------|
| "fix bug in cmd/gogent-validate/main.go" | Module: `cmd/gogent-validate` |
| "refactor internal/routing" | Module: `internal/routing` |
| "add a new hook" | Detect from `cmd/` directories |
| No specific file mentioned | Skip injection |

**Rules:**
1. If user mentions specific files → extract module from file path
2. If user mentions a module/directory → use directly
3. If ambiguous → skip injection (don't guess)
4. Maximum 3 modules injected per dispatch (cap total tokens)

### Feature Flag

Context injection is **experimental**. Gate behind environment variable:

```
GOGENT_CODEBASE_MAP_INJECT=1   # Enable injection (default: OFF)
```

Only enable after `/codebase-map` has been validated on 2+ real repos (see CM-013).

**Measurement plan**: Track agent task completion rate and token usage with/without injection across 10+ dispatches to measure ROI before enabling by default.

### Graceful Degradation

If `.claude/codebase-map/` doesn't exist or has no data:
- **Do nothing**. Don't error, don't warn, just skip injection.
- The agent works fine without context injection - this is an enhancement, not a requirement.

If map data is stale (extracted_at > 7 days ago):
- Include context but add note: `(Note: Module map is {N} days old. Run /codebase-map --incremental to update.)`
