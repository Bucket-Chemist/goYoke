# CM-005b: Tree-sitter Integration + Go Extraction

**Phase**: Phase 1: Go Binary Implementation
**Priority**: CRITICAL
**Blocked By**: CM-005a (CLI skeleton + types), CM-001 (library selection)
**Effort**: 2-3 days
**Parent**: CM-005 (split into CM-005a → CM-005b → CM-005c)

## Description

Integrate the tree-sitter library selected in CM-001 into the binary scaffold from CM-005a. Implement Go language extraction: parsing, query execution, symbol extraction, import classification. After this ticket, the binary can extract Go symbols from real source files.

**Behavior contracts**: Code examples in CM-005 describe WHAT behavior is expected. This ticket's implementing agent writes code against the specific library selected in CM-001. Type definitions from CM-005a are the output contract.

## Deliverables

### 1. Parser (`parser.go`)

- BEHAVIOR: Create a parser for a given language using the selected tree-sitter library
- BEHAVIOR: Parse source bytes into an AST tree
- BEHAVIOR: Return error if parse fails completely
- BEHAVIOR: After parsing, walk tree for ERROR-type nodes and record in `parse_errors`
- BEHAVIOR: Log warning for files with parse errors but continue extraction
- BEHAVIOR: Detect language from file extension via `DetectLanguage()` function

### 2. Go Extractor (`extractor.go`)

Implement `GoExtractor` struct satisfying `LanguageExtractor` interface:

- `Language()` → "go"
- `Extensions()` → [".go"]
- `QuerySource()` → embedded `queries/go.scm`
- `IsExported(name)` → uppercase first letter check
- `ClassifyImport(path, projectModule)` → internal/external/stdlib classification
  - Internal: starts with project module path (from `go.mod`)
  - Stdlib: no dot in first path segment
  - External: everything else
- `ExtractSymbols(tree, source)` → runs queries, builds `[]Symbol` and `ImportGraph`

### 3. Go Queries (`queries/go.scm`)

Tree-sitter query file for Go language, as specified in CM-005:
- Function declarations (name, params, returns)
- Method declarations (receiver, name, params, returns)
- Struct type definitions
- Interface definitions
- Const declarations
- Var declarations
- Import declarations

### 4. Extraction Pipeline

Wire parser + extractor into main.go's extraction loop:
1. For each module's files → parse with tree-sitter
2. Run Go queries → extract symbols
3. Record parse errors (ERROR nodes) in `FileExtract.parse_errors`
4. Build `ModuleExtraction` struct
5. Print progress to stderr

**If CGo is required** (per CM-001): handle `CGO_ENABLED=1`, document C compiler requirement.
