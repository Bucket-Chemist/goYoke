# CM-005a: CLI Scaffolding + File Discovery

**Phase**: Phase 1: Go Binary Implementation
**Priority**: CRITICAL
**Blocked By**: CM-001 (library selection), CM-002 (extraction schema)
**Effort**: 1 day
**Parent**: CM-005 (split into CM-005a → CM-005b → CM-005c)

## Description

Scaffold the `gogent-codebase-extract` binary: entry point, CLI flag parsing, orchestration skeleton, file discovery, and type definitions. This sub-ticket produces a binary that compiles, accepts all CLI flags, and correctly discovers/groups source files by module — but does NOT yet parse or extract symbols.

## Deliverables

### 1. Package Structure (skeleton)

```
cmd/gogent-codebase-extract/
├── main.go              # Entry point, CLI flags, orchestration skeleton
├── discovery.go         # File discovery, .gitignore, filtering
├── types.go             # All struct definitions (from CM-005)
├── discovery_test.go    # File discovery tests
└── queries/             # Empty directory (populated by CM-005b)
```

### 2. CLI Interface

Implement all flags from CM-005 spec:
```
gogent-codebase-extract [flags]

Flags:
  --path <dir>          Project root to scan (default: ".")
  --output <dir>        Output directory (default: ".claude/codebase-map/extract/")
  --lang <language>     Force language (default: "auto")
  --files <file,...>    Extract specific files only (for incremental)
  --exclude <pattern>   Additional exclude patterns
  --include-generated   Include generated files (default: false)
  --manifest <path>     Manifest file path (default: ".claude/codebase-map/manifest.json")
  --skip-call-graph     Skip call graph extraction (CM-007)
  --incremental         Only extract changed files (CM-011)
  --version             Print version and exit
  --verbose             Verbose output to stderr

Exit codes:
  0  Success
  1  Error (parse failure, IO error)
  2  No source files found
```

### 3. File Discovery (`discovery.go`)

Full implementation of `DiscoverFiles()` as specified in CM-005:
- Walk directory tree respecting `.gitignore`
- Skip default exclusions (`.git/`, `vendor/`, `node_modules/`, etc.)
- Skip generated files (detect via first 10 lines)
- Group files by module (directory for Go, package for Python)
- Detect language per module from file extensions
- Support `--files` flag for specific files
- Support `--exclude` flag for additional patterns

### 4. Type Definitions (`types.go`)

All struct definitions from CM-005 spec: `ModuleExtraction`, `FileExtract`, `Symbol`, `Param`, `ReturnVal`, `ImportGraph`, `LanguageExtractor` interface, `DiscoveryOpts`.

Include `error_count` and `parse_errors` fields in `FileExtract` per review finding F3.

### 5. Main Entry Point (`main.go`)

Orchestration skeleton that:
1. Parses flags
2. Calls `DiscoverFiles()`
3. Prints summary of discovered modules to stderr
4. Exits with code 2 if no files found
5. Stub for extraction loop (populated by CM-005b)
