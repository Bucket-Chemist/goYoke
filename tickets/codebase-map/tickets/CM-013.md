# CM-013: Manual Smoke Test on Real Repos

**Phase**: Phase 2: Skill & Agent Integration (after CM-009)
**Priority**: HIGH
**Blocked By**: CM-009 (fork skill must be functional)
**Effort**: 0.5 days

## Description

End-to-end manual validation of `/codebase-map` on 2-3 real repositories. This ticket ensures the pipeline produces correct, useful output on actual codebases — not just test fixtures. No automated tests; this is human-verified QA.

## Test Repos

| Repo | Languages | Est. Files | Purpose |
|------|-----------|-----------|---------|
| **GOgent-Fortress** (this repo) | Go + TypeScript | ~100 | Primary use case, multi-language |
| **Small open-source Go project** | Go | <50 | Pure Go validation, fast iteration |
| **(Optional) TypeScript project** | TypeScript | <100 | If CM-006a shipped, validate TS extraction |

## Validation Procedure

### Test 1: Full Repo Map

Run `/codebase-map` on GOgent-Fortress.

**Verify:**
- [ ] `/codebase-map` completes without errors
- [ ] All Go modules extracted (spot-check 3 modules in `.claude/codebase-map/extract/`)
- [ ] TypeScript modules extracted if CM-006a shipped (spot-check `packages/tui/`)
- [ ] Enriched JSON in `.claude/codebase-map/enriched/` has all semantic fields
- [ ] Enrichment descriptions are meaningful (not generic "This is a function")
- [ ] `module_identity` classifications make sense (e.g., `cmd/*` modules are "command")
- [ ] ARCHITECTURE.md renders in GitHub preview
- [ ] Mermaid diagrams render in VS Code / mermaid.live
- [ ] Module dependency graph is accurate (spot-check 5 edges against actual imports)
- [ ] Manifest.json has per-file timestamps and correct stats

### Test 2: Incremental Update

Modify 1 file in GOgent-Fortress, then run `/codebase-map --incremental`.

**Verify:**
- [ ] Only the modified module is re-extracted
- [ ] Other modules preserved (unchanged enriched JSON files)
- [ ] ARCHITECTURE.md regenerated with updated content
- [ ] Incremental is noticeably faster than full map
- [ ] Manifest.json timestamps updated for changed file only

### Test 3: Scoped Map

Run `/codebase-map cmd/gogent-validate` (single module).

**Verify:**
- [ ] Only the specified module extracted and enriched
- [ ] Output files created only for that module
- [ ] No errors about other modules

### Test 4: Edge Cases

- [ ] `/codebase-map --no-enrich` produces raw JSON only (no enriched files)
- [ ] `/codebase-map --skip-call-graph` works (no `calls`/`called_by` fields)
- [ ] Error reporting: what happens if `gogent-codebase-extract` binary is not built?
- [ ] Empty module: directory with no extractable symbols

### Test 5: Output Quality Spot-Check

For 3 randomly selected modules, verify:
- [ ] `key_types` lists the actual important types (not helper structs)
- [ ] `key_functions` lists the actual important functions (not internal helpers)
- [ ] `architectural_role` assignments are reasonable
- [ ] Cross-module call graph edges (if CM-007 shipped) are accurate

## Bug Documentation

For each issue found, create a sub-entry in `tickets/codebase-map/SMOKE-TEST-RESULTS.md`:

