# CM-011: Git-Aware Incremental Mapping

**Phase**: Phase 3: Incremental & Context Integration
**Priority**: MEDIUM
**Blocked By**: CM-005 (core binary with manifest support), CM-004 (incremental strategy design)
**Effort**: 1-2 days

## Description

Implement incremental mapping support so `/codebase-map --incremental` only re-extracts and re-enriches files that changed since the last map. This makes the tool practical for ongoing use in active repos.

## Implementation Instructions

### Step 1: Change Detection in Go Binary

Add to `cmd/gogent-codebase-extract/manifest.go`:

```go
// DetectChanges compares manifest timestamps against current file state
func DetectChanges(manifestPath, projectRoot string) (*ChangeSet, error) {
    manifest, err := LoadManifest(manifestPath)
    if err != nil {
        // No manifest = first run, map everything
        return &ChangeSet{FullRemap: true}, nil
    }

    changes := &ChangeSet{}

    // Strategy 1: Git-based (preferred, more accurate)
    if isGitRepo(projectRoot) {
        gitChanges, err := gitChangedFiles(projectRoot, manifest.GitCommit)
        if err == nil {
            changes.ChangedFiles = gitChanges.Modified
            changes.NewFiles = gitChanges.Added
            changes.DeletedFiles = gitChanges.Deleted
            changes.Method = "git"
            return changes, nil
        }
        // Fall through to mtime if git fails
    }

    // Strategy 2: mtime comparison (fallback)
    for modPath, modInfo := range manifest.Modules {
        for filePath, fileInfo := range modInfo.Files {
            absPath := filepath.Join(projectRoot, filePath)
            stat, err := os.Stat(absPath)
            if err != nil {
                // File deleted
                changes.DeletedFiles = append(changes.DeletedFiles, filePath)
                continue
            }
            if stat.ModTime().After(parseTime(fileInfo.Mtime)) {
                changes.ChangedFiles = append(changes.ChangedFiles, filePath)
            }
        }
    }

    // Detect new files (in discovered set but not in manifest)
    discovered, _ := DiscoverFiles(projectRoot, DiscoveryOpts{})
    for _, files := range discovered {
        for _, f := range files {
            if !manifest.HasFile(f) {
                changes.NewFiles = append(changes.NewFiles, f)
            }
        }
    }

    changes.Method = "mtime"
    return changes, nil
}

type ChangeSet struct {
    FullRemap    bool
    ChangedFiles []string
    NewFiles     []string
    DeletedFiles []string
    Method       string  // "git" or "mtime"
}

// gitChangedFiles uses git diff to find changes since a commit
func gitChangedFiles(projectRoot, sinceCommit string) (*GitChanges, error) {
    // git diff --name-status <commit> HEAD
    cmd := exec.Command("git", "diff", "--name-status", sinceCommit, "HEAD")
    cmd.Dir = projectRoot
    output, err := cmd.Output()
    if err != nil {
        return nil, fmt.Errorf("git diff: %w", err)
    }

    changes := &GitChanges{}
    for _, line := range strings.Split(string(output), "\n") {
        if len(line) < 3 { continue }
        status := line[0]
        file := strings.TrimSpace(line[2:])
        switch status {
        case 'A':
            changes.Added = append(changes.Added, file)
        case 'M':
            changes.Modified = append(changes.Modified, file)
        case 'D':
            changes.Deleted = append(changes.Deleted, file)
        case 'R':
            // Renamed: treat as delete old + add new
            parts := strings.Split(file, "\t")
            if len(parts) == 2 {
                changes.Deleted = append(changes.Deleted, parts[0])
                changes.Added = append(changes.Added, parts[1])
            }
        }
    }
    return changes, nil
}
```

### Step 2: Incremental Extraction Mode

Add `--incremental` flag to Go binary:

```go
// In main.go
incremental := flag.Bool("incremental", false, "Only extract changed files")

if *incremental {
    changes, err := DetectChanges(*manifestPath, *rootPath)
    if err != nil {
        log.Fatalf("detect changes: %v", err)
    }

    if changes.FullRemap {
        log.Println("No manifest found, performing full extraction")
        // Fall through to full extraction
    } else {
        allChanged := append(changes.ChangedFiles, changes.NewFiles...)
        if len(allChanged) == 0 && len(changes.DeletedFiles) == 0 {
            fmt.Fprintln(os.Stderr, "No changes detected since last map")
            os.Exit(0)
        }

        log.Printf("Incremental: %d changed, %d new, %d deleted (method: %s)",
            len(changes.ChangedFiles), len(changes.NewFiles),
            len(changes.DeletedFiles), changes.Method)

        // Extract only changed/new files
        opts.SpecificFiles = allChanged
    }
}
```

### Step 3: Merge Strategy for Extraction Output

When re-extracting a subset of files, the existing module extraction must be updated, not replaced:

```go
// MergeExtraction updates an existing module extraction with new file data
func MergeExtraction(existing, partial *ModuleExtraction) *ModuleExtraction {
    // Build file index from existing
    fileIndex := make(map[string]int) // path → index
    for i, f := range existing.Files {
        fileIndex[f.Path] = i
    }

    // Update/add files from partial extraction
    for _, newFile := range partial.Files {
        if idx, ok := fileIndex[newFile.Path]; ok {
            // Update existing file
            existing.Files[idx] = newFile
        } else {
            // Add new file
            existing.Files = append(existing.Files, newFile)
        }
    }

    // Re-aggregate imports
    existing.Imports = aggregateImports(existing.Files)
    existing.ExtractedAt = partial.ExtractedAt

    return existing
}

// RemoveDeletedFiles removes files that no longer exist
func RemoveDeletedFiles(extraction *ModuleExtraction, deleted []string) {
    deletedSet := make(map[string]bool)
    for _, d := range deleted { deletedSet[d] = true }

    var kept []FileExtract
    for _, f := range extraction.Files {
        if !deletedSet[f.Path] {
            kept = append(kept, f)
        }
    }
    extraction.Files = kept
    extraction.Imports = aggregateImports(kept)
}
```

### Step 5: Fork Skill Integration

The fork skill (CM-009) handles incremental mode when `$ARGUMENTS` contains `--incremental`:

1. Read manifest.json to check if previous map exists
2. Pass `--incremental` to Go binary
3. Only re-enrich modules that had changes
4. Always re-synthesize ARCHITECTURE.md (reads all modules, regenerates)

The re-synthesis step always runs because even small changes can affect module dependency graphs and diagrams.

## Known Limitations

- **Full re-synthesis on every incremental run**: Incremental synthesis always regenerates the full ARCHITECTURE.md by reading ALL enriched modules. This negates the speed benefit of incremental extraction for the synthesis stage. The extraction and enrichment stages are truly incremental; synthesis is not. Partial synthesis (updating only changed sections of ARCHITECTURE.md) is deferred to a future ticket.
