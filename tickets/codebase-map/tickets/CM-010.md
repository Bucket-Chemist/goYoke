# CM-010: Register Agent in agents-index.json

**Phase**: Phase 2: Skill & Agent Integration
**Priority**: MEDIUM
**Blocked By**: CM-009 (fork skill must exist)
**Effort**: 0.5 days

## Description

Register the `/codebase-map` slash command in the CLAUDE.md dispatch table and optionally add a `codebase-mapper` entry to `agents-index.json` for trigger-based routing.

**Execution model note**: The fork skill (CM-009) is the primary execution model. The agents-index.json entry exists solely so the router can recognize natural-language requests like "map the codebase" and route them to the `/codebase-map` skill. It is NOT a separate agent — it's a routing trigger that invokes the fork skill.

### Step 2: Update CLAUDE.md Dispatch Table

Add to the Tier 1.5 or Tier 2 section (Sonnet tier since it runs as Sonnet):

```markdown
### Tier 2: Sonnet (Implementation)

| Trigger Patterns                             | Agent             | subagent_type   |
| -------------------------------------------- | ----------------- | --------------- |
| map codebase, architecture diagram, module map | `codebase-mapper` | general-purpose |
```

### Step 3: Add Slash Command to CLAUDE.md

Add to the Slash Commands table:

```markdown
| `/codebase-map`         | Map codebase structure, generate ARCHITECTURE.md with Mermaid diagrams    |
```

### Step 4: Router Routing Logic

The router should recognize these patterns and invoke the skill:

| User Request | Route |
|-------------|-------|
| "map this codebase" | `/codebase-map` skill |
| "generate architecture docs" | `/codebase-map` skill |
| "update the architecture diagram" | `/codebase-map --incremental` skill |
| "show me the module dependency graph" | `/codebase-map` skill (then read output) |
| "map just the internal/routing module" | `/codebase-map internal/routing` skill |
