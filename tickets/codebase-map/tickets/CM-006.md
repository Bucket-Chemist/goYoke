## Description

Implement `LanguageExtractor` for Python, TypeScript, and R. Each language gets a tree-sitter query file (`.scm`) and an extractor struct that handles language-specific details (export rules, import classification, receiver syntax).

The `LanguageExtractor` interface is defined in CM-005. This ticket adds three implementations.

**v1 scope (CM-006a)**: TypeScript only — primary use case alongside Go for GOgent-Fortress.
**Phase 4 scope (CM-006b/this file)**: Python + R — deferred until v1 ships and validates the pipeline.

func (p *PythonExtractor) ClassifyImport(importPath, projectRoot string) string {
    // Relative imports (starting with .) → internal
    if strings.HasPrefix(importPath, ".") {
        return "internal"
    }
    // Check against embedded stdlib list (NOT "no dot" heuristic — that fails for
    // dateutil, yaml, PIL, cv2 etc.)
    topLevel := strings.Split(importPath, ".")[0]
    if pythonStdlib[topLevel] {
        return "stdlib"
    }
    // Everything else → external
    return "external"
}

**Queries** (`queries/python.scm`):

```scheme
;; Function definitions
(function_definition
  name: (identifier) @func.name
  parameters: (parameters) @func.params
  return_type: (_)? @func.returns
  body: (block) @func.body
) @func.def

;; Class definitions
(class_definition
  name: (identifier) @class.name
  superclasses: (argument_list)? @class.bases
  body: (block) @class.body
) @class.def

;; Method definitions (inside class)
(class_definition
  body: (block
    (function_definition
      name: (identifier) @method.name
      parameters: (parameters) @method.params
      return_type: (_)? @method.returns
    ) @method.def
  )
)

;; Decorated definitions
(decorated_definition
  (decorator
    (identifier) @decorator.name)
  definition: (_) @decorator.target
)

;; Import statements
(import_statement
  name: (dotted_name) @import.name
)
(import_from_statement
  module_name: (dotted_name) @import.module
  name: (dotted_name) @import.name
)

;; Assignment (top-level constants)
(expression_statement
  (assignment
    left: (identifier) @var.name
    right: (_) @var.value
  )
) @var.def
```

**Python-specific handling**:
- `self` parameter → method receiver. Map `self` param type from enclosing class.
- Decorators → populate `decorators` field
- `__init__` → mark as constructor (kind="method", special handling)
- Type hints → extract from `return_type` and parameter annotations
- Module boundary: directory with `__init__.py`, or standalone `.py` file

### TypeScript Extractor

**File**: `cmd/gogent-codebase-extract/typescript.go`

```go
type TypeScriptExtractor struct{}

func (t *TypeScriptExtractor) Language() string       { return "typescript" }
func (t *TypeScriptExtractor) Extensions() []string   { return []string{".ts", ".tsx"} }
```

**Queries** (`queries/typescript.scm`):

```scheme
;; Function declarations
(function_declaration
  name: (identifier) @func.name
  parameters: (formal_parameters) @func.params
  return_type: (type_annotation)? @func.returns
) @func.def

;; Arrow functions assigned to const/let
(lexical_declaration
  (variable_declarator
    name: (identifier) @func.name
    value: (arrow_function
      parameters: (formal_parameters) @func.params
      return_type: (type_annotation)? @func.returns
    ) @func.body
  )
) @func.def

;; Class declarations
(class_declaration
  name: (type_identifier) @class.name
  body: (class_body) @class.body
) @class.def

;; Method definitions
(method_definition
  name: (property_identifier) @method.name
  parameters: (formal_parameters) @method.params
  return_type: (type_annotation)? @method.returns
) @method.def

;; Interface declarations
(interface_declaration
  name: (type_identifier) @iface.name
  body: (interface_body) @iface.body
) @iface.def

;; Type alias declarations
(type_alias_declaration
  name: (type_identifier) @type.name
  value: (_) @type.body
) @type.def

;; Enum declarations
(enum_declaration
  name: (identifier) @enum.name
  body: (enum_body) @enum.body
) @enum.def

;; Import statements
(import_statement
  source: (string) @import.path
)

;; Export detection
(export_statement) @export
```

**TypeScript-specific handling**:
- `export` keyword → `exported: true`
- Arrow functions assigned to `const` → kind="function"
- Interfaces → kind="interface"
- Type aliases → kind="type"
- Enums → kind="enum"
- `.tsx` files: same grammar, JSX nodes ignored for symbol extraction

### R Extractor

**File**: `cmd/gogent-codebase-extract/r.go`

```go
type RExtractor struct{}

func (r *RExtractor) Language() string       { return "r" }
func (r *RExtractor) Extensions() []string   { return []string{".R", ".r"} }
```

**Queries** (`queries/r.scm`):

```scheme
;; Function assignments (R pattern: name <- function(...) { })
(binary_operator
  lhs: (identifier) @func.name
  operator: "<-"
  rhs: (function_definition
    parameters: (parameters) @func.params
    body: (_) @func.body
  )
) @func.def

;; S4 class definitions
(call
  function: (identifier) @_setClass (#eq? @_setClass "setClass")
  arguments: (arguments
    (argument value: (string) @class.name)
  )
) @class.def

;; S4 method definitions
(call
  function: (identifier) @_setMethod (#eq? @_setMethod "setMethod")
  arguments: (arguments
    (argument value: (string) @method.name)
  )
) @method.def

;; R6 class definitions
(binary_operator
  lhs: (identifier) @class.name
  operator: "<-"
  rhs: (call
    function: (namespace_operator
      rhs: (identifier) @_new (#eq? @_new "new"))
  )
) @class.def

;; Library/require imports
(call
  function: (identifier) @_lib (#match? @_lib "^(library|require)$")
  arguments: (arguments
    (argument value: (_) @import.name)
  )
)

;; Source imports
(call
  function: (identifier) @_source (#eq? @_source "source")
  arguments: (arguments
    (argument value: (string) @import.path)
  )
)
```

**R-specific handling**:
- All top-level functions are exported (R has no private functions by convention)
- S4 classes: `setClass("ClassName", ...)` → kind="class"
- R6 classes: `ClassName <- R6::R6Class(...)` → kind="class"
- S4 methods: `setMethod("methodName", ...)` → kind="method"
- `library()` and `require()` → external imports
- `source()` → internal imports
- **Known limitation**: R tree-sitter grammar may not capture all S4/R6 patterns. Document gaps.

## Code Guidelines

- Each extractor is a single `.go` file with embedded `.scm` query
- All extractors implement the same `LanguageExtractor` interface
- Language detection by file extension (no heuristics)
- If a query fails to compile → log warning, skip that query, continue extraction
- Python stdlib detection: use embedded known stdlib list (NOT "no dot in import path" heuristic — that misclassifies `dateutil`, `yaml`, `PIL`, `cv2` as stdlib)
- **Behavior contracts**: Code examples describe WHAT behavior is expected. The implementing agent writes code against the library selected in CM-001.
