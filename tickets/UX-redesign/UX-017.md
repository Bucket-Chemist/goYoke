# UX-017: Status line: adaptive 1-row at narrow widths

## Description

At Standard width (80-119 cols), compress status line to 1 row with only critical fields: cost, context bar, model, agent count, duration. Drop: permission mode, auth email, git branch, provider, CWD (viewable in Settings tab).

IMPLEMENTATION NOTE: statusLineHeight must be a method on AppModel computing from m.width directly — NOT derived from computeLayout to avoid circular dependency.

## Acceptance Criteria

- [ ] 1-row status line at Standard width tier (80-119)
- [ ] 2-row status line at Wide+ width tier (120+)
- [ ] statusLineHeight implemented as AppModel method
- [ ] lipgloss.Height(statusLine.View()) == computed height at all tiers
- [ ] No circular dependency with computeLayout
- [ ] Boundary tests at {79, 80, 119, 120}
