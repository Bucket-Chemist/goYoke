# TUI-057: Plan mode UX improvements

## Description

Improve the plan mode user experience by adding a status line indicator, step tracking, and progress display. Currently there is no visual differentiation when the TUI is in plan mode.

Modifications:

1. **Status line plan mode indicator** (`statusline.go`):
   - Add `PlanActive bool` and `PlanStep int`, `PlanTotalSteps int` fields to `StatusLineModel`
   - When `PlanActive` is true, show `[PLAN MODE: step N/M]` in the status bar, styled with `theme.WarningStyle()`
   - When `PlanTotalSteps` is 0, show `[PLAN MODE]` without step count

2. **Plan step tracking** (`model/app.go` via `cli_event_handlers.go` from TUI-043):
   - Parse step markers from CLI output (e.g., `AssistantEvent` streaming text)
   - Track current step number and total steps
   - Emit `PlanStepMsg` when step changes:
     ```go
     type PlanStepMsg struct {
         Step  int
         Total int
     }
     ```
   - Add `PlanStepMsg` to `messages.go`

3. **Plan mode entry/exit** (existing `rightPanelMode == RPMPlanPreview`):
   - When `PlanActive` becomes true, show a toast notification: "Plan mode active — press alt+v to view plan"
