# TUI-027: Team polling and team list component

## Description

Create team orchestration display in internal/tui/components/teams/. File: state.go -- TeamRegistry struct with teams map[string]*TeamState, mu sync.RWMutex. TeamState maps to the team config.json format from gogent-team-run: Name, Status, Waves[]Wave, Agents[]TeamAgent, CreatedAt, CompletedAt, TotalCost. File: list.go -- TeamListModel shows all teams with status, cost, wave progress. Uses tea.Every(2*time.Second) to poll team directories (from session dir teams/ folder). Reads config.json from each team directory. File: detail.go -- TeamDetailModel shows wave-grouped view of a selected team: each wave shows its agents with status, cost, and output summary. Wave grouping: agents are grouped by their wave number (from blocked_by dependencies). Display: wave N agents, with running/complete/error indicators. Wire into root AppModel: team polling tick reads team directories, updates TeamRegistry, sends TeamUpdateMsg. TeamListModel subscribes to TeamUpdateMsg for updates. Team views are shown when rightPanelMode == RPMTeamConfig (switched via Alt+T tab).
