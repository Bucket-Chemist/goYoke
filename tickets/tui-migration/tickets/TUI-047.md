# TUI-047: Contextual error message formatting

## Description

Create `internal/tui/util/errors.go` with structured error message rendering for the TUI. Currently errors are displayed as plain strings. This ticket adds semantic formatting using the Theme's error/warning colors and icons.

New file: `internal/tui/util/errors.go`
```go
package util

// ErrorDisplay holds a formatted error with category, message, and optional suggestion.
type ErrorDisplay struct {
    Category   string    // e.g. "Connection", "Permission", "Timeout"
    Message    string
    Suggestion string    // optional fix suggestion
    Timestamp  time.Time
}

// FormatError renders an ErrorDisplay using the given theme's error styling.
// Output format: [icon] Category: message\n  Suggestion: suggestion
func FormatError(ed ErrorDisplay, theme config.Theme) string

// FormatWarning renders a warning-level message.
func FormatWarning(category, message string, theme config.Theme) string

// ClassifyError maps common error patterns to categories:
// - "connection refused" -> "Connection"
// - "permission denied" -> "Permission"
// - "timeout"           -> "Timeout"
// - "rate limit"        -> "Rate Limit"
// - default             -> "Error"
func ClassifyError(err error) string
```

The formatting functions use `theme.ErrorStyle()`, `theme.Icons().Error` etc. from TUI-044/045. `FormatError` produces a multi-line string with the icon colored by severity, the category in bold, and the suggestion in muted style.

This is a utility module — no UI integration yet. Integration happens when components adopt these formatters in Phase 10d/e.
