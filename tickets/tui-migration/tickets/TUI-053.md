# TUI-053: Slash command dropdown

## Description

Create a new `slashcmd/` package with a dropdown autocomplete component that appears when the user types `/` in the chat input. The dropdown is a non-modal component that allows continued typing (not an interruption).

Create `internal/tui/components/slashcmd/` package:
```go
package slashcmd

type SlashCommand struct {
    Name        string // e.g. "explore"
    Description string // e.g. "Structured codebase exploration"
}

type SlashCmdModel struct {
    commands []SlashCommand
    filtered []SlashCommand
    selected int
    query    string    // current input after "/"
    visible  bool
    width    int
}
```

The dropdown renders above the input line. When `/` is typed, `SlashCmdModel.Show(query)` is called; as the user continues typing, `Filter(query)` narrows results. Up/Down arrows navigate the list; Enter selects.

**Command list:** Reference `CLAUDE.md` as the source of truth for available slash commands. The current list includes at minimum:
`/explore`, `/braintrust`, `/review`, `/review-plan`, `/ticket`, `/implement`, `/init-auto`, `/benchmark`, `/benchmark-meta`, `/memory-improvement`, `/explore-add`, `/dummies-guide`, `/team-status`, `/team-result`, `/team-cancel`, `/plan-tickets`, `/teams`

Note from review (m-2): the initial spec listed 11 commands but CLAUDE.md has 16+. Reference CLAUDE.md at build time or embed the list in config.

**Rendering:** The dropdown is rendered inside the Claude panel's `View()` method, positioned above the text input line. Use lipgloss border + background to visually separate it from the conversation history.

**Widget interface:** Add `slashCmdWidget` interface to `model/interfaces.go`.
