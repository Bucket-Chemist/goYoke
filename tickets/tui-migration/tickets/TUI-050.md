# TUI-050: Interactive settings panel with tree navigation

## Description

Transform the display-only `SettingsModel` into an interactive tree-based settings panel. Current `settings.go` (156 lines) renders static key-value pairs. This ticket creates a new `settingstree/` component following the proven pattern of `agents/tree.go`.

Create `internal/tui/components/settingstree/` package:
```go
package settingstree

type SettingType int
const (
    SettingToggle  SettingType = iota  // bool on/off
    SettingSelect                       // pick from list
    SettingDisplay                      // read-only display
)

type SettingNode struct {
    Key         string
    Label       string
    Type        SettingType
    Value       string        // current value
    Options     []string      // for SettingSelect
    Description string        // help text shown below
}

type SettingsTreeModel struct {
    sections        []SettingsSection
    selectedSection int
    selectedItem    int
    width, height   int
    focused         bool
}

type SettingsSection struct {
    Title    string
    Nodes    []SettingNode
    Expanded bool
}
```

Sections:
1. **Display:** Theme (Select: Dark/Light/High Contrast), ASCII Icons (Toggle), Vim Keys (Toggle)
2. **Session:** Model (Display), Provider (Display), Permission Mode (Display), Session Dir (Display)
3. **Status:** MCP Servers (Display), Agent Count (Display), Git Branch (Display)

Navigation: Up/Down moves between items, Enter toggles/cycles `Select` values, section headers expand/collapse with Enter.

When a setting changes, emit `SettingChangedMsg` to the parent:
```go
type SettingChangedMsg struct {
    Key   string
    Value string
}
```

Add `settingsTreeWidget` interface to `model/interfaces.go`. Keep old `settings/` package as-is (display-only, used when `settingstree` is not yet active).
