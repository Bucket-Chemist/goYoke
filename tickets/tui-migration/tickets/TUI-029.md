# TUI-029: Provider switching logic + message isolation (P7-7b)

## Description

Implement provider switching logic in internal/tui/state/provider.go and wire into AppModel. When switching providers: (1) save current messages to ProviderState.Messages[currentProvider], (2) save current session ID to ProviderState.SessionIDs[currentProvider], (3) save current model to ProviderState.Models[currentProvider], (4) update Active to new provider, (5) restore messages from ProviderState.Messages[newProvider], (6) restore session ID and model, (7) restart CLI driver with new model and optional adapter env vars. For non-Anthropic providers, the CLI needs different arguments: --model flag with provider-specific model ID, adapter path env vars (e.g., CLAUDE_ADAPTER for Google/OpenAI). Each provider switch triggers a CLIDriver restart: Shutdown old -> Start new with updated opts. Message history isolation: each provider has its own message array. Switching providers shows only that provider's conversation. Wire into root AppModel: handle ProviderSwitchMsg in Update(), call ProviderState.SwitchProvider(), restart CLIDriver, update ClaudePanelModel messages. Provider switch triggered by Shift+Tab keybinding.
