# TUI-045: Icon library with Unicode and ASCII fallback

## Description

Replace ASCII icon constants in `config/theme.go` with a dual-mode icon system that uses Unicode by default and falls back to ASCII on terminals that don't support UTF-8.

Current icons (`config/theme.go` lines 43-61):
- `IconRunning = '>'`, `IconComplete = '*'`, `IconError = '!'`, `IconPending = '.'`, `IconCancelled = 'x'`, `IconPaused = '~'`

New design:
```go
// IconSet holds all icons for a given mode (Unicode or ASCII).
type IconSet struct {
    Running   string
    Complete  string
    Error     string
    Pending   string
    Cancelled string
    Paused    string
    Info      string   // new
    Warning   string   // new
    Success   string   // new
    Search    string   // new
    Settings  string   // new
    Arrow     string   // new (breadcrumb separator)
}

var UnicodeIcons = IconSet{
    Running: "\u25B6", Complete: "\u2713", Error: "\u2717",
    Pending: "\u25CB", Cancelled: "\u2715", Paused: "\u23F8",
    Info: "\u2139", Warning: "\u26A0", Success: "\u2714",
    Search: "\uD83D\uDD0D", Settings: "\u2699", Arrow: "\u203A",
}

var ASCIIIcons = IconSet{
    Running: ">", Complete: "*", Error: "!",
    Pending: ".", Cancelled: "x", Paused: "~",
    Info: "i", Warning: "!", Success: "*",
    Search: "/", Settings: "@", Arrow: ">",
}
```

Add `Icons()` method to `Theme` that returns the active `IconSet`. Add `UseASCII bool` field to `Theme` struct. When `UseASCII` is true, return `ASCIIIcons`; otherwise `UnicodeIcons`.

**Keep the old `const Icon*` values for backward compatibility** but mark them deprecated. Existing consumers (`agents/tree.go` `statusIcon()`, `toast/toast.go` `levelStyle()`, `taskboard/taskboard.go` `taskIcon()`) will be migrated in subsequent tickets.

ASCII fallback detection: the `UseASCII` flag can be set from the TUI settings panel (TUI-050) or auto-detected from `LANG`/`LC_ALL` environment variables.
