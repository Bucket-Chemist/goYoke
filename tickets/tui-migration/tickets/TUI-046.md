# TUI-046: Theme switching infrastructure

## Description

Add runtime theme switching support to the TUI. Currently the theme is set once via `DefaultTheme()`. This ticket adds the ability to switch themes at runtime without restarting.

Add `activeTheme *config.Theme` to `sharedState` in `app.go`:
```go
type sharedState struct {
    // ... existing fields ...
    activeTheme *config.Theme  // active theme variant, nil = DefaultTheme
}
```

Add to `AppModel`:
```go
func (m *AppModel) SetTheme(t config.Theme)
func (m *AppModel) Theme() config.Theme
```

Add `ThemeChangedMsg` to `messages.go`:
```go
type ThemeChangedMsg struct {
    Variant config.ThemeVariant
}
```

In `app.go` `Update()`, handle `ThemeChangedMsg` by calling `m.shared.activeTheme = &newTheme` and propagating to components. The propagation strategy is:
- Store the active `Theme` in `sharedState`
- Components that need theme-awareness get a `SetTheme()` method added to their widget interface
- For Phase 10, only new components use the `Theme` struct directly; existing components continue using package globals until a future migration

Add theme persistence: save the active `ThemeVariant` in `session.SessionData` so it survives restarts. Add `ThemeVariant config.ThemeVariant` field to `session/types.go` `SessionData` struct.

This is infrastructure only — no visual changes yet. The settings panel (TUI-050) will provide the UI to trigger theme switches.

## Review Amendments

**M-5 (Major — must address):** The propagation mechanism must be explicitly specified. The chosen approach is: `activeTheme` stored in `sharedState`; components that access `sharedState` via `HandleMsg` pointer receiver read the theme from shared state. Components without `sharedState` access continue using package-level `config.StyleXxx` defaults. The ~5 component files that need modification for proof of concept are:
- `internal/tui/components/statusline/statusline.go` (receives theme via `SetTheme()`)
- `internal/tui/components/settingstree/settingstree.go` (new component, theme-aware from start)
- `internal/tui/components/slashcmd/slashcmd.go` (new component, theme-aware from start)
- `internal/tui/components/search/search.go` (new component, theme-aware from start)
- `internal/tui/model/app.go` (propagates `ThemeChangedMsg` to the above)
