# TUI-043: Extract key handlers from app.go

## Description

Extract keyboard handling logic from `app.go` into dedicated files in `internal/tui/model/`. Current `app.go` is 995 lines; this extraction keeps it under 400 lines as pure dispatch.

Create three new files:

1. **`key_handlers.go`**: Move `handleKey()`, `handleModalKey()`, `handleClaudeKey()`, `handleAgentsKey()`, `syncFocusState()` from `app.go` (lines 885-994).
2. **`cli_event_handlers.go`**: Move all `cli.*` event case handlers from `Update()` switch: `CLIStartedMsg`, `SystemInitEvent`, `AssistantEvent`, `UserEvent`, `ResultEvent`, `CLIDisconnectedMsg`, `CLIReconnectMsg`. Also move `extractDiffs()`, `toolUseResultWithPatch`, `waitForCLIEvent()`.
3. **`ui_event_handlers.go`**: Move `ProviderSwitchMsg`, `ProviderSwitchExecuteMsg`, `BridgeModalRequestMsg`, `ModalResponseMsg`, Agent/Team events, `ToastMsg`, `ShutdownCompleteMsg`, `SessionAutoSaveMsg`, `SystemHookEvent`/`StatusEvent`/`RateLimitEvent`/`StreamEvent`/`CLIUnknownEvent` handlers.

All files stay in `package model`, use `(m AppModel)` value receiver pattern. The `Update()` method in `app.go` becomes a pure dispatcher that calls `m.handleKeyMsg(msg)`, `m.handleCLIEvent(msg)`, `m.handleUIEvent(msg)` etc.

**Pattern to follow:** `provider_switch.go` already demonstrates this extraction pattern — it is in `package model` with `(m AppModel)` receiver.

This is a pure refactor with zero behavioral changes. The goal is to bring `app.go` under 450 lines and establish the handler file pattern for all Phase 10 additions.
