# TUI-015: TUI-side UDS listener and IPC bridge

## Description

Implement the TUI side of the Unix domain socket IPC bridge in internal/tui/bridge/server.go. The IPCBridge struct: socketPath string, listener net.Listener, program *tea.Program, pendingModals map[string]chan ModalResponse, mu sync.Mutex. On Init(), create the UDS at $XDG_RUNTIME_DIR/gofortress-{pid}.sock (following existing pattern from lifecycle/process.go). Start a goroutine that accepts connections and reads JSON requests. For each request: (1) Parse IPCRequest, (2) Based on Type, create appropriate tea.Msg and send via program.Send(), (3) For modal_request: create a response channel, store in pendingModals[request.ID], block goroutine on channel, write response back to UDS when received. The ModalResponseMsg handler in the root AppModel should call bridge.ResolveModal(id, response) which writes to the pending channel. For agent_register/agent_update/agent_activity: send the corresponding tea.Msg via program.Send() (fire-and-forget, no response needed). For toast: send ToastMsg. Include Shutdown() that closes the listener, removes the socket file, and closes all pending modal channels with a cancellation response. Export the socket path via SocketPath() for passing to child processes as GOFORTRESS_SOCKET env var. File: internal/tui/bridge/server.go for the UDS server. File: internal/tui/bridge/types.go for IPCRequest/IPCResponse types (shared with mcp/protocol.go or imported from a common package).
