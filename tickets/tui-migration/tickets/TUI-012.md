# TUI-012: NDJSON event type definitions

## Description

Create comprehensive Go struct definitions for all NDJSON event types in internal/tui/cli/events.go. Use the spike results (task-003) as the source of truth for field names and types. Define: (1) CLIEvent base struct with Type string and Subtype string for discriminated union parsing. (2) SystemInitEvent with fields: SessionID, CWD, Tools[]ToolInfo, MCPServers[]string, Model, PermissionMode, APIKeySource, ClaudeCodeVersion, SlashCommands[]string, Agents[]string, UUID. (3) SystemStatusEvent with Status *string, PermissionMode string. (4) CompactBoundaryEvent with CompactMetadata{Trigger, PreTokens, PostTokens}. (5) AssistantEvent with Message{ID, Role, Content[]ContentBlock, Usage *MessageUsage, Model, StopReason}, ParentToolUseID *string. (6) UserEvent with Message{Role, Content[]ContentBlock}. (7) ResultEvent with Subtype, TotalCostUSD, DurationMS, DurationAPIMS, NumTurns, SessionID, Usage{InputTokens, OutputTokens, CacheCreationInputTokens, CacheReadInputTokens}, ModelUsage map, PermissionDenials[]Denial. (8) StreamEvent wrapping Anthropic streaming types: MessageStart, ContentBlockStart, ContentBlockDelta (text_delta, input_json_delta, thinking_delta, signature_delta), ContentBlockStop, MessageDelta, MessageStop. (9) RateLimitEvent. (10) ToolUseSummaryEvent. (11) ContentBlock interface with concrete types: TextBlock, ToolUseBlock{ID, Name, Input json.RawMessage}, ToolResultBlock{ToolUseID, Content, IsError}. (12) MessageUsage struct. Use *string and *int for optional fields. Include json tags on all fields. Add a ParseCLIEvent(line []byte) (tea.Msg, error) function that does two-pass parsing: first unmarshal the discriminator, then unmarshal the full type. Unknown types return CLIUnknownMsg with raw JSON preserved.
