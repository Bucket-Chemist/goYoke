# TUI-013: CLI subprocess driver

## Description

Create the CLI subprocess driver in internal/tui/cli/driver.go. The CLIDriver struct manages the Claude Code CLI subprocess lifecycle. Fields: cmd *exec.Cmd, stdin io.WriteCloser, stdout io.ReadCloser, eventCh chan tea.Msg, state DriverState (enum: Idle, Streaming, Error, Dead), sessionID string, mu sync.Mutex. Constructor NewCLIDriver(opts CLIDriverOpts) where CLIDriverOpts has: SessionID, Model, MCPConfigPath, ProjectDir, Verbose, PermissionMode, PermissionPromptTool string. The Start(ctx context.Context) tea.Cmd method: (1) builds exec.Command with args [claude, --output-format, stream-json, --verbose, --include-partial-messages, --permission-prompt-tool, stdio, --mcp-config, <path>], (2) optionally adds --resume <id>, --model <model>, (3) sets up stdin/stdout pipes, (4) starts the process, (5) launches consumeEvents goroutine. The consumeEvents goroutine reads stdout line by line via bufio.Scanner, calls ParseCLIEvent for each line, and sends the resulting tea.Msg on eventCh. On scanner error or EOF, send CLIDisconnectedMsg. The SendMessage(text string) tea.Cmd writes user text to stdin (format determined by spike). The Interrupt() method sends SIGINT to the child process. The Shutdown() method sends SIGTERM, waits 2s, then SIGKILL. Include WaitForEvent() tea.Cmd using the channel-to-Cmd re-subscription pattern: func(ch <-chan tea.Msg) tea.Cmd { return func() tea.Msg { return <-ch } }. The root AppModel's Update() must call WaitForEvent after processing each CLI event to maintain the subscription.
