# TUI-058: 4-tier responsive layout

## Description

Extend the layout system to support 4 distinct layout tiers based on terminal width. Currently the layout uses a basic 70/30 split. This ticket adds proper breakpoints and per-tier layout adjustments.

Add `LayoutTier` enum to `model/layout.go`:
```go
type LayoutTier int
const (
    LayoutCompact  LayoutTier = iota // < 80 cols
    LayoutStandard                   // 80-119 cols
    LayoutWide                       // 120-179 cols
    LayoutUltra                      // >= 180 cols
)
```

Modify `computeLayout()` in `model/layout.go` to return a `LayoutTier` based on width:
- Compact (< 80): single-column, status line abbreviated, right panel hidden or overlaid
- Standard (80-119): current 70/30 split (baseline behavior)
- Wide (120-179): 60/40 split, right panel shows more detail
- Ultra (>= 180): 50/50 split, both panels fully expanded

Add `SetTier(tier LayoutTier)` to all relevant widget interfaces. Components can branch on tier to abbreviate or expand their rendering.

**CRITICAL:** Write table-driven tests for ALL 8 boundary widths BEFORE modifying `computeLayout()`:
- Test widths: 79, 80, 99, 119, 120, 149, 179, 180
