# MCP-001: Standalone MCP server binary scaffold with ping tool

## Description

Create `cmd/gofortress-mcp-standalone/` as a new binary that exposes GOgent-Fortress agent-spawning capabilities over the Model Context Protocol using stdio transport. Unlike `gofortress-mcp` (which requires a running TUI and `GOFORTRESS_SOCKET`), this binary is fully standalone and works in headless/CI contexts.

**Key deliverables:**
- `cmd/gofortress-mcp-standalone/main.go` — entry point wiring MCP SDK stdio transport
- `cmd/gofortress-mcp-standalone/tools.go` — `test_mcp_ping` and `spawn_agent` tool registrations
- `cmd/gofortress-mcp-standalone/tools_test.go` — unit tests for both tools
- `RegisterAll(server)` function for clean tool registration

**MCP server identity:** `gofortress-mcp-standalone` v1.0.0

**Ping tool contract:**
- Input: `{ echo?: string }`
- Output: `{ status: "PONG", timestamp: string, echo?: string }`
