# TUI-021: Agent sync from NDJSON events

## Description

Implement agent state extraction from NDJSON events in internal/tui/cli/agent_sync.go. This translates the logic from useAgentSync.ts. When AssistantMsg arrives: scan Content[]ContentBlock for ToolUseBlock entries. If the tool name is 'Task' or matches an agent spawn pattern, extract: tool_use_id as agent ID, parse Input for agent description/model/type. Register the agent via AgentRegistry.Register(). When UserEvent (tool results) arrives: scan for ToolResultBlock. Match tool_use_id to registered agents. If the result is final (not partial), mark agent as Complete or Error based on is_error field. Extract activity from assistant messages: implement extractToolTarget() that parses tool_use Input to determine what file/resource is being accessed (e.g., Read -> file path, Bash -> command preview, Grep -> pattern). Update agent activity via AgentRegistry.SetActivity(). Create helper function activityFromToolUse(block ToolUseBlock) AgentActivity. Wire into root AppModel.Update(): after processing AssistantMsg and ToolResultMsg, call the sync functions and return updated model with fresh tree cache. Dedup: the same agent may be registered via NDJSON (tool_use detection) and via MCP (spawn_agent IPC). The AgentRegistry.Register() dedup on agentType+description handles this.
