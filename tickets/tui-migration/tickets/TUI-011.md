# TUI-011: CLI entry point (cmd/gofortress)

## Description

Create the main entry point binary at cmd/gofortress/main.go using Cobra for CLI argument parsing. Flags: --session-id string (resume a specific session), --verbose bool (enable verbose logging), --debug bool (write all tea.Msg to debug log file), --model string (initial model override), --permission-mode string (initial permission mode: default, acceptEdits, plan). The main function should: (1) parse flags, (2) call lifecycle.CleanupStaleSockets(), (3) create the root AppModel with parsed options, (4) create tea.NewProgram with tea.WithAltScreen() and tea.WithMouseCellMotion(), (5) store the program reference on the AppModel for Send() from goroutines, (6) run p.Run() and handle error on exit. Add a --version flag that prints the version (injected via ldflags). Add a --mcp-server flag that, when present, runs the MCP server mode instead of the TUI (separate ticket implements the server, this just adds the flag dispatch). Also create cmd/gofortress-mcp/main.go as a stub that prints 'MCP server not yet implemented' -- this will be the MCP server entry point built in Phase 3.

## Review Amendments

**⚠️ Review M-1:** Cobra is NOT in go.mod. Either add `github.com/spf13/cobra` to go.mod OR change to stdlib `flag` package (recommended for 3-4 flags). The go.md conventions say "Start Simple, Add Complexity Only When Needed" — with only 5-6 flags, the stdlib `flag` package is simpler and eliminates an external dependency. Decision: if Cobra is added, run `go get github.com/spf13/cobra` first. If stdlib flag is used, update this ticket's description accordingly and drop the Cobra-specific patterns.
