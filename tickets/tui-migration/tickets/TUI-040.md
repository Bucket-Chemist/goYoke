# TUI-040: Performance benchmarks

## Description

Create performance benchmarks in internal/tui/bench_test.go. (1) BenchmarkStartup: measure time from program creation to ready state (SystemInitMsg processed). Target: under 200ms (Go binary startup, no Node.js). (2) BenchmarkModalRoundTrip: measure time from ModalRequestMsg to ModalResponseMsg. Target: under 100ms (in-process channel, should be <1ms). (3) BenchmarkNDJSONParsing: parse 1000 NDJSON lines from a captured session. Measure throughput. Target: sustain 10,000 lines/sec (streaming events arrive at ~100/sec). (4) BenchmarkViewRendering: render View() at 120x40 with 100 messages, 10 agents, 3 teams. Target: under 16ms (60fps). (5) BenchmarkUDSRoundTrip: measure IPC round-trip between TUI bridge and MCP server. Target: under 5ms. Use testing.B benchmarks. Create test fixtures for realistic data volumes. Report results in benchmark output format compatible with benchstat.
