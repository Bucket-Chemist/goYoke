# TUI-017: Modal model types and queue

## Description

Create the modal system in internal/tui/components/modals/. Files: types.go (ModalType enum, ModalRequest, ModalResponse structs), model.go (ModalModel with Init/Update/View), queue.go (modal queue management). ModalType enum: Ask, Confirm, Input, Select, Permission. ModalRequest struct: ID string, Type ModalType, Message string, Header string, Options []string, TimeoutMS int, ResponseCh chan ModalResponse (for blocking callers). ModalResponse struct: Type ModalType, Value string, Cancelled bool. ModalModel: manages the currently active modal overlay. Fields: request ModalRequest, selectedIdx int, inputText string, inputMode bool (for 'Other' free-text). Update() handles Up/Down for option navigation, Enter for selection, Escape for cancel, and character input when in inputMode. For Ask modals: user selects from options list, or starts typing to enter free-text 'Other' mode. For Confirm modals: two options (Yes/No or Allow/Deny). For Input modals: textinput.Model for free text. For Select modals: list of options. View() renders the modal as a centered overlay using lipgloss.Place() with a double-border box. Queue management in the root AppModel: when a ModalRequestMsg arrives, append to modalQueue. If no modal is active, pop the first and create ModalModel. When ModalResponseMsg arrives, resolve the pending request (send response on ResponseCh), pop next from queue if any.
