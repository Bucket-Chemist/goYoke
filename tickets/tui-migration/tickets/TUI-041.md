# TUI-041: Unknown event resilience test

## Description

Create resilience tests in internal/tui/cli/resilience_test.go. Test the log-and-continue pattern for unknown NDJSON events. (1) Inject fabricated NDJSON events with unknown types: {"type": "future_event", "data": {"key": "value"}}. Verify ParseCLIEvent returns CLIUnknownMsg with raw JSON preserved. Verify the parser continues processing subsequent valid events. (2) Inject malformed JSON lines. Verify parser logs error and continues. (3) Inject events with known type but extra/missing fields. Verify parsing succeeds (optional pointer fields handle missing). (4) Inject events with known type but unknown subtype. Verify subtype falls through to default case. (5) Stress test: alternate valid and invalid events at high rate (1000/sec). Verify no events are dropped and no goroutine leaks. (6) Test rate_limit_event handling: verify it's logged as informational (no state change required). Wire CLIUnknownMsg handling into root AppModel: log at WARN level, accumulate in telemetry for discovery.

- [ ] Unknown event types produce CLIUnknownMsg with raw JSON

- [ ] Extra fields on known types handled gracefully
- [ ] Unknown subtypes fall through to default
- [ ] Stress test: no dropped events, no goroutine leaks
- [ ] rate_limit_event handled as informational
