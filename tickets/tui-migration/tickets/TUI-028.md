# TUI-028: Multi-provider config types (P7-7a)

## Description

Create provider configuration types in internal/tui/state/provider.go. ProviderID type: string with constants Anthropic, Google, OpenAI, Local. ProviderConfig struct per provider: ID ProviderID, Name string, Models []ModelConfig, AdapterPath string (for non-Anthropic providers), EnvVars map[string]string (e.g., OPENAI_API_KEY). ModelConfig struct: ID string, Name string, ContextWindow int. Define all 4 providers with their models: Anthropic (claude-sonnet-4-20250514, claude-opus-4-20250514, claude-haiku-3-20250307), Google (gemini-2.5-pro, gemini-2.5-flash), OpenAI (gpt-4.1, gpt-4.1-mini, o3), Local/Ollama (llama-4-maverick, qwen3-32b, deepseek-r1). ProviderState struct: Active ProviderID, Configs map[ProviderID]ProviderConfig, Messages map[ProviderID][]DisplayMessage (per-provider message history), SessionIDs map[ProviderID]string (per-provider session IDs), Models map[ProviderID]string (per-provider active model), ProjectDirs map[ProviderID]string. Methods: GetActiveConfig() ProviderConfig, GetActiveModel() string, GetActiveMessages() []DisplayMessage, SwitchProvider(id ProviderID) -- switches active without losing state.
