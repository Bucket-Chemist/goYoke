# TUI-019: AgentRegistry with mutex and canonical IDs

## Description

Create the agent registry in internal/tui/state/agent.go. AgentRegistry struct: agents map[string]*Agent, rootAgentID string, selectedID string, mu sync.RWMutex. Agent struct: ID string, ParentID string, AgentType string, Description string, Model string, Tier string, Status AgentStatus, Activity *AgentActivity, StartedAt time.Time, Duration time.Duration, Cost float64, Tokens int, ErrorOutput string, Children []string. AgentStatus type: StatusPending, StatusRunning, StatusComplete, StatusError, StatusKilled. AgentActivity struct: Type string, Target string, Preview string, Timestamp time.Time. Canonical ID strategy: Task-spawned agents use tool_use_id from NDJSON (matches current TS behavior). MCP-spawned agents use UUID from spawn_agent. Dedup key: agentType + description -- if an agent with the same key already exists and is pending/running, skip registration. Methods: Register(agent Agent) error (applies dedup), Update(id string, fn func(*Agent)) error, SetActivity(id string, activity AgentActivity), Remove(id string), Get(id string) *Agent, GetChildren(parentID string) []*Agent, Tree() []*AgentTreeNode (computed tree projection for View), Count() AgentStats. All methods acquire appropriate locks (RLock for reads, Lock for writes). The Tree() method returns a flat list of AgentTreeNode{Agent, Depth, IsLast} for rendering. None of these methods should be called from View() -- call Tree() in Update() when agent state changes and cache the result.

## Review Amendments

**⚠️ Review M-3:** Register() must NOT directly modify treeCache. Instead: Register() modifies agents map under Lock, then calls program.Send(AgentRegisteredMsg{}). The Update() handler for AgentRegisteredMsg recomputes treeCache. This maintains the single-threaded Update/View invariant. The specs say "All state mutations flow through Update()" — this must be enforced for tree cache recomputation.

The plan is internally contradictory on this point: it correctly states "All state mutations flow through Update()" in Race Prevention Rules, but the AgentRegistry Register() method definition implies direct mutation. Resolve in favor of the tea.Msg approach: Register() modifies agents map (which is mutex-protected), but treeCache is ONLY recomputed in Update() after receiving AgentRegisteredMsg. This prevents the data race between the IPC bridge goroutine (which calls Register()) and View() (which reads treeCache on the main goroutine).
