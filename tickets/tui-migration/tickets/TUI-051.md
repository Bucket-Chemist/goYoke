# TUI-051: High-contrast mode

## Description

Implement WCAG AA high-contrast mode using the theme infrastructure from TUI-046. Add contrast validation utilities to ensure all high-contrast color pairs meet the 4.5:1 minimum ratio.

Add `ContrastRatio()` and `ValidateContrast()` to `config/theme.go`:
```go
// ContrastRatio computes the WCAG contrast ratio between two hex colors.
// Returns a float64 in range [1.0, 21.0]. WCAG AA requires >= 4.5 for normal text.
func ContrastRatio(fg, bg string) float64

// ValidateContrast checks all semantic color pairs in the theme against WCAG AA.
// Returns a map of field name -> ratio, and an error if any pair fails.
func (t Theme) ValidateContrast() (map[string]float64, error)
```

The `ThemeHighContrast` variant from TUI-044 is verified to pass `ValidateContrast()`. All hex color pairs are tested.

**Note on AdaptiveColor:** `ContrastRatio` must specify which variant (Light or Dark) to check. The implementation checks the Dark variant colors by default (the most common terminal background). Document this assumption in the function comment.

Wire the high-contrast mode activation:
1. User selects "High Contrast" in the settings panel (TUI-050)
2. `SettingChangedMsg{Key: "theme", Value: "high-contrast"}` is emitted
3. `AppModel.Update()` handles it by calling `m.SetTheme(config.NewTheme(config.ThemeHighContrast))`
4. `ThemeChangedMsg` propagates to theme-aware components
