# TUI-044: Semantic color system

## Description

Extend `internal/tui/config/theme.go` to add a semantic color mapping API. The current `Theme` struct has 7 colors (`Primary`, `Secondary`, `Accent`, `Success`, `Warning`, `Error`, `Muted`) and 10 styles. Add method-based semantic accessors.

Add to `Theme` struct:
- `InfoColor lipgloss.AdaptiveColor` (alias for `ColorPrimary`/Cyan)
- New method: `(t Theme) ErrorStyle() lipgloss.Style` — returns bold + `ColorError` foreground
- New method: `(t Theme) WarningStyle() lipgloss.Style` — returns `ColorWarning` foreground
- New method: `(t Theme) SuccessStyle() lipgloss.Style` — returns `ColorSuccess` foreground
- New method: `(t Theme) InfoStyle() lipgloss.Style` — returns `ColorPrimary` foreground
- New method: `(t Theme) DangerStyle() lipgloss.Style` — returns bold + `ColorError` foreground + underline (for destructive actions)

Add `ThemeVariant` type:
```go
type ThemeVariant int
const (
    ThemeDark ThemeVariant = iota
    ThemeLight
    ThemeHighContrast
)
```

Add `NewTheme(variant ThemeVariant) Theme` factory:
- `ThemeDark`: current `DefaultTheme()` colors (ANSI 0-15)
- `ThemeLight`: lighter foregrounds on light backgrounds
- `ThemeHighContrast`: WCAG AA compliant colors (4.5:1 ratio minimum). Use hex codes: `error=#FF0000`, `success=#00AA00`, `warning=#FFAA00`, `info=#0088FF`

**Keep `DefaultTheme()` unchanged for backward compatibility.** All existing code continues to work.

Existing pattern: `config/theme.go` lines 127-172 show the `Theme` struct and `DefaultTheme()`.
