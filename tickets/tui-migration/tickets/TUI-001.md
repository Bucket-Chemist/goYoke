# TUI-001: Spike - Permission wire format capture

## Description

Run Claude Code CLI with `--permission-prompt-tool stdio --output-format stream-json` and capture the exact wire format of permission control messages. Steps: (1) Create a test directory with a file that requires Write permission. (2) Run `claude --output-format stream-json --permission-prompt-tool stdio --verbose 2>stderr.log | tee ndjson.log` and ask Claude to modify the file. (3) Capture the `control_request` message format from stdout -- document the exact JSON structure including all fields. (4) Document the stdin response format expected by the CLI. (5) Determine whether control_request messages are interleaved with NDJSON on the same stdout pipe or use a separate prefix/envelope. (6) Test whether the response goes to the CLI's stdin pipe (which Go TUI would control) or terminal stdin (which Bubble Tea would own). (7) Write findings to a markdown document at the spike output path. Output file: `tickets/tui-migration/spike-results/permission-protocol.md`. Include raw JSON examples of every control_request subtype observed (can_use_tool at minimum, also check for EnterPlanMode, ExitPlanMode, AskUserQuestion).
