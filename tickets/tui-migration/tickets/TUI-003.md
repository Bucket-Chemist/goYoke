# TUI-003: Spike - NDJSON event catalog capture

## Description

Run a comprehensive Claude CLI session with `--output-format stream-json --include-partial-messages` and capture the complete NDJSON event catalog. Steps: (1) Run `claude --output-format stream-json --include-partial-messages --verbose 2>stderr.log | tee ndjson-capture.log`. (2) Conduct a session that exercises: multi-turn conversation, tool use (Read, Write, Grep, Bash), context compaction (send enough to trigger), model information. (3) Parse every line of ndjson-capture.log and categorize by type and subtype. (4) For each event type, extract a representative JSON example with all fields. (5) Document the complete catalog in tickets/tui-migration/spike-results/ndjson-catalog.md. (6) Compare against the v2.0 catalog from braintrust-handoff-v2.md Section 4.3 -- flag any new or missing types. (7) Verify the stdin format for sending user messages (plain text vs JSON). (8) Document the system.init message structure with ALL fields. (9) Document the result message structure with ALL fields including total_cost_usd, modelUsage, permission_denials. (10) Document stream_event subtypes (message_start, content_block_start, content_block_delta variants, etc.).
