# MCP-003: Relationship validation (spawned_by / can_spawn bidirectional check)

## Description

Implement agent relationship validation that mirrors the TypeScript `gofortress-interactive` server. Validates that a spawning agent has permission to spawn the target agent, using bidirectional checks against `agents-index.json`.

**Key deliverables:**
- `cmd/gofortress-mcp-standalone/validator.go` — `validateRelationship` function
- `cmd/gofortress-mcp-standalone/validator_test.go` — unit tests
- `pkg/routing/agents.go` — add `ContextRequirements` field to `Agent` struct

**Validation logic:**
1. `parentType` from `GOGENT_PARENT_AGENT` env var (verified — set by prior spawn)
2. `callerType` from MCP input `caller_type` field (unverified — self-reported)
3. Bidirectional check when both present: target's `spawned_by` includes parent AND parent's `can_spawn` includes target
4. Router spawns (no parent): check target's `spawned_by` includes "router"
5. Returns `ValidationResult{Valid, Warnings, Errors}`

**`ContextRequirements` addition:** New field on `Agent` struct for future context injection hints (conventions, tools, etc.), loaded from agents-index.json.
