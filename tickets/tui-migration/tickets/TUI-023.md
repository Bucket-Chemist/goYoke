# TUI-023: Markdown rendering via Glamour

## Description

Implement markdown rendering in internal/tui/util/markdown.go using charmbracelet/glamour. Add glamour dependency to go.mod: github.com/charmbracelet/glamour. Create RenderMarkdown(content string, width int) (string, error) function that uses glamour.NewTermRenderer with DarkStyle (auto-detect dark/light from terminal). Configure Glamour with WordWrap(width) to respect panel width. Handle code blocks with syntax highlighting (Glamour does this natively). Handle edge cases: empty content, very long lines, nested code blocks. Create a cached renderer that reuses the glamour.TermRenderer instance (expensive to create). Wire into ClaudePanelModel: when rendering assistant messages, pass through RenderMarkdown. For streaming content, render markdown on each complete block boundary (not per-token -- Glamour is too slow for per-token rendering). Detect block boundaries by looking for double newlines or code fence markers. Add internal/tui/util/syntax.go if additional syntax highlighting beyond Glamour is needed (likely not, as Glamour uses chroma internally).
