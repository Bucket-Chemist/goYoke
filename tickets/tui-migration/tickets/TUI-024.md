# TUI-024: Cost tracker ported from TS

## Description

Port the cost tracking logic from TS (cost/tracker.ts and getSessionCostTracker) to Go in internal/tui/state/cost.go. CostTracker struct: SessionCost float64, PerAgentCosts map[string]float64, BudgetUSD *float64 (nil = no budget), OverBudget bool, mu sync.RWMutex. Methods: UpdateFromResult(result ResultEvent) -- extracts total_cost_usd from result message, updates SessionCost. UpdateAgentCost(agentID string, cost float64) -- increments per-agent cost. CheckBudget() returns (remaining float64, overBudget bool). FormatCost(cost float64) string -- formats as '$X.XX'. GetSessionCost() float64. GetAgentCosts() map[string]float64. The session cost comes from the result event's total_cost_usd field -- this is the cumulative session cost, not per-message. Per-agent costs come from agent completion data (the result event within agent context includes the agent's cost). Budget enforcement: if BudgetUSD is set and SessionCost exceeds it, set OverBudget = true. The TUI should show a warning in the status line but NOT block execution (budget blocking is CLI-side). Wire into root AppModel: on ResultMsg, call costTracker.UpdateFromResult(). Update status line cost display. On agent completion, update per-agent cost.
