# TUI-048: Status line semantic colors and context-aware display

## Description

Upgrade the status line to use semantic colors from the theme system. Currently `statusline.go` uses `config.StyleMuted` and `config.StyleStatusBar` for all fields. This ticket adds color coding based on values.

Modify `internal/tui/components/statusline/statusline.go` `View()`:

1. **Cost field:** Green when < $0.10, Yellow when < $1.00, Red when >= $1.00
2. **Context percent:** Green when < 50%, Yellow when < 80%, Red when >= 80%
3. **Permission mode:** Green for `default`, Yellow for `plan`, Red for `allow-all`
4. **Streaming indicator:** Use `theme.InfoStyle()` for the spinner
5. **Auth status:** Green for authenticated, Red for `N/A`

Add a `SetTheme(t config.Theme)` method to `StatusLineModel`. Use the semantic color accessors from TUI-044.

Also add new fields:
- `UncommittedCount int` ā€” displayed as `uncommitted:N` in row 2 when > 0, styled with warning color. Populated from a periodic `git status --porcelain | wc -l` command following the `gitBranchCmd()` pattern.
- `AgentCount int` ā€” displayed as `agents:N` in row 2, styled normally.

New background command: `uncommittedCountCmd()` follows the same pattern as `gitBranchCmd()` at line 332.

Existing patterns to follow: `statusline.go` lines 172-253 for `View()` rendering, lines 332-360 for background commands.
