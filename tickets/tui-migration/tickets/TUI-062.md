# TUI-062: Vim-style keybindings (optional)

## Description

Add optional vim-style keybindings as an overlay on top of the existing standard keybindings. Vim keys are off by default and enabled via the settings panel (TUI-050). Standard keys always work regardless of vim mode.

Create `internal/tui/config/vim_keys.go`:
```go
package config

// VimKeys extends KeyMap with vim-style bindings.
// When enabled, these keys are active in addition to standard bindings.
// Standard bindings remain active at all times.
type VimKeys struct {
    // Normal mode navigation
    Up     key.Binding // k
    Down   key.Binding // j
    Left   key.Binding // h
    Right  key.Binding // l
    Top    key.Binding // gg
    Bottom key.Binding // G
    // Insert mode toggle
    Insert key.Binding // i (enter insert mode)
    Escape key.Binding // Esc (return to normal mode)
}

type VimMode int
const (
    VimNormal VimMode = iota
    VimInsert
)

// DefaultVimKeys returns the standard vim binding set.
func DefaultVimKeys() VimKeys
```

Add `VimEnabled bool` field to `KeyMap` struct in `config/keys.go`. When true, vim keys are overlaid on the standard keybindings.

In `key_handlers.go` (from TUI-043): when `VimEnabled` is true and the current mode is `VimNormal`, handle `j`/`k` for up/down navigation, `h`/`l` for panel switching.

**Mode transitions:** `i` in normal mode → insert mode (enables text input). `Esc` in insert mode → normal mode.

Wire the `VimEnabled` toggle from `SettingChangedMsg{Key: "vim_keys", Value: "true/false"}`.
