# TUI-064: Spring animation framework (harmonica)

## Description

Integrate `charmbracelet/harmonica` as the animation framework for smooth spring physics-based transitions. This is infrastructure — no animated components yet (those come in TUI-065, TUI-061).

Create `internal/tui/util/animate.go`:
```go
package util

// SpringAnimation wraps harmonica.Spring for use in Bubbletea.
// It maintains a value that springs toward a target.
type SpringAnimation struct {
    spring   harmonica.Spring
    value    float64
    velocity float64
    target   float64
    settled  bool
}

// NewSpring creates a spring with given stiffness and damping.
func NewSpring(stiffness, damping float64) SpringAnimation

// SetTarget sets the target value the spring will animate toward.
func (s *SpringAnimation) SetTarget(target float64)

// Tick advances the spring animation by one frame (16ms at 60fps).
// Returns the current value and whether the spring has settled.
func (s *SpringAnimation) Tick() (value float64, settled bool)

// AnimateTickMsg triggers a new animation frame.
type AnimateTickMsg struct{}

// AnimateTickCmd returns a tea.Cmd that sends AnimateTickMsg after ~16ms.
func AnimateTickCmd() tea.Cmd
```

Add `AnimateTickMsg` to `messages.go`.

The spring is considered "settled" when both value and velocity are within 0.1 of the target. Animated components stop requesting ticks once settled (preventing infinite tick loops).

**Dependency:** `github.com/charmbracelet/harmonica v0.2.0` must be added to `go.mod`. Verify compatibility with Bubbletea v1.3.10 before implementation. If `v0.2.0` is not compatible, use manual easing functions in `util/animate.go` with the same interface.
