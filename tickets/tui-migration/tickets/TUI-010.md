# TUI-010: Root layout compositor (70/30 split + responsive)

## Description

Implement the root View() method in internal/tui/model/app.go to compose the full multi-panel layout using Lipgloss. Layout structure (top to bottom): Banner (3 rows, full width) -> TabBar (1 row, full width) -> Main content area (left panel 70% + right panel 30%) -> TaskBoard (collapsible, 0 or 8 rows, full width) -> StatusLine (2 rows, full width). Responsive breakpoints: width < 80 -> hide right panel entirely (left panel 100%), width < 100 -> left 75% right 25%, width >= 100 -> left 70% right 30%. Use lipgloss.JoinVertical for the main column, lipgloss.JoinHorizontal for the left/right split. Calculate content heights by subtracting fixed elements from total height. Account for border sizes using lipgloss frame size calculations. The left panel renders ClaudePanelModel.View(). The right panel renders different content based on rightPanelMode: RPMAgents -> AgentTreeModel + AgentDetailModel (60/40 split vertically), RPMDashboard -> DashboardModel, RPMSettings -> SettingsModel, RPMTelemetry -> TelemetryModel. Wire up the layout into the Update() method to propagate WindowSizeMsg to all children with their computed widths and heights.

## Acceptance Criteria

- [ ] Layout renders correctly at 120x40 terminal size
- [ ] Responsive breakpoint at width < 80 hides right panel

- [ ] No off-by-one border calculation errors
