# TUI-014: Go MCP server with 7 tools

## Description

Implement the Go MCP server binary in cmd/gofortress-mcp/main.go using github.com/modelcontextprotocol/go-sdk with mcp.StdioTransport{}. The server registers 7 tools: askUser, confirmAction, requestInput, selectOption, spawnAgent, teamRun, testMcpPing. Tool implementations are in internal/tui/mcp/tools.go. The MCP server communicates with the TUI process via Unix domain socket (side channel) for interactive tools. On startup, the MCP server reads the GOFORTRESS_SOCKET environment variable to find the TUI's Unix domain socket path. It connects to this socket and uses it for modal request/response IPC. For interactive tools (askUser, confirmAction, requestInput, selectOption): the handler sends a JSON request over the UDS to the TUI, blocks waiting for the JSON response, and returns the result to Claude CLI. For spawnAgent: reuse pkg/routing for validation (LoadAgentConfig, ValidateRelationship) and internal/lifecycle for process management. Build the claude CLI command with proper flags and environment, spawn as subprocess, capture stdout, send agent registration to TUI via UDS. For teamRun: invoke the gogent-team-run binary as a background process. For testMcpPing: return static {status: 'pong'} response. Define the UDS protocol types in internal/tui/mcp/protocol.go: IPCRequest{Type, ID, Payload json.RawMessage} and IPCResponse{Type, ID, Payload json.RawMessage}. Request types: modal_request, agent_register, agent_update, agent_activity, toast. Response types: modal_response.
