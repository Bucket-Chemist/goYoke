# TUI-036: Component unit tests (teatest)

## Description

Create comprehensive unit tests for all Bubble Tea components using the charmbracelet/x/exp/teatest package (or direct function call testing). Files: internal/tui/components/*/..._test.go for each component. Test strategy: (1) For each component, test Update() with mock messages and verify state changes. (2) Test View() output contains expected strings. (3) Test keyboard handling (send tea.KeyMsg, verify state transition). (4) For the root AppModel, test the full startup sequence: WindowSizeMsg -> ready=true, CLI events -> state updates, key routing -> correct child receives input. Create test fixtures in internal/tui/testdata/: sample NDJSON events (system.init, assistant, result), sample agent tree data, sample team configs. Test the re-subscription pattern: after each CLI event, verify WaitForEvent cmd is returned. Test modal queue: send multiple ModalRequestMsg, verify sequential processing. Test auto-scroll behavior: add messages, verify viewport at bottom, scroll up, verify auto-scroll disabled, add message at bottom, verify auto-scroll re-enabled.
