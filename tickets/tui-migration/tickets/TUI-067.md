# TUI-067: Two-step confirmation dialogs

## Description

Add a two-step type-to-confirm pattern for destructive actions. When the user initiates a destructive action (e.g., clearing conversation history, deleting a session), they must type a specific phrase to confirm. This prevents accidental execution.

Create `internal/tui/components/modals/confirm2.go`:
```go
package modals

// TwoStepConfirmModal requires the user to type a confirmation phrase.
// The action only executes when the typed phrase exactly matches RequiredPhrase.
type TwoStepConfirmModal struct {
    Title          string
    Description    string   // explains what will happen
    RequiredPhrase string   // e.g. "delete session"
    typedPhrase    string   // what the user has typed so far
    input          textinput.Model
    confirmed      bool
}

func NewTwoStepConfirmModal(title, description, requiredPhrase string) TwoStepConfirmModal
func (m TwoStepConfirmModal) Init() tea.Cmd
func (m TwoStepConfirmModal) Update(msg tea.Msg) (TwoStepConfirmModal, tea.Cmd)
func (m TwoStepConfirmModal) View() string

// IsConfirmed returns true when the typed phrase matches RequiredPhrase (case-insensitive).
func (m TwoStepConfirmModal) IsConfirmed() bool
```

Add `TwoStepConfirm` to the `ModalType` enum in `types.go`.

**Matching:** Case-insensitive. Partial matches show progress (e.g., phrase is 60% complete, color the matched portion). On exact match, the confirm button activates.

**Styling:** Uses `DangerStyle()` border from TUI-066. Matched portion of phrase rendered in green, unmatched in red.
