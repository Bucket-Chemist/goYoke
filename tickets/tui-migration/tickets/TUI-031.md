# TUI-031: Provider session resume (P7-7d)

## Description

Implement per-provider session resume in internal/tui/state/provider.go. When the TUI starts with --session-id, restore per-provider session state from the saved session data. Each provider has its own session ID (stored in SessionData.ProviderSessionIDs map). On resume: (1) load SessionData from session.json, (2) populate ProviderState.SessionIDs from saved data, (3) start CLIDriver with --resume flag using the active provider's session ID. On provider switch: if the switched-to provider has a saved session ID, pass --resume to the new CLIDriver instance. SessionData persistence (from task-033) must save ProviderState.SessionIDs. Also handle --model flag per-provider: if the user specified --model on startup, it applies only to the active provider. Subsequent provider switches use the per-provider saved model.
