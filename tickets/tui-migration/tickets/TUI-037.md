# TUI-037: CLI driver integration test

## Description

Create integration tests for the CLI driver in internal/tui/cli/driver_integration_test.go. Create a mock Claude CLI script (internal/tui/testdata/mock-claude.sh) that emits known NDJSON events on stdout and accepts messages on stdin. The script should: (1) emit a system.init event with test session ID and tools, (2) wait for stdin input, (3) emit an assistant event with a test response, (4) emit a result event with cost and token data, (5) exit cleanly on SIGINT. Test scenarios: (a) Normal flow: start driver, receive system.init, send message, receive response, receive result. (b) Error recovery: mock CLI exits unexpectedly, verify CLIDisconnectedMsg and reconnection attempt. (c) Interrupt: send Interrupt(), verify SIGINT received by mock. (d) Shutdown: send Shutdown(), verify SIGTERM then SIGKILL sequence. (e) Unknown event: emit an event with unknown type, verify CLIUnknownMsg and continued parsing. (f) Permission denial: emit a result event with permission_denials array, verify the denied tool input is captured correctly. Use testing timeouts (5s per test case) to prevent hangs.
