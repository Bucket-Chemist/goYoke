# TUI-008: Root AppModel scaffold + message types

## Description

Create the root application model and centralized message types. Files: internal/tui/model/app.go (root AppModel struct with Init/Update/View), internal/tui/model/messages.go (all tea.Msg types). The AppModel struct should contain: width/height int, focus FocusTarget, activeTab TabID, rightPanelMode RightPanelMode, ready bool (set on first WindowSizeMsg). Child model fields are initially empty structs that will be filled by later tickets. For now include placeholder types: BannerModel, TabBarModel, ClaudePanelModel, AgentTreeModel, AgentDetailModel, StatusLineModel, ToastModel. Also include: modalQueue []ModalRequest, modalActive *ModalModel, session SessionState, keys KeyMap, program *tea.Program. The messages.go file should define all tea.Msg types: SystemInitMsg, StatusUpdateMsg, CompactMsg, AssistantMsg, ToolResultMsg, ResultMsg, StreamEventMsg, ModalRequestMsg, ModalResponseMsg, AgentRegisteredMsg, AgentUpdatedMsg, AgentActivityMsg, TeamUpdateMsg, ToastMsg, TickMsg, CLIEventMsg (raw fallback). Each message type should be a struct with the relevant fields. The Update() method should handle tea.WindowSizeMsg (store dimensions, set ready=true, propagate to children) and tea.KeyMsg (route to handleKey which dispatches based on focus). The View() method should return 'Initializing...' if not ready, otherwise render a placeholder layout string. Init() should return tea.Batch(tea.EnterAltScreen).
