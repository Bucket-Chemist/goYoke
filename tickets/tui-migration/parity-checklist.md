### Feature 7: Context Compaction Notification

**Status:** pass
**Test method:** code-review

**How verified:**

- `cli/events.go` `SystemStatusEvent` covers `subtype="compact_boundary"` (the compaction event type documented in the NDJSON catalog). The `parseSystemEvent` default branch routes all non-init/hook subtypes to `SystemStatusEvent`, preserving the `Status` and `PermissionMode` fields.
- `app.go` handles `cli.SystemHookEvent, cli.SystemStatusEvent, cli.RateLimitEvent, cli.StreamEvent, cli.CLIUnknownEvent` in a single re-subscribe arm — the model remains responsive when a compaction boundary is emitted.
- The NDJSON catalog (TUI-003 spike) documented `compact_boundary` as a known-but-unobserved subtype; the parser handles it without dropping the event or panicking.

**Note:** The TUI does not currently display a visible compaction notification in the UI — the event is received and re-subscribed but not surfaced as a toast or status update. A future enhancement could forward `SystemStatusEvent{Subtype:"compact_boundary"}` to the toast component.

### Feature 14: Graceful Shutdown (Within 10s)

**Status:** pass
**Test method:** automated

**How verified:**

- `internal/tui/lifecycle/shutdown.go` implements `ShutdownManager` with a 5-phase LIFO sequence (session save → CLI driver shutdown → bridge shutdown) with a 10-second total budget.
- `ShutdownManager.Shutdown` is idempotent (protected by `sync.Once`) and returns `ErrShutdownTimeout` if the budget is exceeded.
- `app.go` `handleKey` handles `ForceQuit` (Ctrl+C): first press initiates `shutdownFunc()` which returns `ShutdownCompleteMsg`; second press forces immediate `tea.Quit`.
- SIGTERM/SIGINT are handled by `lifecycle.ProcessManager.StartSignalHandler` which calls `sm.Shutdown()`.
- `internal/tui/lifecycle/shutdown_test.go` tests: successful shutdown, timeout behaviour, double-call idempotency, signal handler wiring.

**10-second budget:** CLI driver gets 2s SIGTERM + 2s SIGKILL escalation; bridge gets 1s; session save is synchronous. Total worst case is within 10s.
