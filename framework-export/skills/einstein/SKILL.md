# Einstein Skill v1.0

## Purpose

Process escalated problems that exceeded lower-tier agent capabilities. Einstein receives **only** the GAP document context, avoiding Task tool inheritance overhead.

**What this skill does:**
1. Validate - Ensure GAP document is well-formed
2. Load - Read GAP document + best-practices reference
3. Analyze - Deep reasoning on bounded problem
4. Deliver - Output analysis to specified location

**What this skill does NOT do:**
- Accept vague prompts (requires GAP document)
- Inherit conversation context (isolation is the point)
- Execute implementations (analysis only, delegate execution)

---

## Invocation

| Command | Behavior |
|---------|----------|
| `/einstein` | Auto-detect most recent GAP doc in `.claude/tmp/` |
| `/einstein path/to/gap.md` | Process specified GAP document |
| `/einstein "question"` | Quick mode: generate minimal GAP, analyze |

## Workflow

### Phase 1: Document Detection

**If path provided:**
```bash
# Validate GAP document exists and is well-formed
~/.claude/scripts/validate-gap-document.sh "$GAP_PATH"
```

**If no path (auto-detect):**
```bash
# Find most recent GAP document
GAP_PATH=$(ls -t .claude/tmp/einstein-gap-*.md 2>/dev/null | head -1)
if [[ -z "$GAP_PATH" ]]; then
    echo "[einstein] No GAP document found. Generate one via escalate_to_einstein or provide a path."
    exit 1
fi
```

**If inline question (quick mode):**
```
[einstein] Quick mode: Generating minimal GAP document...
```
Create ephemeral GAP with:
- Question as primary question
- No attempt log
- Current session context as architectural context

### Phase 2: Pre-Flight Check

```
[einstein] GAP Document: {{path}}
[einstein] Estimated tokens: {{count}}
[einstein] Escalated by: {{agent}}
[einstein] Primary question: {{question_preview}}

Proceed with analysis? (y/n/edit)
```

**On response:**
- `y` → Proceed to analysis
- `n` → Abort
- `edit` → Open GAP document for modification, then re-validate

### Phase 3: Load Context

```bash
# Read GAP document
cat "$GAP_PATH"

# Read best practices reference
cat ~/.claude/skills/einstein/references/best-practices.md
```

**Context budget:**
- GAP document: ~3-8K tokens (depending on excerpts)
- Best practices: ~2K tokens
- **Total input: ~5-10K tokens** (vs 60K+ with Task tool)

### Phase 4: Invoke Einstein

**Option A: Direct API (if ANTHROPIC_API_KEY set)**

```bash
# Check for API key
if [[ -n "$ANTHROPIC_API_KEY" ]]; then
    ~/.claude/scripts/invoke-einstein-api.sh "$GAP_PATH" "$OUTPUT_PATH"
else
    echo "[einstein] No API key. Using interactive mode."
    # Fall through to Option B
fi
```

**Option B: Interactive Analysis (no API key)**

Present combined context and analyze in current session:
```
[einstein] Analyzing in current context (no API key detected)...
```

Then perform analysis directly, using extended thinking.

### Phase 5: Deliver Output

```
[einstein] Analysis complete.
[einstein] Output: {{output_path}}
[einstein] Token usage: {{input_tokens}} in / {{output_tokens}} out
[einstein] Estimated cost: ${{cost}}

Archive GAP document? (y/n)
```

**On archive:**
- Move GAP doc to `.claude/gap_logger/escalated/`
- Move analysis to `.claude/gap_logger/resolved/`
- Add resolution metadata
