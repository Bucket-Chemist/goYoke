# ADR-028: JSONL Handoff Format with Markdown Rendering

**Status**: Accepted

**Date**: 2026-01-19

**Deciders**: [Team/Individuals]

**Technical Story**: GOgent-028 series - Session handoff generation

---

## Context

The bash `session-archive.sh` hook generates session handoffs as markdown-only:
- Overwrites `.claude/memory/last-handoff.md` each session
- Human-readable format
- No machine-queryable history
- No append-only JSONL log

### Problem Statement

We need session handoffs that are:
1. **Human-readable** for immediate review
2. **Machine-queryable** for retrospective analysis
3. **Append-only** for historical tracking
4. **Schema-versioned** for future evolution
5. **Compatible** with existing load-routing-context.sh hook

### Constraints

- Must maintain compatibility with SessionStart hook (reads markdown)
- Must support querying across sessions (e.g., "violations last 10 sessions")
- Must be testable (unit tests for JSONL parsing, integration tests for workflow)
- Must handle schema evolution (versioning)

---

## Decision

**Implement dual-format handoff generation**:

1. **Primary Format: JSONL** (`handoffs.jsonl`)
   - Append-only log (never overwritten)
   - Machine-readable JSON Lines format
   - Schema versioning via `schema_version` field
   - Queryable history for retrospectives

2. **Secondary Format: Markdown** (`last-handoff.md`)
   - Rendered from JSONL on-demand
   - Human-readable for immediate review
   - Overwritten each session (latest handoff)
   - Compatible with load-routing-context.sh

### Implementation

```
SessionEnd → gogent-archive
            ↓
  GenerateHandoff(metrics)
            ↓
  Append to handoffs.jsonl (JSONL)
            ↓
  LoadHandoff(handoffs.jsonl)
            ↓
  RenderMarkdown(handoff) → last-handoff.md
            ↓
  ArchiveArtifacts()
```

### Schema

JSONL schema (v1.0):
```json
{
  "schema_version": "1.0",
  "timestamp": 1234567890,
  "session_id": "session-abc123",
  "context": {
    "metrics": {"tool_calls": 42, "errors": 5, "violations": 3},
    "git_info": {"branch": "main", "dirty": false},
    "active_ticket": "GOgent-028"
  },
  "artifacts": {
    "sharp_edges": [...],
    "violations": [...],
    "error_patterns": [...]
  },
  "actions": ["Review sharp edges", "Fix violations"]
}
```

Markdown rendered sections:
- Session Metrics
- Sharp Edges (formatted tables)
- Routing Violations (top 10)
- Recommended Actions

## Alternatives Considered

### Alternative 1: Markdown-Only (Status Quo)

**Pros**:
- Simple implementation
- Human-readable
- Compatible with existing hooks

**Cons**:
- ❌ No queryable history
- ❌ Can't answer "violations last 10 sessions"
- ❌ Overwrites previous handoff (data loss)
- ❌ No schema versioning
- ❌ Difficult to parse programmatically

**Rejected**: Doesn't support retrospective analysis.

### Alternative 2: JSONL-Only

**Pros**:
- Machine-queryable
- Append-only history
- Schema-versioned

**Cons**:
- ❌ Not human-readable
- ❌ Breaks load-routing-context.sh compatibility
- ❌ Requires jq or custom tooling for immediate review

**Rejected**: Breaks existing SessionStart hook, poor UX.

### Alternative 3: SQLite Database

**Pros**:
- Queryable with SQL
- Efficient indexing
- Relational data support

**Cons**:
- ❌ Adds dependency (sqlite3)
- ❌ Binary format (not human-readable)
- ❌ Requires schema migrations
- ❌ Overkill for append-only log
- ❌ Breaks load-routing-context.sh compatibility

**Rejected**: Too complex for use case.

### Alternative 4: Dual Format (Chosen)

**Pros**:
- ✅ Human-readable (markdown)
- ✅ Machine-queryable (JSONL)
- ✅ Append-only history (JSONL)
- ✅ Schema-versioned (JSONL)
- ✅ Hook-compatible (markdown)
- ✅ Testable (both formats)

**Cons**:
- ⚠️ Slightly more complex (two file writes)
- ⚠️ Potential inconsistency if markdown render fails

**Accepted**: Best balance of all requirements.

---

## Consequences

### Positive

1. **Queryable History**:
   - `gogent-archive list` shows all sessions
   - `gogent-archive stats` aggregates metrics
   - Custom scripts can parse JSONL

2. **Schema Evolution**:
   - `schema_version` field enables future migrations
   - Old JSONL entries readable with versioned parsers

3. **Human UX**:
   - Markdown rendering for immediate review
   - Compatible with existing workflows

4. **Testability**:
   - Unit tests for JSONL generation
   - Integration tests for markdown rendering
   - Schema validation tests

5. **Append-Only**:
   - Never lose historical data
   - Supports trend analysis over time

### Negative

1. **Implementation Complexity**:
   - Requires two file writes (JSONL append + markdown render)
   - Requires synchronization (render from JSONL, not independently)

2. **Disk Usage**:
   - Append-only JSONL grows over time
   - Mitigation: Archival rotation after N sessions (future enhancement)

3. **Potential Inconsistency**:
   - If markdown render fails, JSONL exists but markdown doesn't
   - Mitigation: RenderMarkdown is non-fatal; JSONL is source of truth

### Migration Path

From bash markdown to Go JSONL:

1. **Phase 1**: Deploy Go hook, generate both formats
2. **Phase 2**: Verify load-routing-context.sh works with Go markdown
3. **Phase 3**: Retire bash hook
4. **Phase 4**: Optionally migrate old bash handoffs to JSONL (best-effort)

### Future Enhancements

- **Querying API**: REST API for handoff queries
- **Dashboard**: Web UI for session trends
- **Rotation**: Auto-archive JSONL after 100 sessions
- **Compression**: gzip old JSONL entries
