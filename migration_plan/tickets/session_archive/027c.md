# GOgent-027c: Implement AnalyzeToolDistribution()

## Description
Analyze tool usage patterns from parsed transcript to understand session characteristics.

## Implementation Intention
Count tool usage (Read, Edit, Write, Task, Bash, etc.) to enable session phase detection and adaptive handoff formatting.

## Intended End State
- Function returns map of tool names to usage counts
- Example output: {"Read": 30, "Edit": 10, "Task": 2}
- Handles empty events slice
- Test coverage ≥80%

## Dependencies
- GOgent-027b: ParseTranscript() must be complete

## Acceptance Criteria
- [ ] AnalyzeToolDistribution(events []ToolEvent) map[string]int implemented
- [ ] Counts occurrences of each tool type
- [ ] Handles empty events slice (returns empty map)
- [ ] Handles unknown tool names gracefully
- [ ] Test coverage ≥80%
- [ ] Integration test with real transcript patterns

## Implementation Details

### Files Modified
- `pkg/session/transcript.go` - Add new function

### Function Signature
```go
func AnalyzeToolDistribution(events []ToolEvent) map[string]int {
	distribution := make(map[string]int)
	for _, event := range events {
		distribution[event.ToolName]++
	}
	return distribution
}
```

### Expected Output Format
```go
map[string]int{
	"Read":  30,
	"Edit":  10,
	"Write": 5,
	"Task":  2,
	"Bash":  8,
	"Glob":  12,
	"Grep":  15,
}
```

### Test Requirements
- Test: Empty events slice (returns empty map)
- Test: Single tool type only (e.g., 10x Read)
- Test: Mixed tool types
- Test: Unknown tool names (should still count)
- Test: Verify count accuracy

## Why This Matters
Tool distribution is the foundation for session phase detection. A session with 70% Read/Glob/Grep is "discovery", while 70% Edit/Write is "implementation". This enables adaptive handoff formatting.
