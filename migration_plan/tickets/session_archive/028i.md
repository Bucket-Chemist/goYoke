# GOgent-028i: Error Recovery & Fallback

## Description
Create wrapper script that tries Go CLI first, falls back to bash hook on failure. Provides production safety net during migration period.

## Implementation Intention
Ensure SessionEnd hook never fails completely even if Go CLI crashes.

## Intended End State
- Wrapper script `~/.claude/hooks/session-archive-wrapper.sh`
- Tries gogent-archive first
- Falls back to bash on error
- Logs failures for debugging
- Production-safe deployment

## Implementation Details

### Wrapper Script
File: `scripts/session-archive-wrapper.sh`

```bash
#!/bin/bash
# Session Archive Hook with Go/Bash Fallback
set -euo pipefail

# Read STDIN into temp file (allows retry)
TMPFILE=$(mktemp)
trap "rm -f $TMPFILE" EXIT
cat > "$TMPFILE"

# Try Go implementation first
if command -v gogent-archive &>/dev/null; then
    if gogent-archive < "$TMPFILE" 2>/dev/null; then
        # Success
        echo "[INFO] Go hook succeeded" >&2
        exit 0
    else
        EXIT_CODE=$?
        echo "[ERROR] Go hook failed with exit code $EXIT_CODE" >&2
        echo "$(date): gogent-archive failed (exit $EXIT_CODE)" >> ~/.gogent/hook-failures.log
    fi
else
    echo "[WARN] gogent-archive not found in PATH" >&2
fi

# Fallback to bash hook
echo "[INFO] Falling back to bash hook" >&2
if [[ -f ~/.claude/hooks/session-archive.sh ]]; then
    ~/.claude/hooks/session-archive.sh < "$TMPFILE"
    echo "$(date): Bash fallback succeeded" >> ~/.gogent/hook-failures.log
    exit 0
else
    echo "[ERROR] Bash hook not found at ~/.claude/hooks/session-archive.sh" >&2
    exit 1
fi
```

### Installation
Add to `Makefile`:

```makefile
install-wrapper:
	@echo "Installing session-archive wrapper hook..."
	mkdir -p ~/.claude/hooks
	mkdir -p ~/.gogent
	cp scripts/session-archive-wrapper.sh ~/.claude/hooks/session-archive-wrapper.sh
	chmod +x ~/.claude/hooks/session-archive-wrapper.sh
	@echo "✅ Wrapper installed"
	@echo ""
	@echo "Update hook config to use:"
	@echo "    command = \"~/.claude/hooks/session-archive-wrapper.sh\""
```

```bash
#!/bin/bash
set -euo pipefail

echo "Testing Go → Bash fallback..."

# Simulate Go failure by using invalid binary path
export PATH="/nonexistent:$PATH"

TMPDIR=$(mktemp -d)
trap "rm -rf $TMPDIR" EXIT

# Create bash hook mock
mkdir -p "$TMPDIR/.claude/hooks"
cat > "$TMPDIR/.claude/hooks/session-archive.sh" <<'EOF'
#!/bin/bash
# Mock bash hook
cat >/dev/null  # Consume STDIN
echo '{"hookSpecificOutput":{"source":"bash_fallback"}}'
EOF
chmod +x "$TMPDIR/.claude/hooks/session-archive.sh"

# Run wrapper (should fall back to bash)
export HOME="$TMPDIR"
cat > "$TMPDIR/session.json" <<EOF
{"session_id":"test","timestamp":123,"hook_event_name":"SessionEnd"}
EOF

OUTPUT=$(scripts/session-archive-wrapper.sh < "$TMPDIR/session.json" 2>&1)

if echo "$OUTPUT" | grep -q "bash_fallback"; then
    echo "✅ Fallback to bash hook succeeded"
else
    echo "❌ Fallback failed"
    echo "$OUTPUT"
    exit 1
fi
```

## Why This Matters
Addresses GAP-028-8: No error recovery (MINOR). During migration rollout, Go CLI bugs may surface in production. Without fallback, SessionEnd hook fails and sessions end without handoff archival. Wrapper provides safety net.

**Production Safety**: Guarantees handoff always generated (via Go or bash).
