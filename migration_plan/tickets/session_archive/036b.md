# GOgent-036b: Track Failures by File + Error Type

## Description
Refine failure tracking to use file + error_type composite key instead of file-only, preventing false positives.

## Implementation Intention
Avoid triggering sharp edge detection when 3 different errors occur on same file. Only trigger when same error occurs 3 times.

## Intended End State
- FailureKey struct with FilePath and ErrorType
- CountRecentFailures() uses composite key
- Prevents false positives (different errors on same file)
- Preserves true positives (same error multiple times)
- Test coverage ≥80%

## Dependencies
- GOgent-036: Base failure tracking complete

- [ ] FailureKey struct defined with FilePath and ErrorType fields
- [ ] CountRecentFailures(key FailureKey) uses composite key for filtering
- [ ] Test: 3 different errors on same file (should NOT trigger)
- [ ] Test: Same file+error 3 times (should trigger)
- [ ] Test: Same error on different files (tracked separately)

```go
type FailureKey struct {
	FilePath  string
	ErrorType string
	Function  string // Optional, for GOgent-036c
}

// String returns canonical key representation
func (k FailureKey) String() string {
	if k.Function != "" {
		return fmt.Sprintf("%s:%s:%s", k.FilePath, k.ErrorType, k.Function)
	}
	return fmt.Sprintf("%s:%s", k.FilePath, k.ErrorType)
}
```

### Updated CountRecentFailures
```go
func (ft *FailureTracker) CountRecentFailures(key FailureKey) (int, error) {
	f, err := os.Open(ft.ErrorLogPath)
	if err != nil {
		if os.IsNotExist(err) {
			return 0, nil
		}
		return 0, fmt.Errorf("[failure-tracker] Failed to open error log: %w", err)
	}
	defer f.Close()

	cutoff := time.Now().Unix() - int64(ft.FailureWindow)
	count := 0

	scanner := bufio.NewScanner(f)
	var lines []string
	for scanner.Scan() {
		lines = append(lines, scanner.Text())
		if len(lines) > 50 {
			lines = lines[1:]
		}
	}

	for _, line := range lines {
		var entry FailureEntry
		if err := json.Unmarshal([]byte(line), &entry); err != nil {
			continue
		}

		// Match on FilePath AND ErrorType
		if entry.File == key.FilePath &&
			entry.ErrorType == key.ErrorType &&
			entry.Timestamp > cutoff {
			count++
		}
	}

	return count, nil
}
```

## Why This Matters
File-only tracking has false positive rate. Three unrelated errors on one file shouldn't trigger sharp edge detection. Composite key ensures only genuine debugging loops are captured.

**Example False Positive**:
```
File: src/main.go
- Line 10: TypeError (wrong assertion)
- Line 50: ImportError (missing import)
- Line 100: ValueError (bad input)
→ Without composite key: 3 failures, sharp edge triggered
→ With composite key: 1 failure each, no trigger (correct)
```
